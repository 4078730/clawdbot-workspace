## 全体フロー

```
STEP 0 上司確認・予算コード確認
  ↓
STEP 1 ホテル・航空券の見積もり取得
  ↓
STEP 2 EPOCH で海外渡航申請（発券は承認後）
  ↓
STEP 3 ESTA 申請（アメリカの場合）
  ↓
STEP 4 各種手配確定（発券・ホテル確定）
  ↓
STEP 5 出張実施
  ↓
STEP 6 帰国後精算
```

参考：[らくナビ - 海外出張のフロー](https://jppanasonic.sharepoint.com/sites/ONL02392/SitePages/%E5%87%BA%E5%BC%B5%E3%81%99%E3%82%8B.aspx#%E6%B5%B7%E5%A4%96%E5%87%BA%E5%BC%B5%E3%81%AE%E3%83%95%E3%83%AD%E3%83%BC)

## STEP 0：事前確認

上司に確認すること：

- 出張の目的・効果
- 渡航の可否
- 予算コード（例：ZA600）
- チケット・WiFi等の立替可否・仮出金可否（経理にも確認）

---

## STEP 1：ホテル・航空券の見積もり

### ホテル手配：HRS

社内で実績のある最安予約サービス。

1. [HRS](https://www.hrs.com/) で目的地周辺を検索
2. 公共交通1時間以内・価格順でフィルタ
3. 「キャンセル可（2日前まで）」プランを選ぶ
4. 予約確認画面・金額をPDF保存（EPOCH添付用）

> **2026-02 実績ホテル（アメリカ出張）**
> - ヒューストン：[Courtyard by Marriott Houston The Woodlands](https://www.marriott.com/en-us/hotels/houwd-courtyard-houston-the-woodlands/overview/)
> - シアトル/ベルビュー：[Courtyard Seattle Bellevue Downtown](https://www.marriott.com/en-us/hotels/bvudt-courtyard-seattle-bellevue-downtown/overview/)

### 航空券見積：MERON → エムハートツーリスト

社内手続きと連動しているため、原則このルートを使う。

1. MERON「海外出張 → 航空券予約 → 依頼フォーム」から見積依頼
2. 記載内容（例）：

```
【出張者】氏名
【予算コード】ZA600
【フライト希望（全区間）】

① 羽田 → サンフランシスコ
   NH108 / 日付 22:55発 / エコノミー

② サンフランシスコ → ヒューストン
   UA1672 / 日付 14:47発 / エコノミー

③ ヒューストン → シアトル
   UA366 / 日付 10:02発 / エコノミー

④ シアトル → 羽田
   NH117 / 日付 16:45発 / エコノミー
```

3. エムハートから仮予約メールが届く（発券はEPOCH承認後）
4. フライト明細を保存

> ⚠️ 2025年12月より「B+PLUS」→「**ビズバンス**」、「B-TORU」→「**plover**」に移行済み

## STEP 2：EPOCH 海外渡航申請

### 入力前の準備（PDF化しておくもの）

1. ホテル予約確認書（HRS最終画面）
2. 渡航決裁用PDF（参考：MIT海外渡航決裁.pdf）
3. WiFiレンタル受付内容（後述）

### 入力のポイント

| 項目 | 記入例 |
|------|--------|
| 負担区分 | プロジェクト番号（例：ZA600） |
| 出張目的 | 技術協業先訪問、研究機関視察 等 |
| 費用補足 | 予約元（HRS・MERON等）と為替換算を記録 |

### 承認ルート（参考）

| 区分 | 担当者 |
|------|--------|
| 部門1 | 出張者本人 |
| 部門2 | 大坪 幹二（部門長） |
| 輸出管理 | 齋藤 政夫（PCO所属） |
| 人事 | 川口 健一郎 |
| 経理 | 松永 大 |
| 最終承認 | 大坪 幹二（ディレクター） |

> ⚠️ **航空券・ホテルの「発券・確定」はEPOCH承認後に実施すること**（変更・払戻に影響あり）

## STEP 3：ESTA 申請（アメリカ渡航必須）

- 公式サイト：**https://esta.cbp.dhs.gov/**（$21、約¥3,000）
- 代行サイトに注意（見た目が本物そっくりで高額請求される）
- 通常は数時間〜数日で承認、余裕を持って申請すること
- 承認済みのESTAは2年間有効（期限内なら再申請不要）

## STEP 6：帰国後精算

- 経理サイト：[経理・財務本部](https://jppanasonic.sharepoint.com)

精算時の自由記述欄（ZP000補填の場合）：
```
R&Dプロジェクト費（ZP000）にて補填
（1/9に承認いただきました）
```

## 延泊（個人旅行）を出張に組み込む場合

業務出張の前後に個人旅行を追加する場合の注意点：

| 項目 | 扱い |
|------|------|
| 個人滞在日のホテル | **自費**（精算不可） |
| フライト費用 | 出張相当日程のチケット代まで会社負担、差額は自費 |
| 労災 | 個人行動中は**適用外**。書面での明示が必要 |
| EPOCH申請 | 渡航期間を延長日まで変更して申請 |
| 承認 | 上司承認＋人事承認が必要（口頭のみは不可） |

合理的な理由（技術視察・研究機関訪問等）があれば、渡航期間の延長として申請しやすい。

## よくある失敗

| 失敗 | 対策 |
|------|------|
| ESTAを直前に申請して承認が間に合わない | 出張決定後すぐに申請 |
| 代行サイトでESTAを高額取得 | 必ず公式 `esta.cbp.dhs.gov` から |
| EPOCH承認前に発券してしまう | 仮予約状態のまま待つ |
| 個人延泊の労災リスクを口頭だけで処理 | 必ず書面で「個人旅行」と明示 |
