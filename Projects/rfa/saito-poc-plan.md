### 1.3 α版で扱う複雑さ

α版では「オペレーションの標準化が不可能」な部分はスコープ外とし、
**商品の形状・包装状態の複雑さ**に対するVLA制御の有効性を実証する。

具体的にα版が扱う課題：

| 課題 | 具体例 |
|------|--------|
| **形状の多様性** | 天板（平面パーツ：端の把持点が一意に定まらない）、不定形パーツ、小型部品 |
| **包装状態の多様性** | 段ボール内のプチプチ包装、袋入り、仕切り付きビン |
| **立てかかってつかみにくい** | 掴む場所が決めにくい、一度で掴めない |
| **段ボールの中に入っている** | 下に埋まっている |

→ **VLAによる操作制御ポリシー**で、形状・状態に応じた把持・取り出し動作を生成

**α版のスコープ外（β版以降）：**
- 個装（袋入れ・納品書同封）
- 梱包（段ボール開封含む）
- 移動（台車配置）
- 複数スキル切替のAgenticな制御

## 2. フェーズ定義

### 設計原則

> **ルールベースとVLAの役割分担:**
> VLAはルールベースの「置き換え」ではなく「段階的補完」。
> ルールベースで対応可能なものはルールベースで処理し、VLAが困難なタスクを補完。
> 段階的にVLAの対応範囲を拡大し、最終的にVLA主力へ移行。

> **複雑さの3軸（時間的に分離して取り組む）:**
> 1. **手の複雑さ** — エンドエフェクタの種類・指の制御
> 2. **腕の複雑さ** — 単腕/双腕、操作制御の汎用性
> 3. **移動の複雑さ** — 固定ステーション / 移動を伴う作業

### 2.1 α版（CA: Controlled Availability）— Q2末/9月

**ゴール:** RoboSync上でVLA制御ポリシーが動作し、形状・包装状態が複雑なパーツのピック → 作業台配置を実現する

#### スコープ

| 項目 | 内容 |
|------|------|
| **IN** | ビン内パーツのピック → 作業台に配置 |
| **OUT** | 個装、梱包、台車への移動、納品書処理、Agentic制御 |
| **対象パーツ** | 形状・包装状態が複雑なパーツ（具体ケースはRoboSyncチームと選定） |
| **人の役割** | 個装・梱包・納品書処理は人が担当。リカバリー作業も人が実施（人×ロボット協働） |

**人の介在について:**
ロボットのピック失敗・異常時には人が手動でリカバリーを行う。
β版以降では、テレオペレーション（遠隔操作）での介入や、UMI等での操作ログ記録による
リカバリーデータ蓄積を目指すが、α版では開発工数を考慮し手動リカバリーとする。

#### ハードウェア

| 項目 | 仕様 |
|------|------|
| **ロボットアーム** | UR5（単腕） |
| **エンドエフェクタ** | 吸着 / 平行グリッパ（タスクに応じて選択。要：奥村さんと検討） |
| **指** | 使用しない（研究検討中。指の制御自体が研究対象） |
| **カメラ** | Intel RealSense D405（チーム共通） |
| **移動機構** | なし（固定ステーション） |

#### 技術要件

| # | 要件 | 詳細 |
|---|------|------|
| T-1 | VLA制御ポリシー | 形状・包装状態が複雑なパーツに対して、操作軌道を生成・実行 |
| T-2 | RoboSync統合 | VLAスキルブロックとしてRoboSync上で実装・実行・管理 |
| T-3 | ログ収集 | 実行ログ（画像・ロボット状態・成功/失敗）の記録。学習に直接使えるかは未定だが、少なくともログとして残す |
| T-4 | 安全制御 | RoboSync標準安全制約に準拠。物理的な怪我につながる危険箇所は絶対に対処。どのレイヤーまで安全性を担保するかは要定義 |
| T-5 | ルールベース連携 | ルールベースで対応可能なケースはルールベースで処理。VLAは困難なケースを補完 |

※ Agentic要素（複数スキル切替、自律的再計画等）はα版には含まない

#### 成功基準（仮）

※ 期初時点の目安。現場や実際のタスクや技術レベルに合わせて調整。

| 指標 | 目標 | 備考 |
|------|------|------|
| ピック成功率 | 90% | 対象パーツを絞った上での成功率 |
| リトライ込み成功率 | 95% | 1回失敗後のリトライを含む |
| ログ蓄積 | 稼働 | 実行ログが記録される状態 |
| 安全性 | インシデント0 | 物理的な怪我に繋がる事故は絶対にゼロ |

#### 事業的価値（内部レビュー向け説明）

> 彩都パーツセンターの出庫作業では、ビン内のパーツが多様な形状・包装状態で存在する。
> 例えば、平面パーツ（天板等）は端の把持点が一意に定まらず、
> 不定形パーツやプチプチに包まれた状態のパーツは吸着のみでは対応不可能。
>
> α版では、こうした**形状・状態の複雑さ**に対してVLA制御ポリシーが有効であることを、
> 「ビンからパーツをピックして作業台に置く」という限定されたスコープで実証する。
> 個装は人が担当し、人×ロボット協働でオペレーションを成立させる。
>
> ※ 具体的な対象パーツの例と「ビンからピック→作業台配置」の図解

### 2.2 β版 — Q4末/3月

**ゴール:** 固定ステーション内の出庫作業サイクル全体をカバー。Agenticな制御で複数スキルを切り替えながらサイクルを回す。「ちゃんと動くこと」が最優先。

#### スコープ

| 項目 | 内容 |
|------|------|
| **IN** | 固定ステーション内の出庫作業サイクル全体（下記参照） |
| **OUT** | 出荷用の梱包作業、ステーション外への移動 |
| **個装** | ✅ 含む |
| **台車配置** | ✅ 含む（作業空間内の姿勢変化） |
| **Agentic制御** | ✅ 含む（複数スキル切替・サイクル管理に必須） |
| **対象パーツ** | α版で扱った範囲を多品種に拡張 |

**β版の作業サイクル（Agenticが全体を制御）：**

```
┌─────────────────────────────────────────────────────────┐
│ β版: 固定ステーション内の出庫作業サイクル全体            │
│                                                         │
│  [1] バーコードスキャン（納品書読み取り）                │
│       ↓                                                │
│  [2] モニター操作（ビン呼び出し）                       │
│       ↓                                                │
│  [3] ビン内段ボール開封（必要時）                       │
│       ↓                                                │
│  [4] パーツのピック（α版を拡張）                        │
│      ├─ 緩衝材からの取り出し                           │
│      └─ 指定数量のピック（「3個取り出す」等）           │
│       ↓                                                │
│  [5] 残りをビンに戻す                                  │
│       ↓                                                │
│  [6] 個装（袋入れ + 納品書同封 + 袋閉じ）              │
│       ↓                                                │
│  [7] 台車/ワゴンに配置（作業空間内の姿勢変化）         │

└─────────────────────────────────────────────────────────┘
```

#### β版で加わるタスク詳細

**ピック関連（α版からの拡張）：**

| タスク | 内容 | 技術的課題 |
|--------|------|-----------|
| 段ボール開封 | ビン内の段ボールの封を開ける | 双腕（押さえ+開封）、力制御 |
| 緩衝材からの取り出し | プチプチ等から1個取り出す | VLA制御の精度 |
| ビンへの戻し | 取り出し後、残りをビンに戻す | 慎重な配置制御 |
| 数量指定ピック | 袋から指定数だけ取り出す | 認識+カウント+判断 |

**個装：**

| タスク | 内容 | 技術的課題 |
|--------|------|-----------|
| 袋入れ | パーツを専用袋に入れる | 双腕（袋を開く+パーツを入れる） |
| 納品書同封 | 袋の中にパーツと納品書をセット | 双腕 |
| 袋閉じ | 袋を密封する | 双腕、柔軟物操作 |
| 輪ゴム留め | パーツと納品書を輪ゴムでくくる | 双腕必須、高難度 |

**システム連携：**

| タスク | 内容 | 技術的課題 |
|--------|------|-----------|
| バーコードスキャン | コードリーダーで納品書をスキャン | ツール操作、位置合わせ |
| モニター操作 | 画面操作で次のビンを呼び出す | UI操作の自動化、システム連携 |

**配置：**

| タスク | 内容 | 技術的課題 |
|--------|------|-----------|
| 台車/ワゴンへの配置 | 個装済み商品を台車に載せる | 作業空間内での姿勢変化、リーチ |

**Agentic制御：**

| タスク | 内容 | 技術的課題 |
|--------|------|-----------|
| スキル切替 | [1]〜[8]の工程間をシームレスに繋ぐ | 状況に応じた切替判断 |
| 商品識別→手順選択 | 商品ごとに異なる個装手順を判断 | 意味理解と手順計画 |
| 数量管理 | 指定数のピック、在庫確認 | カウント+判断ループ |
| 異常検知→再計画 | ピック失敗、間違い商品等 | 状態判定→代替手順の自律選択 |
| サイクル管理 | 複数オーダーの順次処理 | タスクキュー管理 |

#### ハードウェア

| 項目 | 仕様 |
|------|------|
| **ロボットアーム** | 双腕（個装・段ボール開封に必須。具体機種は要選定） |
| **エンドエフェクタ** | 吸着 / 平行グリッパ |
| **指** | △ 要検討（研究は並行で進行中 — 福岡拠点） |

#### 技術要件

| T-7 | 段ボール開封 | ビン内段ボールの封を開ける制御（双腕・力制御） |
