# 彩都パーツセンター見学 議事録

**日時:** 2026年2月3日 午前  
**場所:** 彩都パーツセンター（MFLP茨木 6階）  
**参加者:** 黄瀬、榊原、宮島、一力、海藤  
**目的:** サービスパーツ倉庫の現場見学およびVLA適用可能性の検討

---

## 5行サマリー

1. **彩都パーツセンター**はサービスパーツの出荷拠点（5,000㎡、12万品番、3万点/月、24カ国出荷）
2. **オートストア出庫タスク**がVLA適用の有力候補：AGVが運んできたビンから人間が商品をピッキングする作業
3. **現在のピッキング時間**は54秒/件（2018年導入時160秒から改善）、目標30秒、将来は無人化
4. **VLA適用時の課題**：上積み保管による別商品回避、複数ビンからのピック、他ポートとの重複待機
5. **次ステップ**：実験室で現場環境を再現し、VLAによる作業データ収集を検討

### 3. 一力さんによるIndustrial Embodied AI取り組み紹介

#### 3.1 IE × DXの考え方

- **1秒の差が大きい**：1秒 × 1万回 = 3時間の差
- **作業者間のばらつき**：Aさん5秒、Bさん8秒、Cさん12秒 → 計画との乖離
- **サイバーとフィジカルのギャップ**：倉庫の充填率はシステム上のデータと実際で大きく異なる

### 4. AS出庫プロセス（VLA適用対象）

```
【オートストア出庫タスク】

①注文発生
  ↓
②AGVが倉庫内のビンを自動取得
  ↓
③出庫レーン（6箇所）にビン搬送
  ↓
④[人間作業] 売上伝票を確認
  ↓
⑤[人間作業] ビンから商品をピック
  ※上積み保管のため、別商品を避けて取る必要あり
  ※複数ビンからのピックあり
  ↓
⑥[人間作業] バーコードスキャン
  ↓
⑦[人間作業] 個装ラベル貼付
  ↓
⑧出荷エリアへ移動
```

**VLA適用範囲:** ④〜⑦の人間作業部分

### 5. VLA適用時の技術的課題

| 課題 | 内容 |
|------|------|
| 上積み保管 | 対象商品の上に別商品が存在する可能性 → 回避動作が必要 |
| 複数ビンピック | 1注文で複数ビンからピックするケース |
| ポート重複 | 他ポートと商品が重複した場合の待機判断 |
| 商品多様性 | 12万品番 → 形状・サイズの多様性への対応 |
| リカバリー | ピック失敗時のリトライ動作 |

### 6. VLA適用の可能性

#### 6.1 konukiの見解

> オートストア出庫の作業そのものがVLAで対応できるのではないか。現場で使われているコンテナや商品そのものを対象にして、実験室でその作業自体を記録することができれば、かなり貢献できるようになるはず。

#### 6.2 池内先生ディスカッションとの接続

- **GMR（Grasp-Manipulation-Release）**の観点：ビンから商品を取り出す動作は典型的なGMR
- **リカバリースキル**：上積み商品を避ける動作、ピック失敗時の再試行
- **継続学習**：現場での失敗データを蓄積し、改善ループを回す

### 7. 導入されているソリューション

| ソリューション | 内容 |
|---------------|------|
| 積載量可視化 | カート・コンテナ・棚の充填率自動計測 |
| V-SLAM | 非GPS空間での自己位置推定（フォークリフト・牽引車の動き把握） |
| ダッシュボード | 進捗管理、生産性分析、動線分析、収支分析、在庫管理の統合可視化 |
| AI画像処理 | 標準作業プロセスに基づくリアルタイム作業分解・分析 |
