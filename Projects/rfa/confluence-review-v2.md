## エグゼクティブサマリー

製造・物流現場の非定型作業を自律実行するロボット知能システムを現場実証する。

- **技術:** LLM Agent + VLA（視覚→行動生成）をRobo Sync上に統合
- **戦略:** 完全自律を待たず段階導入。導入現場からデータが蓄積され続ける構造で競争優位を構築
- **目標:** 9月α版（ピック→配置の現場検証） → 3月β版（出庫サイクル全体＋Agentic制御） ★変更
- **実証先:** 彩都パーツセンター（オートストア出庫作業） ★変更

## 1. プロジェクト内容

📄 詳細：FY26 RFA 事業計画書（§3. プロジェクト概要）

### 1.1 ビジョン

人の指示や意図を理解し、製造現場における「属人的」かつ「複雑な推論」を要するタスクを自律実行する知能システムを実現する。

### 1.2 技術アーキテクチャ

RFAは、言語指示を物理ロボット動作へつなぎ、自律的にタスクを完遂する知能システムである。
RFAは4つの技術レイヤーで構成される。

**Architecture（技術構成）**
- **Task Planning Hub（LLM Agent）：** 言語理解、タスク計画、推論、失敗時のリカバリー
- **Symbol Hub（Symbol Integration）：** 言語と物理の記号接地、VLMによる状態認識
- **Sensorimotor Hub（VLA）：** 視覚情報からの行動生成、複雑な非定型制御
- **Robo Sync + Data Platform：** 物理ロボット実行基盤、安全制御、データ収集・管理

**Target State（目指す姿）**
- 自然言語による指示：「この商品を梱包して」といった言語理解、タスク計画、推論、失敗時のリカバリー
- Agentによる自律判断：状況を認識し、最適な手順を自動選択
- 粘り強い実行と進化：失敗しても自動的にリトライ、データが蓄積され、使うごとに賢くなる

### 1.3 解決する課題

- **非定型作業への対応：** 袋物・箱物・柔軟物など形状・サイズ・素材が多様な物品に対し、従来のルールベース制御では個別のプログラミングが必要で現実的でない
- **暗黙知の形式化：** 熟練者が無意識に行う「重いものは下」「壊れ物は上」といった判断や、把持失敗時の持ち替えなど、言語化が困難な知識をシステムに埋め込む
- **労働力不足への対応：** 物流・製造現場では人手不足が構造的課題となっており、属人作業の自動化が急務

### 1.4 事業価値

- **省人化・品質安定・24h運用：** 入出庫・梱包等の属人作業を自動化し、人手に依存しない安定した現場運用を実現
- **データによる競争優位：** Robo Sync導入現場から蓄積される実運用データにより、BigTechが持ち得ない現場固有の学習資産を構築
- **Robo Syncとの相乗効果：** 既存のRobo Sync顧客基盤を活用し、RFAをオプション機能として提供することでスケール化

### 1.5 実行戦略：4つの柱

現場価値の実証のため、4つの柱にリソースを集中する。

| 柱 | キーワード | 内容 |
|----|----------|------|
| ① Robo Sync × VLA | 実行基盤 | VLAを研究モデルからRoboSyncの「スキルブロック」へ。安全な実行基盤の確立。 |
| ② DataOps | 資産化 | 現場の実行ログを学習データに変換するパイプラインを構築。「使うほど賢くなる」データフライホイールを構築。 |
| ③ Agent Operation | 自律適応 | 単なる成功だけでなく、失敗検知時の再計画、人とのやり取りの自律性の実現。 |
| ④ Field PoC | 現場実証・事業接続 | 彩都パーツセンターでの実運用検証。価値とコストの定量評価。 ★変更 |

### 1.7 勝ち筋（差別化戦略）

**段階的自律化：完全自律を待たずに価値を提供する**

RFAは「完全自律」を初期目標とせず、段階的に自律度を高める戦略を採る。Phase 1（FY26-27）ではRobo Syncのルールベース制御を主力とし、失敗した数%のケースをVLAがリカバリー、それでも失敗した場合は人が介在する。この「不完全だが価値がある」構成により、完全自律を待たずに顧客への省人化価値を提供しながら、データを蓄積していく。

**データフライホイール：価値提供とデータ蓄積の同時性**

BigTechは大規模テレオペ施設で「力技」でデータを収集する（Training-Time Scaling）。RFAは現場導入と同時にデータ収集が始まる構造を取る（Deploy-Time Scaling）。価値提供（顧客への省人化）とデータ収集（学習用蓄積）が同時に発生し、データ収集の追加コストはほぼゼロ。このサイクルが回るたびにOOD領域が縮小し、VLAの能力が向上していく。

**現場データが真の参入障壁**

Robo Sync導入現場から蓄積される実運用データ、特に失敗・リカバリーデータは、BigTechのテレオペ施設では得られない現場固有の学習資産となる。時間とともに蓄積されるデータ量の差は拡大する一方であり、後発参入者が追随困難な競争優位を構築する。

## 2. 課題 ★新規追加

### 2.1 技術課題

- **データプラットフォーム標準化:** MicrosoftとAzure連携で構築を進めているが、商用に耐えうるかは検証が必要。データ基盤活用に向けて対応中

### 2.2 事業化課題

- **収益モデルの具体化:** 「RoboSyncオプション（有償）」の価格設定・導入形態の定義が必要。α版/β版の実証結果から投資対効果を定量化する
- **Responsible AI戦略:** Microsoftとのワークショップを通じて構成を進めている段階。商用展開に向けた安全性・倫理面の方針を策定中

### 2.3 現場課題

- **彩都パーツセンターの調整:** 一力さんからは承諾済み。サイト倉庫側との導入スケジュール・窓口の確保が未調整
- **代替・並行案:** 調整が難しい場合は神戸工場での実施を検討。並行して進める可能性もあり

### 2.4 リソース課題

- **追加人員の確保:** プロジェクト期間中に追加人員が不足。類似技術を扱う別PJからのアサインで吸収する見込み
