## 1. 基本原理: GMRの汎化条件

池内先生の核心的指摘:
> 「物体ごとにスキルを分けるとスケールしない。物理特性で分類すべき」
> 「ペットボトル持たせようがコーヒーカップ持たせようが、本質は同じ」

### GMRが同一と見なせる条件

2つのタスクが**同じGMR**で実行可能な条件:

| 条件 | 内容 | 例 |
|------|------|-----|
| ① 接触状態遷移が同じ | ICF遷移パターンが一致 | ビンPC→把持→作業台PC |
| ② Grasp Typeが同じ | 把持の物理パターンが一致 | 両方ともLateral Pinch |
| ③ 力学的制約が同範囲 | 重量・剛性・摩擦が同オーダー | 50g〜500g、非脆性 |
| ④ Semantic Constraintが同種 | 意味的制約のカテゴリが一致 | 両方とも「袋を破かない」 |

> **条件①〜④を全て満たす → 同一GMR（同一VLA Policy）で対応可能**

## 2. 彩都品番のGMRマッピング

### 2.1 品番の物理特性による分類

彩都パーツセンター: 12万品番、月間3万点出荷

風見さん動画 + 中田さんヒアリングから推定した品番の物理特性分布:

| 物理特性クラス | 推定比率 | ビン内状態 | 必要Grasp Type | GMR |
|---|---|---|---|---|
| **袋入り小〜中型部品** | ~35% | S2相当 | Lateral Pinch (Pinch-Bag) | **GMR-P2** |
| **袋in袋（多層包装）** | ~25% | S1相当 | 複合（開封必要） | GMR-P4（複数GMR） |
| **箱物（中型段ボール）** | ~15% | S5相当 | Medium Wrap | **GMR-P1** |
| **バラ小物（ネジ等）** | ~10% | S6相当 | Precision（指必要） | 対応不可 |
| **段ボール内パーツ** | ~10% | S3相当 | Medium Wrap + 開封 | GMR-P3（双腕） |
| **ゴム・シート・柔軟物** | ~5% | S7等 | Vacuum / Lateral | GMR-P2' |

> ⚠️ 統計データがないため推定値。一力さん/中田さんから実データ入手後に更新。

### 2.2 各GMRのカバレッジ推定

#### GMR-P1: 箱物ピック（Medium Wrap）

```
対象: 直方体の段ボール・箱物。サイズ ≤85mm幅。重量 ≤5kg
推定カバー品番: ~15% × 12万 = ~18,000品番
制約: 2F-85のストローク（85mm）に収まるもの
```

| 同一GMR内のバリエーション | 対応可否 |
|---|---|
| 電源ユニット（箱） | ✅ 同じMedium Wrap |
| リモコン（箱） | ✅ 同じMedium Wrap |
| プロジェクターランプ箱 | ✅ 同じMedium Wrap |
| 大型パネル（箱） | ❌ ストローク超過 |

**汎化の鍵:** 箱のサイズ・重量が変わっても、Grasp Type（Medium Wrap）とICF遷移（ビンPC→把持→作業台PC）が同じなら同一Policy。VLAは画像から箱サイズを認識し、把持幅を自動調整する。

#### GMR-P2: 袋物ピック（Pinch-Bag / Lateral Pinch）

```
対象: 袋入りパーツ（そのまま出荷可能なもの）
推定カバー品番: ~35% × 12万 = ~42,000品番
制約: 袋の端が把持可能であること
```

| 同一GMR内のバリエーション | 対応可否 |
|---|---|
| 袋入りケーブル | ✅ 袋端をPinch |
| 袋入りコネクタ | ✅ 袋端をPinch |
| 袋入りフィルター | ✅ 袋端をPinch |
| 袋入りネジ（小袋） | ✅ 小袋端をPinch |
| プチプチ包装品 | △ 吸着に切替の可能性 |

**汎化の鍵:** 袋の中身が何であれ、**「袋の端を挟んで持ち上げる」という物理操作は同一**。VLAは袋の形状・向き・掴み位置を画像から推論する。これがまさにVLAの価値。

> **池内先生の理論で言えば:**
> GMR-P2内の全品番は「同じICF遷移 + 同じGrasp Type + 同じSemantic Constraint（袋を破かない）」
> → 潜在空間上で近い位置にマッピングされるはず
> → 1つのPolicyで全てカバー可能

## 3. GMR数 × カバレッジのトレードオフ

### 最小GMR構成でのカバレッジ

| GMR数 | 構成 | 推定カバレッジ | 備考 |
|-------|------|-------------|------|
| **1** | GMR-P2（袋物） | **~35%** | α版Stage 2の単一Policy |
| **2** | + GMR-P1（箱物） | **~50%** | α版完成形 |
| **3** | + GMR-P2'（吸着系） | **~55%** | 平面パーツ・プチプチ対応 |
| **5+** | + 開封系・戻し系 | **~80%** | β版で到達 |
| 全て | + バラ小物・道具 | **~95%** | GA以降 |

```
カバレッジ
100% ─────────────────────────────────────── ●
 95% ─────────────────────────────── ●       GA
 80% ──────────────────── ●                  β版
 55% ──────────── ●                          α版+吸着
 50% ────────── ●                            α版（2 GMR）
 35% ────── ●                                α版（1 GMR）
  0% ● 
     0    1    2    3    4    5   ...10   全て
                  GMR数 →

     ↑ 最初の2 GMRで50%カバー = 急激な立ち上がり
       残り50%は長いテールに分散
```

### 事業的インパクト

| 指標 | 2 GMR（α版） | 5 GMR（β版） |
|------|-------------|-------------|
| カバー品番 | ~60,000 | ~96,000 |
| 月間処理可能点数 | ~15,000 | ~24,000 |
| 学習データ量 | 400 ep (2×200) | 1,000 ep (5×200) |
| 学習時間 | 2日 | 5日 |
| VLAの汎化証明 | **1 Policy = 42,000品番** | — |

> **最も強力なメッセージ:** 「200エピソード（90分）の学習で、42,000品番に対応する1つのGMR Policyが作れる」

榊原・松山提案のGMRマーケットプレイスに対して、彩都のデータは以下の価値を持つ:

## 5. 池内先生へのディスカッション論点

### 「1 GMR = 多品番」の汎化条件

- GMR-P2（袋物Pinch-Bag）が42,000品番をカバーするという仮説の妥当性
- 潜在空間上で「袋入りケーブル」と「袋入りコネクタ」は近い位置にマッピングされるか？
- **PCA可視化**（宿題）でこの仮説を検証できるか？

### GMRの粒度設計の最適化

- 2 GMRで50%カバーは**十分に価値がある**か？
- それとも、より細かくGrasp Type別に分離すべきか？（Vacuum-Flat Policy、Pinch-Bag Policy...）
- 「粗いGMR（高カバレッジ・低精度）」vs「細かいGMR（低カバレッジ・高精度）」のトレードオフ

### カバレッジ曲線の理論的解釈

- 「2 GMRで50%、5 GMRで80%」という急激な立ち上がりは、物流タスクの一般的特性か？
- パレートの法則（20%のGMRで80%のタスクをカバー）がロボットスキルにも成立するか？
