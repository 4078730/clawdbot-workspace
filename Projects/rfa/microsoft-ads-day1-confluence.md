## Session 1: Project Overview（20分）

---

### S1-1. ビジョンと解決する課題（7分）

**RFA（Robotics Foundation Agent）** は、人間の言語指示を理解し、物流・製造現場の複雑な物理タスクをロボットが自律実行する知能システム。

#### 解決する4つの課題

| # | 課題 | 説明 |
|---|------|------|
| 1 | **抽象的言語指示の自律的解釈** | 「すきまなく詰めて」等の曖昧指示から暗黙的意図（重いものは底、壊れ物は上）を推論し、具体的作業手順に自動分解 |
| 2 | **専門知識不要な作業指示** | 従来の1つ1つの動作教示から、現場作業者の自然言語指示のみでロボット作業を実行。SIer依存を解消 |
| 3 | **未知・複雑形状物体の汎用操作** | 事前登録なしでも視覚認識から適応的把持・操作を実現。従来ルールベース制御の限界を突破 |
| 4 | **動的環境での自律的問題解決** | 把持失敗・物体落下等の予期せぬ状況で自律的リカバリーと計画修正による作業継続 |

#### 事業価値

SCM領域の労働力不足が深刻な物流・倉庫現場で、入出庫・梱包等の属人作業を自動化。ロボット制御PFをベースにこの技術をスケールしていく。

---

### S1-2. 技術アーキテクチャ概要（7分）

5つの主要技術要素による統合スタック。

#### 4レイヤー構造

| レイヤー | 役割 | 要素技術 |
|---------|------|---------|
| **Task Planning Hub**（LLM Agent — 頭脳） | ② Planning：抽象指示→具体的作業手順に自動分解 / ③ Skill Composition：サブタスクをスキル組み合わせで達成 | DRIP（EMNLP 2025）、MTP（ICRA 2026）、ViReSkill（ICRA 2026） |
| **Symbol Hub**（④ Symbol Integration） | 言語指示を画像上の具体的操作点（どこを掴む、どこに置く）に変換 | KeypointVLA（RSJ 2025）、IndexVLA（ICRA 2026）、Gemini-ER |
| **Sensorimotor Hub**（⑤ VLA-Robot — 実行） | 視覚認識・言語理解・物理動作を統合したロボット制御 | π0 Fine-Tuning、~200 episodes/skill、~1 day training |
| **RoboSync + Data Platform**（① 実証基盤） | ロボット制御PFベースの安全な実行・データ収集・継続学習サイクル | **VLA Skill Block（FY26最優先）**、MCP Agent、Deploy-Time Scaling |

#### 技術的差別化ポイント

- **言語指示の直接実行** — 複雑な事前プログラミング不要
- **未知環境への適応** — 学習済みパターン以外でも自律的に判断・実行
- **失敗からの自動学習** — エラー時の自律的分析・改善による継続的性能向上

### S2-1. アーキテクチャ詳細と統合構造（25分）

#### Task Planning Hub

- **タスク理解・分解：** 自然言語指示→サブタスク列に構造化
- **Skill Alignment：** VLAスキル or RoboSync既存ブロックを選択
- **Re-Planning：** 論理ミス→LLM再計画 / 制御ミス→VLAパラメータ修正

#### Symbol Hub

- **記号接地：** 「重い箱」「壊れ物」→座標・Keypoint・マスクに変換
- **VLM状態認識：** Gemini-ERで成功/失敗判定
- **IndexVLA：** Point/Trajectory/Vector表現の使い分け（パースの記号論ベース）

#### Sensorimotor Hub

- **VLA Policy：** π0ベース、200エピソード/90分で1スキル学習
- **KeypointVLA：** 高精度タスク向けKeypoint中間表現（成功率+40%）
- **HIL Interface：** 人間教示を受入れ→継続学習データに利用

#### RoboSync統合 — 2つのルート

| ルート | 説明 | 優先度 |
|-------|------|-------|
| **① Agent経由** | MCP経由でRoboSyncに接続。完全自律化ルート | 中長期 |
| **② VLAスキルブロック** | GUI配置・組み合わせ | **FY26最優先** |

MCPを共通プロトコルとして、RoboSync / WMS / MES / Gemini-ER と疎結合に接続。

### S2-2. 保有技術の詳細と成果（20分）

#### ① ロボット制御PF統合パイプライン（FUK拠点）

- ロボット制御PF経由のデータ収集→π0学習→実動作サイクルを実現
- BinPickingタスクで1000エピソードの動作を学習し再現
- 計3500エピソードの混合データセットで10種類の物体を扱える汎用モデル
- 未学習物体への汎化性を実証（スキンクリームで学習→コーンフレーク把持成功）
- **リトライ動作を教えていないのに自律的に獲得**

#### ② VLA-Robot環境（Idea Forge）

- 双腕Aloha Stationary AI + SO-101を活用した実験環境
- ACT / π0 / smolVLA等の複数VLAモデル比較評価
- CXC展示・iREXでの継続的デモ開発

#### ③ KeypointVLA → IndexVLA

- KeypointVLA：画像上に「掴む点」「置く点」を(x,y)座標で指定。既存VLAモデルの構造変更不要
- ACTモデルで**最大40%の成功率向上**（電池梱包タスク 30%→70%）
- IndexVLA：3種の指示モードを体系化
  - **Point型** — ピック＆プレース
  - **Trajectory型** — なぞり・拭き取り
  - **Vector型** — 押し込み・コネクタ挿入

#### ④ MTP / ViReSkill

- **MTP：** 過去の成功経験を参照し新環境でのタスクプランニングに活用。Sim→Real転移
- **ViReSkill：** 失敗動作の映像分析から自律的に計画修正。Lifelong Learning

#### ⑤ DRIP（Planning）

- Backward Reasoning — ゴールから逆算してサブゴールに分解
- テキスト→Minecraft 3D仮想環境に拡張、複雑な構造タスクの達成を実証
- EMNLP 2025投稿済

#### ⑥ iREX2025統合デモ

- Gemini-ER × π0による多段タスクVLAデモ
- シェーバー梱包タスク：各ステップの完了/失敗をVLMで判定
- 次ポリシーへの遷移 or リトライ/人間介入への分岐を実装
- 今後の現場PoCで活用可能な**標準パターン**のたたき台

### S2-3. データ戦略と継続学習サイクル（15分）

#### Deploy-Time Scaling — 使いながら賢くなる構造

| データソース | 特性 | 用途 |
|-------------|------|------|
| **Human Teleoperation** | 最高価値・高コスト | 初期スキル獲得、高精度Fine-Tuning |
| **Synthetic Data** | スケール可能・Sim2Realギャップ | バリエーション拡張、ロバスト化 |
| **Human Videos** | 大量に存在・ロボットコマンド含まず | 事前学習、意味表現の土台 |

#### データライフサイクル（RoboSync経由）

```
Model Deploy → Logging → Data Export（RLDS形式） → Train/Fine-tune → Re-Deploy
```

- 現場導入と同時にデータ蓄積が始まる
- データ収集の追加コストはほぼゼロ
- 現場の失敗データ・リカバリーデータは固有の学習資産

#### BigTechとの差別化

| | Training-Time Scaling（BigTech） | Deploy-Time Scaling（RFA） |
|---|---|---|
| 方式 | 大規模テレオペ施設での力技 | 現場導入と同時にデータ蓄積 |
| 初期投資 | 数百億円 | 低い |
| データ品質 | 理想環境 | 現場固有（失敗・リカバリー含む） |

#### 現在の課題（→ Microsoft協業ポイント）

- FT→再デプロイのサイクルがまだ手動 → **Azure ML Pipelines**
- モデルバージョン管理が未整備 → **MLOps基盤**
- 合成データ生成の体系化 → **Microsoft Simulation**
