## 1. 出発点：VLAを「どう分割するか」という問題

### 1.1 RFAプロジェクトが直面していた課題

RFAプロジェクトでは、VLA（Vision-Language-Action Model）を使って、物流・製造の現場タスクを自動化しようとしています。

VLAは「画像を見て、言語指示を理解して、ロボットを動かす」ことができる強力なモデルです。しかし、実際に現場で使おうとすると、根本的な問題にぶつかりました。

> **「VLAをどの粒度で学習・呼び出すべきか？」**

例えば、梱包タスクを考えたとき：
- 「梱包全体」を1つのVLAで学習する？ → 複雑すぎて学習が困難
- 「Pick」「Place」という動作ごとに学習する？ → 粒度が細かすぎて組み合わせが爆発

**【図①挿入箇所】**: 粒度の問題を示す図
- 左：粒度が大きすぎる（梱包全体）→ 学習困難
- 右：粒度が小さすぎる（Pick/Place）→ 組み合わせ爆発
- 中央：適切な粒度 → GMR

### 1.2 なぜ「動詞ベース」の分類では不十分か

最初は「Pick」「Place」「Insert」といった動詞でスキルを分類しようとしました。しかし、これには問題がありました。

**同じ「Pick」でも、実際には全く違う動作が必要**：
- 箱物を上から掴む
- 袋物を横から掴む
- ケーブル付きの製品を慎重に持ち上げる

動詞は人間にとってわかりやすいですが、**ロボットにとっては曖昧すぎる**のです。

LLMが「これをPickして」と指示しても、VLAには「どう掴めばいいか」がわからない。ここに、**LLMが扱う「意味的スキル」と、ロボットが実行する「物理的スキル」の間にギャップ**がありました。

## 2. 池内先生のアプローチ：接触状態で考える

### 2.1 池内先生の専門領域

この問題を解決するために、池内克史先生との連携を開始しました。

池内先生は、**Learning-from-Observation（観察からの学習）** を30年以上研究されてきた方です。特に、**接触状態遷移（Contact State Graph）** という考え方で、ロボット操作を物理的に厳密に記述する理論を確立されています。

この理論が、私たちの「VLAをどう分割するか」という問題に、明確な答えを与えてくれました。

### 2.2 核心的なアイデア：スキル = 接触状態の遷移

池内先生の考え方の核心は、こうです。

> **スキルを「動詞」ではなく「接触状態の遷移」で定義する**

ロボット操作とは、結局のところ「物体と環境の接触状態を変化させること」です。

| 接触状態 | 意味 | 例 |
|---------|------|-----|
| NC（非接触） | 触れていない | 空中移動中 |
| PC（面接触） | 面で接触 | テーブル上に置いた状態 |
| FT（完全固定） | 動かない | ビス締結後 |

この考え方だと、スキルは以下のように定義できます：

- **Pick** = NC（非接触）→ 把持状態
- **Place** = 把持状態 → PC（面接触）
- **Insert** = PC → FT（完全固定）

**これなら物理的に一貫性があり、曖昧さがない**。LLMとVLAの間の「翻訳」が明確になります。

## 3. GMR：VLAの学習単位

### 3.1 GMRとは何か

接触状態の考え方を踏まえて、池内先生が提唱しているのが **GMR（Grasp-Manipulation-Release）** という概念です。

**GMR = 把持（Grasp）→ 操作（Manipulation）→ 解放（Release）**

| フェーズ | 内容 | 例 |
|---------|------|-----|
| **Grasp（把持）** | 対象物を安定して掴む | コップの側面を掴む |
| **Manipulation（操作）** | 掴んだまま動かす | コップを移動、フタを回す |
| **Release（解放）** | 安全に手を離す | 棚の上に置いて離す |

日常・物流・製造の作業の大半は、この「**掴んで → 何かして → 離す**」というパターンに分解できます。

**【図②挿入箇所】**: GMRの3フェーズを示す図
- Grasp → Manipulation → Release の流れ
- 各フェーズでの接触状態の変化を図示

### 3.2 GMRが「ちょうどいい粒度」である理由

GMRは、VLAの学習単位として「ちょうどいい粒度」です。

```
高レベル:  「箱を棚に収納する」（言語指示）  ← LLMが扱う
           ↓
中レベル:  GMR（把持→操作→解放）           ← VLAの学習単位
           ↓
低レベル:  joint / torque / waypoint        ← ロボット制御
```

- **大きすぎない**: 1つのGMRは数秒〜数十秒で完了する操作
- **小さすぎない**: 意味のあるまとまり（1つの目的を達成する）
- **再利用可能**: 同じGMRを異なるタスクで使い回せる

### 3.3 梱包タスクでのGMR分割例

具体的に、梱包タスクをGMRで分割するとこうなります：

| GMR | 内容 | 特徴 |
|-----|------|------|
| 商品ピッキング | コンテナから商品を取り出す | 片手、不定形対応 |
| 箱詰め（片手） | 小物を箱に入れる | 片手 |
| 箱詰め（両手） | 大物を箱に入れる | 両手協調 |
| 箱閉め | フラップを折り畳む | 両手、シーケンス |
| テープ貼り | 封緘テープを貼る | 軌道追従 |

**重要なポイント**：
- 「シェーバーを入れる」と「チャージャーを入れる」は **同じGMR**（物理特性が同じ）
- 「片手で入れる」と「両手で入れる」は **別のGMR**（物理特性が異なる）

つまり、**物体ごとではなく、物理特性でスキルを分類**します。これにより、再利用性が大幅に向上します。

- 梱包フロー全体を示し、どこがどのGMRに対応するかを図示

## 5. 現場運用：継続学習サイクル

### 5.1 完璧を目指さない

池内先生との議論で、現場運用について重要な合意がありました。

> 「100%って絶対難しいから。90%やっといて、10%があったらなんか呼ぶっていうのが一番いい」

最初から完璧な自動化を目指すのではなく、**エラー時は人間を呼ぶ運用が現実的**です。

そして重要なのは、**人間が介入したデータを学習に活用する**ことです。これが「Human-in-the-Loop」の考え方です。

### 5.2 継続学習サイクルの設計

この考え方に基づいて、以下のような継続学習サイクルを設計しています。

**【図④挿入箇所】**: 継続学習サイクルの全体図

```
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    ▼                                                  │
 現場で実行                                             │
    │                                                  │
    ▼                                                  │
 VLMが状態監視 ───→ 成功 ───→ 成功データとして蓄積 ────┤
    │                                                  │
    │ 失敗検知                                          │
    ▼                                                  │
 自律リカバリー試行                                      │
    │                                                  │
    │ リカバリー不可                                    │
    ▼                                                  │
 人間が介入 ───→ 介入データとして蓄積 ─────────────────┤
                                                       │
                   ┌───────────────────────────────────┘
                   │
                   ▼
              データ蓄積
                   │
                   ▼
             学習 / Fine-tune
                   │
                   ▼
              再デプロイ ───→ 現場で実行（改善版）
```

**ポイント**：
- 失敗も価値あるデータになる
- 人間介入が多いスキルから優先的に改善
- 「使いながら賢くなる」仕組み

## 6. 事業的意義：なぜこれが重要か

### 6.1 スキル再利用による展開コスト削減

GMRベースでスキルを設計することの最大のメリットは、**新しいタスクへの展開コストが下がる**ことです。

**従来**（スキル再利用なし）：
- 新タスクごとに一からデータ収集・学習
- タスク数に比例してコストが増大

**GMRベース**（スキル再利用あり）：
- 既存GMRの組み合わせで多くのタスクをカバー
- 新GMRが必要な場合のみ追加学習
- **タスク数が増えてもコストの増加が緩やかになる**

**【図⑤挿入箇所】**: 展開コストの比較グラフ
- X軸: タスク数
- Y軸: 累積コスト
- 2本の曲線: 「従来（線形増加）」vs「GMRベース（漸近的に収束）」

### 6.2 理論的基盤があることの意味

池内先生のフレームワーク（接触状態遷移、GMR）を採用することで、**技術に理論的な裏付けが生まれます**。

| 効果 | 内容 |
|------|------|
| **顧客への説明** | 「なぜこのスキル分割なのか」を物理的根拠で説明できる |
| **技術者の共通言語** | チーム内・パートナーとの議論が効率化 |
| **デバッグの容易化** | 問題が「どのGMRの、どの接触遷移で起きたか」を特定しやすい |

### 6.3 研究と事業の両立

現在、IndexVLA（池内先生の理論をVLAに適用した手法）の論文をRA-L（トップジャーナル）に投稿予定です。

これは「単にデモを作る」ではなく、**学術的にも価値のある技術を事業に活用している**ということです。技術的な強みとしてのブランディング、採用への寄与も期待できます。

## 7. VLAが価値を出す領域

### 7.1 少量多品種・動的変化が主戦場

池内先生との議論を通じて、VLAが本当に価値を出せる領域が明確になりました。

| 領域 | VLAの適性 | 理由 |
|------|----------|------|
| 少量多品種 | ◎ | SKU変更のたびに再教示不要 |
| 動的状況変化 | ◎ | 毎回違う状態に対応可能 |
| 属人的判断 | ◎ | 暗黙知をデータから学習 |
| 一品種大量・固定タスク | △ | 従来技術でも対応可能 |

**重要な示唆**：一品種大量生産の固定タスクは従来技術でも自動化できます。VLAの真価は、**「毎回状況が変わるタスク」**で発揮されます。

### 7.2 梱包タスクが最適な理由

池内先生のフレームワークで梱包タスクを分析した結果、興味深いことがわかりました。

> **梱包は「物理的には単純」だが「意味的には複雑」**

多くの工程が**STG（Semantic Task Group）**に属します。これは、物理的な制約だけでなく、「壊れ物を守る」「水平を保つ」といった意味的な制約を含むタスクです。

こうしたタスクは、ルールベースで書き下すのが難しく、**VLAがデータから学習することで価値を出しやすい**領域です。

## 9. まとめ

池内先生との連携を通じて、RFAプロジェクトの技術的方針が明確になりました。

### 得られた3つの知見

1. **VLAの学習単位はGMR（Grasp-Manipulation-Release）**
   - 「動詞」ではなく「接触状態の遷移」でスキルを定義
   - 中間レベルの抽象表現として、LLM/VLAとの相性が良い

2. **継続学習サイクルで「使いながら賢くなる」**
   - 完璧を目指さず、エラー時は人間を呼ぶ
   - 人間介入データを学習に活用

3. **スキル再利用で展開コストを削減**
   - 物理特性でスキルを分類 → 新タスクへの横展開が容易
   - 理論的基盤により説明可能性・信頼性が向上

### 事業的意義

- **コスト削減**: 新タスク追加のコスト・時間が大幅に下がる
- **差別化**: 「使いながら賢くなる」仕組みが競争優位に
- **技術的強み**: 学術的価値を持つ技術を事業に活用
