## 1. 国土交通省 航空局とは

### 位置づけ
- 国土交通省の**内部部局**（Civil Aviation Bureau）
- 日本の航空行政全般を所管

> 📎 [航空局 組織と主な業務 — 国土交通省](https://www.mlit.go.jp/about/soshiki_koku.html)
> 📎 [航空局の役割 — 国土交通省](https://www.mlit.go.jp/koku/15_bf_000715.html)
> 📎 [航空局 — Wikipedia](https://ja.wikipedia.org/wiki/%E8%88%AA%E7%A9%BA%E5%B1%80)

### 航空局の4部門

| 部門 | 主な役割 |
|------|---------|
| **総務課** | 国際関係事務の総括、ICAO連絡、予算管理 |
| **航空ネットワーク部** | ★ 空港の設置・管理、航空運送の発達・改善、国際航空政策 |
| **安全部** | 航空機の安全確保、航空従事者の資格、空港の安全確保、無人航空機 |
| **交通管制部** | 航空交通管制、航空保安施設の設置・管理 |

### WSに関連が深いのは「航空ネットワーク部」
- **航空ネットワーク企画課**: 空港を活用した地域振興、騒音対策
- **国際航空課**: 国際航空運送の発達・改善、航空に関する国際協定
- **首都圏空港課**: 羽田・成田の設置管理
- **近畿圏・中部圏空港課**: 関空・中部の設置管理
- **空港計画課**: 空港の整備計画、改善のための調査・研究

> 📎 [航空局 組織図（PDF）— 国土交通省](https://www.mlit.go.jp/koku/15_bf_000291.html)

### 航空局のミッション（公式）
1. **国際交流の拡大・観光立国の推進** — インバウンド倍増
2. **地域間交流の拡大・地域活性化**
3. **都市の再生** — 都市の国際競争力強化の戦略的基盤
4. **経済社会の活性化・国際競争力の向上**

> 💡 **WSでのポイント**: 公募元は航空局。「観光立国」「インバウンド拡大」「国際競争力」が彼らの最上位目標。提案はこれらに紐づける必要がある。

> 📎 [空港一覧 — 国土交通省](https://www.mlit.go.jp/koku/15_bf_000310.html)

### 空港の分類（航空法）

| 種別 | 例 | 管理者 |
|------|-----|--------|
| **会社管理空港**（4空港） | 成田、関西、大阪（伊丹）、中部 | 空港会社 |
| **国管理空港**（19空港） | 羽田、福岡、那覇、新千歳 他 | 国（コンセッションの場合は民間が運営） |
| **地方管理空港** | 各地方空港 | 地方自治体 |

### ステークホルダーマップ（空港の入出国に関わるプレイヤー）

```
【政府機関】
├── 国交省 航空局 ← 空港の設置・管理、今回の公募元
├── 法務省 出入国在留管理庁（入管） ← 入出国審査、顔認証ゲート
├── 財務省 税関 ← 持ち込み品検査
├── 厚労省 検疫所 ← 感染症対策
└── (上記3つ = CIQ)

【空港運営】
├── 空港会社（NAA, TIAT, 関西エアポート等）← ターミナル管理、旅客動線
├── 航空会社（ANA, JAL, LCC各社）← チェックイン、搭乗
└── グランドハンドリング会社 ← 手荷物、地上支援

【サービス】
├── 免税店・商業施設
├── ラウンジ運営
├── レンタカー・交通
└── 両替・保険
```

> 💡 **WSでのポイント**: 空港は多数のプレイヤーが絡む複雑なエコシステム。UX改善は「誰の管轄か」が常に問題になる。CIQ（税関・入管・検疫）は3省庁にまたがる。

## 3. CIQ（入出国プロセス）の仕組み

### CIQとは
**C**ustoms（税関）+ **I**mmigration（出入国管理）+ **Q**uarantine（検疫）の総称。国際線旅客が必ず通過する3つの手続き。

> 📎 [CIQとは — 訪日ラボ](https://honichi.com/news/2017/01/23/imbaundoniokeruciqtai/)
> 📎 [CIQ PRルーム — 関西国際空港](https://www.kansai-airport.or.jp/service/other/02.html)

### 出国の流れ（日本人が海外へ出る場合）

```
空港到着 → チェックイン/手荷物預け → 保安検査 → 税関申告（必要時）
→ 出国審査（パスポート/顔認証ゲート）→ 免税エリア → 搭乗
```

### 入国の流れ（海外から日本に戻る/来る場合）

```
到着 → 検疫（Q: 体温チェック等）→ 入国審査（I: パスポート/在留資格確認）
→ 手荷物受取 → 税関検査（C: 申告書/持ち込み品検査）→ 到着ロビー
```

### Visit Japan Web（デジタル化の現状）
- 入国手続きのオンライン事前登録サービス
- 検疫・入国審査・税関申告を事前にスマホで入力
- QRコードで手続き簡略化
- ただし完全自動化にはまだ遠い（有人窓口も残存）

> 📎 [Visit Japan Web](https://www.vjw.digital.go.jp/)

## 4. 国際線旅客のペインポイント（よくある不満）

### 出国時

| フェーズ | ペインポイント | 深刻度 |
|---------|-------------|--------|
| **チェックイン** | 長蛇の列。自動チェックイン機の操作が分からない | ★★★ |
| **手荷物預け** | 重量オーバーの対応、液体物の制限が分かりにくい | ★★ |
| **保安検査** | 混雑時30分以上待ち。PC/液体の取り出しが煩雑 | ★★★ |
| **出国審査** | 顔認証ゲートの存在を知らない人が有人ゲートに並ぶ | ★★ |
| **免税エリア** | 搭乗ゲートまで遠い。案内が分かりにくい | ★★ |
| **搭乗** | ゲート変更の通知が遅い。優先搭乗の混乱 | ★★ |

### 入国時

| フェーズ | ペインポイント | 深刻度 |
|---------|-------------|--------|
| **検疫** | 感染症流行時の長い列 | ★★（時期依存） |
| **入国審査** | 外国人の長蛇の列（繁忙期60分超）。日本人は顔認証で改善済み | ★★★ |
| **手荷物受取** | 待ち時間が長い。どのレーンか分からない。破損・紛失時の対応 | ★★★ |
| **税関** | 申告書の書き方が分からない。Visit Japan Webの認知不足 | ★★ |
| **到着後の移動** | 鉄道/バスの案内が不十分。切符の買い方が分からない（外国人） | ★★★ |

### 共通の課題

| 課題 | 詳細 |
|------|------|
| **言語バリア** | 英語以外の対応が弱い（中国語、韓国語、タイ語等） |
| **デジタルデバイド** | 高齢者・非スマホユーザーが置き去り |
| **情報の分断** | 航空会社アプリ、空港アプリ、Visit Japan Webが別々 |
| **リアルタイム情報不足** | 混雑状況、待ち時間、ゲート変更がリアルタイムで分からない |
| **乗り継ぎの不安** | MCT（最低乗り継ぎ時間）内に移動できるか不安 |
| **時間の無駄** | 「2-3時間前に来てください」→ 手続き15分で終わり、残り暇 |

> 📎 [国際線を利用する時は何時間前に到着すべき？ — JALカード](https://jalcard.jal.co.jp/column/overseas/checkin2009/)

## 5. 業界トレンド: IATA One ID & Face Express

### IATA One ID
- **IATA（国際航空運送協会）**が提唱する次世代旅客体験のグローバル標準
- **コンセプト**: 生体認証（顔）1つで空港の全タッチポイントを通過
- **デジタルIDウォレット**: パスポート等をスマホに格納し、生体認証で共有に同意
- チェックイン → 手荷物預け → 保安検査 → 入国審査 → 搭乗を**すべて顔パス**に

> 📎 [IATA One ID（公式）](https://www.iata.org/en/programs/passenger/one-id/)
> 📎 [Fact Sheet — One ID — IATA](https://www.iata.org/en/iata-repository/pressroom/fact-sheets/fact-sheet-one-id/)
> 📎 [Digital Identity Program — IATA](https://www.iata.org/en/programs/innovation/digital-identity/)
> 📎 [Fully Digital Travel Experience Closer to Reality — IATA（2024年10月）](https://www.iata.org/en/pressroom/2024-releases/2024-10-30-03/)

### Face Express（日本版One ID）
- 成田空港が先行導入（NEC製システム）
- 顔認証で搭乗手続きをシームレス化
- 国交省航空局と連携して保安対策基準を見直し

> 📎 [顔認証を活用した新しい搭乗手続き「Face Express」— NEC技報](https://jpn.nec.com/techrep/journal/g21/n01/210109.html)
> 📎 [日本におけるデジタルIDウォレットの活用ケース：空港での顔認識とOne IDの継続利用 — IISE/NEC](https://note.com/nec_iise/n/ne08d90841011)

### 世界の先進事例
- **シンガポール・チャンギ空港**: 自動入国ゲート＋顔認証搭乗
- **ドバイ空港**: スマートトンネル（歩くだけで入国審査完了）
- **アムステルダム・スキポール空港**: 自動手荷物預け＋生体認証

## 7. WSで使える「パナソニックの強みを活かした提案の種」

### パターンA: 顔認証で完全シームレス旅（One ID拡張）
- 現状: 出国審査だけ顔認証 → **全タッチポイント顔認証に拡張**
- チェックイン、手荷物預け、保安検査、免税購入、搭乗を全部顔パス
- Visit Japan Web + パナソニック顔認証の統合

### パターンB: AI統合空港コンシェルジュ
- 旅客個人に最適化されたリアルタイムナビゲーション
- 「あなたの搭乗まであと47分。保安検査の現在の待ち時間は8分。免税店で25分ショッピングできます」
- サイネージ + スマホアプリ + AI対話の三位一体

### パターンC: インバウンド旅客の入国〜移動をシームレス化
- 到着前にアプリで手続き完了 → 到着後は顔認証で通過
- 手荷物追跡のリアルタイム表示
- 交通案内（鉄道・バス・タクシー）までワンストップ

### パターンD: 空港混雑の予測・最適化
- AI映像解析で混雑をリアルタイム検知
- 旅客に最適な動線を提案（サイネージ + プッシュ通知）
- 空港運営側にもダッシュボードで可視化

## 8. 水平思考: 他の人が出さない視点

### 「空港」を他の体験に置き換える

| 空港の体験 | 実は似てるもの | そこから盗めるUX |
|-----------|-------------|----------------|
| チェックイン→搭乗 | **テーマパークの入園→アトラクション** | ディズニーのスタンバイパス、待ち時間表示、動線設計 |
| CIQ審査の行列 | **病院の受付→診察→会計** | 番号呼び出し、進捗の可視化、待ち時間の有効活用 |
| ターミナルでの迷い | **巨大ショッピングモール** | Google Maps Indoor、AR案内 |
| 出発前の不安 | **初めてのライブ会場** | 「持ち物チェックリスト」「入場の流れ動画」の事前配信 |

### 他業界から盗めるアイデア

| 業界 | やってること | 空港に転用すると |
|------|-----------|----------------|
| **Uber** | リアルタイムで車の位置が見える | 手荷物の位置がリアルタイムで見える |
| **スタバ モバイルオーダー** | 着く前に注文、着いたら受け取るだけ | 免税品を事前注文、ゲート前で受け取り |
| **Netflix** | パーソナライズされたレコメンド | 「あなたの乗り継ぎ時間なら、3Fの寿司屋がちょうどいい」 |
| **任天堂 USJ** | 待ち時間自体をエンタメにする | 保安検査の待ち列でAR観光案内を表示 |
| **コンビニ無人店舗** | 歩いて出るだけで決済完了 | 免税店をウォークスルー決済に |

### 常識を疑う問い

1. **「空港に2-3時間前に来る」は本当に必要？** → 全プロセス顔パスなら30分前でよくない？ →「空港滞在時間を最小化する」という逆方向の提案
2. **「空港でやる」必要があるのか？** → チェックインは自宅完了。手荷物は自宅集荷。税関はVisit Japan Web。→ **「空港に着いたら歩くだけ」** = 究極のゴール
3. **旅客だけがユーザーか？** → 送迎する家族、迎えに来る人、空港で働く人。「お迎え側」のUXは完全に無視されてる

### 「2030年の空港」から逆算する

| 2026（今） | 2030（こうなってるべき） |
|-----------|----------------------|
| パスポートを持って空港に行く | **パスポートという物理物が不要**（デジタルIDウォレット） |
| 空港で手荷物を預ける | **自宅のドアで手荷物を渡したら到着地のホテルに届いてる** |
| 2-3時間前に到着 | **30分前に着いて歩くだけ** |
| 案内板を探す | **ARグラスやスマホが動線を導く** |
| 免税店を歩き回る | **欲しいものが搭乗ゲートに届いてる** |

### 極端なペルソナで本質を見る

WSでペルソナを作るとき、極端なユーザーを考えると本質が見える：

| 極端なペルソナ | なぜ面白い？ |
|-------------|-----------|
| **5歳の子ども一人旅** | 案内の分かりやすさ、安全性の極限テスト |
| **85歳、スマホなし、車椅子** | デジタルデバイド＋物理的バリアの同時解決 |
| **日本語も英語もできないタイ人家族5人** | 多言語対応の限界、非言語UXの重要性 |
| **年100回飛ぶ出張族** | 「余計なこと言わないでくれ」型。効率最優先 |
| **初めて飛行機に乗る人** | 何もかも分からない。でも空港はこの人を想定していない |

→ **極端なユーザーに最適化すると、普通のユーザーにも良くなる**（ユニバーサルデザインの原則）

### 「待ち時間」をリフレームする

全員が「待ち時間を減らそう」と言うはず。でも：
- **待ち時間 = 空白 → 待ち時間 = 価値ある体験** に変換できないか？
- ディズニーは「行列 = アトラクションの前座演出」にしている
- 空港なら：**「待ってる間に到着地の予習ができる」**（AR、サイネージ、AI案内）
- あるいは：**「空港自体が目的地」**になる（チャンギ空港のJewelはまさにこれ）

### 空港を「国のインターフェース」として見る

空港は**その国の第一印象**。旅行の最初と最後の体験。
- 入国＝**「日本へようこそ」の体験設計**
- 出国＝**「また来たい」と思わせるラストタッチ**
- これは単なる効率化じゃなくて**ブランディング**の問題
- パナソニックは「日本の玄関のUXを作っている」と言える

### 対立するニーズ（Challenge Statementの種）

| ニーズA | vs | ニーズB |
|--------|-------|--------|
| セキュリティ強化 | ⇔ | 通過スピード |
| パーソナライズ | ⇔ | プライバシー |
| 自動化・無人化 | ⇔ | 困った時に人がいてほしい |
| 効率的な動線 | ⇔ | 免税店に寄ってほしい（空港の収益源） |
| デジタル化 | ⇔ | デジタルが使えない人 |

→ **トレードオフを「どっちも実現する」のがイノベーション**。Challenge Statementをここに置くと強い。

### WS中に使えるキラーフレーズ

> 「空港のUXは**手続きの効率化**じゃなくて、**不安の解消**だと思う」

> 「空港に着いた瞬間から、**自分の旅のステータスが全部見えている**状態が理想」

> 「パナソニックは既に日本の空の玄関に203台の顔を覚えるカメラを持っている。**次はそのカメラに知性を加える番**」

## 9. konukiの国際線体験メモ（WS用）

- **最後の利用**: 2025年8月、羽田空港
- **羽田/成田/ターミナルの混乱**: どっちの空港か、第何ターミナルか不安になる → 出発前の情報設計の問題
- **行列**: 荷物検査・審査で並ぶ → 混雑可視化・顔認証拡大の提案根拠
- **移動距離**: 搭乗口まで遠い → 動線最適化・ナビゲーション
- **座席不足**: 座るところが少ない → リアルタイム空席案内
- **店舗の偏り**: 免税店ばかりでコンビニが少ない → ユーザーニーズとの乖離
