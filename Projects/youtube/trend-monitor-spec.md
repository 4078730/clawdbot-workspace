# SNSトレンドモニタリングツール — 企画ドキュメント

> **作成日:** 2026-02-13
> **ステータス:** ドラフト
> **関連:** AnimeLiveAction チャンネル運営

---

## 1. 背景と課題

### チャンネル概要
- **チャンネル名:** AnimeLiveAction
- **コンセプト:** アニメの名シーンをAI実写再現（Dreamina / Seedance 2.0）
- **フォーマット:** 縦型 9:16、一人のキャラクターにフォーカス
- **制作ツール:** Dreamina（Seedance 2.0）、中国語スタイルプリセット + 日本語脚本プロンプト

### クリエイティブ方針
- ✅ 名シーン実写再現（一人のキャラに焦点）
- ✅ 縦型動画（9:16）→ 一人構図が映える
- ✅ 同一作品内のIFバトル（品のある範囲で）
- ✅ 実写化のクオリティとリスペクト
- ❌ 異作品クロスオーバー
- ❌ アニメ画像をそのまま動かすだけ

### 課題
SNSのトレンドは日々変化しており、以下の2軸を常に追う必要がある：

1. **AIコンテンツのトレンド** — どんなAI生成動画がバズっているか？新しいツール・手法・スタイルは？
2. **アニメのトレンド** — どの作品・キャラ・シーンが今話題か？新作放送・映画公開・記念日等

現状、これらを手動で追っており：
- 情報収集に時間がかかる
- タイミングを逃す（トレンドは数日〜1週間で変わる）
- 「今何を作るべきか」の判断が直感頼み

---

## 2. 目的

**リアルタイムのトレンド情報を自動で収集・整理し、「今作るべきコンテンツ」の判断を高速化する。**

### ゴール
- トレンドの**検知**が遅れない（1日以内にキャッチ）
- **判断に必要な情報**が整理された状態で届く（生データではなく）
- konukiの**制作判断**を支援する（「今週はこのシーンを作るべき」）

---

## 3. モニタリング対象

### 3.1 AIコンテンツトレンド

| プラットフォーム | 対象 | 取得したい情報 |
|----------------|------|--------------|
| **X (Twitter)** | AI動画クリエイター、AI art関連 | バズった動画、使用ツール、プロンプト手法、新モデル発表 |
| **YouTube / Shorts** | AI実写化系チャンネル | 再生数急上昇、新しいスタイル、競合の動向 |
| **TikTok** | AI生成動画タグ | トレンドハッシュタグ、バズ動画、音源トレンド |
| **Reddit** | r/StableDiffusion, r/aivideo, r/midjourney | 新ツール・ワークフロー、コミュニティの反応 |
| **中国系** | Douyin、小紅書（RED）、Bilibili | Dreamina/Seedance活用例、中国発トレンド |

### 3.2 アニメトレンド

| プラットフォーム | 対象 | 取得したい情報 |
|----------------|------|--------------|
| **X (Twitter)** | アニメ公式、ファンアート、実況 | 話題のシーン、キャラ人気、放送直後の反応 |
| **YouTube** | アニメ解説・考察チャンネル | 話題の作品・シーン分析 |
| **TikTok** | アニメ系タグ | バズっているキャラ・シーン・ミーム |
| **Google Trends** | アニメ作品名・キャラ名 | 検索ボリュームの急上昇 |
| **MyAnimeList / AniList** | 放送スケジュール | 新作・続編の放送日、話題の回 |

### 3.3 クロス分析（最も価値が高い）

> **「今バズっているアニメ」×「AI実写化で映えるシーン」**の交差点を見つける

例：
- 呪術廻戦の最終回が放送 → 五条悟の名シーンが急上昇 → **今週作るべき動画が決まる**
- 新しいDreamina機能がリリース → そのスタイルで作りやすいアニメシーンを提案

## 5. 技術的アプローチ（案）

### Phase 1: 手動 + LLM要約（すぐ始められる）
- 定期的にWebスクレイピング / API取得
- LLM（Claude / GPT）で要約・分析
- OpenClawのcronジョブで毎日自動実行
- Discordに結果を送信

### Phase 2: 自動化パイプライン
- X API / YouTube Data API / Google Trends API 等を活用
- データ収集 → フィルタリング → LLM分析 → レポート生成
- ダッシュボード化（Notion / Streamlit等）

### Phase 3: インテリジェント提案
- 過去のバズ実績データと照合
- 「このトレンド × konukiのスタイル」でスコアリング
- 制作優先度の自動ランキング
