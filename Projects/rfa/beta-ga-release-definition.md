## 2. 機能ブロック定義マトリクス

| 機能ブロック | β版（9月末）の要件 | GA（3月末）の追加要件 |
|------------|------------------|-------------------|
| **VLA推論エンジン** | Azure remote inferenceで動作。制御周期10Hz。エッジ↔クラウド間通信確立。 | CEC基盤（本番Azure環境）に移行済み。本番SLA・可用性対応。 |
| **スキルブロック** | ビンピッキングタスク（仮）が動作。複雑形状パーツをビンから取り出し、台に置く動作を研究室（KBS/YHM）で実証。 | 複数スキル追加・スキル切替対応。Agentive制御（状況に応じた再計画）。 |
| **DataOpsパイプライン** | ログ収集・LeRobot形式変換・データ蓄積が動作。MS協業成果物を本番環境に移植済み。 | 自動再学習パイプライン。データ品質管理・モニタリング。 |
| **安全制御** | RoboSync標準準拠。緊急停止・衝突検知が動作。研究室環境での安全動作確認済み。 | QMSテスト完了（単体・機能・E2E・セキュリティ）。現場展開基準クリア。 |
| **エンドエフェクタ制御** | Robotiq 2F-85 + SQUARE Hand SQHW-01 基本動作。RoboSync制御下での統合動作確認。 | 多様な物体形状・包装状態への対応拡張。 |
| **RoboSyncインテグレーション** | VLAスキルブロックとしてRoboSync上で呼び出し可能な状態。APIインターフェース定義済み。 | 本番グレードのAPI。バージョン管理・後方互換性保証。 |

## 3. β版の定義

### 3.1 ゴール

> RoboSync上でVLAスキルブロックが動作し、**ビンピッキングタスク（仮）** — 複雑形状のパーツをビンから取り出し、台に置くタスク — を研究室（KBS・YHM）で実証できる状態。プレス発表・ライブデモが可能。

### 3.2 実証環境

| 項目 | 内容 |
|------|------|
| **拠点** | 京橋研究室（KBS）・横浜研究室（YHM） |
| **HW構成** | UR5e + Robotiq 2F-85（主EE）+ SQUARE Hand SQHW-01（副EE）|
| **カメラ** | RealSense D405 × 4台 |
| **推論環境** | Azure remote inference（クラウドGPU）|
| **制御周期** | 10Hz |

### 3.3 β版のスコープ外（GA以降）

- 現場実証（彩都PC・神戸工場）— β版リリース後にサブプロジェクトとして展開
- QMSテスト全項目クリア
- CEC基盤（本番環境）への移行
- 複雑スキル拡張・Agentive VLA機能
- 複数スキル切替

## 4. 公式リリース（GA）の定義

### 4.1 ゴール

> β版の全機能に加え、QMS準拠・CEC基盤移行・複数スキル対応が完了し、顧客環境への本番展開が可能な製品品質に到達した状態。

### 4.2 GAに必要な追加要件

| 区分 | 要件 |
|------|------|
| **テスト** | 単体テスト・機能テスト・E2Eテスト・セキュリティテスト 全項目完了 |
| **インフラ** | CEC基盤（本番Azure環境）への移行・動作確認 |
| **スキル** | 複数スキル対応・スキル切替機能 |
| **Agent** | Agentive VLA（状況に応じた再計画・異常対応） |
| **プロセス** | 社内承認フロー完了（QMS全項目） |

### 4.3 現場実証との関係

```
β版リリース（9月末）
  └─ 現場実証 サブプロジェクト開始（β版以降・並行）
       ├─ 彩都PC 現場実証（仮）
       └─ 神戸工場 現場実証（仮）
  └─ GA向け開発継続（50%リソース）
  └─ 複雑スキル拡張 + Agentive VLA（50%リソース）
         ↓
GA 公式リリース（2027年3月末）
```

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| **VLAスキルブロック** | RoboSync上でモジュールとして呼び出し可能なVLA制御ポリシー単位。 |
| **ビンピッキングタスク（仮）** | 複雑形状のパーツをビンから取り出し、作業台に置くタスク。カリアン（名称暫定）。 |
| **β版** | 最低限の機能が一通り揃い、研究室環境で実証可能な状態。RoboSyncのリリース定義に準拠。 |
| **GA（公式リリース）** | QMS全項目クリア・CEC基盤対応済みの本番展開可能な製品状態。 |
| **KBS** | 京橋研究室 |
| **YHM** | 横浜研究室 |
| **CEC基盤** | CECが構築するAzure上の本番クラウド基盤。POC環境（MS協業）とは別物。 |
