# 経産省「AIロボティクス戦略検討会議」第1回 ブリーフィング資料

> 出典: 経済産業省 第1回 AIロボティクス戦略検討会議（2026年1月21日）

---

## 📋 概要（30秒で掴む）

経産省が **AIロボティクス戦略** を2026年3月までに策定するため、有識者による検討会議を設置。
**多用途ロボット（ヒューマノイド含む）の開発促進と社会実装を一体的に推進** する国家戦略を作る動き。

**背景:** 日本は産業用ロボットで世界シェア約7割だが、サービスロボットでは米欧中に後れ。AI融合の波に乗り遅れている危機感。

## 🏗️ 検討体制

| レイヤー | 名称 | 役割 |
|---------|------|------|

| 有識者 | AIロボティクス戦略検討会議（経産省設置） | 戦略素案・実装ロードマップ案を策定 |

**委員:** アカデミア・ロボットメーカー・SIer・需要側の各業界代表 19名

## 🔬 技術的ブレイクスルー

### ソフトウェア側
- **ロボット基盤モデル（VLA/VLM）** の進化 → 教示時間が数週間→数分に短縮
- 新たな部品・生産ラインに柔軟かつ高速に対応可能に

### ハードウェア側
- EV/自動運転の部品量産 → ロボット部品のコスト大幅低下
  - センサー（LiDAR等）: 4年で**約50%低下**
  - バッテリー: 1990年代→2018年で**97%低下**

### AIロボティクスが解決する構造問題
- 従来: 少量多品種市場（ロングテール）向けはコスト高・開発期間長で困難
- AIで: **多用途ロボット** により多様なタスクに1台で対応可能に

---

## 🎯 戦略の4本柱

### ① サプライチェーン構築（供給側）
- **SDR（Software Defined Robot）** への移行を見据えた産業構造設計
- 多用途ロボットOEMメーカー・SIerの育成
- キーコンポーネント（モーター、減速機、コンピューティング基盤等）の特定と強化
- ファブレス/EMS/水平分業も含めたオープンな産業構造

### ② AI基盤モデル開発（技術側）
- **オープンなデータ基盤** で基盤モデル開発を加速
- **GENIAC（Generative AI Accelerator Challenge）** にAIロボティクス新類型を追加
- **AIRoA（AI Robot Association）** が2024年12月に設立済み
  - 正会員18社（ABEJA, NEC, トヨタ, デンソー, KDDI等）
  - データ収集・基盤モデル開発・安全性評価を推進

### ③ 需要側の導入環境整備
- **先行官需の創出**: 災害対応、防衛・宇宙、インフラ保守
- **ロングテール市場**: 小売、製造、警備、建築、運輸、農林水産、介護、造船等
- 各分野の**実装ロードマップ**を策定（10年程度のスパン）
- 認証制度の検討（プライバシー、セーフティ、セキュリティ）

### ④ Center of Excellence（CoE）整備
- 物理空間（モックアップ、開発・検証設備）+ サイバー空間（データ基盤）
- 世界中からトップ人材・情報が集まる拠点
- ハッカソン・コンペ等の人材育成

## 💬 委員からの主要発言（議事要旨より）

### 技術・研究の視点

| 委員 | 所属・分野 | 主張 |
|------|-----------|------|
| **河原塚** | ロボット基盤モデル | VLAは注目されるが上半身操作・言語理解は課題大。脚ロボも産業利用にはまだ遠い。柔軟な身体・能動探索・自律学習・接触覚が重要 |
| **瀬川** | ロボティクス全般 | CESで日本の存在感薄れている。米中の先を見抜き後追いではなく先行すべき。1-2年でなく3-5-10年の継続戦略が必要 |
| **橋本(康)** | 産業用ロボット | 分野横断で共通要素を抽出・共通化し開発コスト分散。日本の現場ノウハウとAIの統合が鍵 |
| **原田（座長）** | フィジカルAI | VLAエンドツーエンドと計画ベースの使い分けが必要。SIノウハウのデータ化・規格化が普及の鍵 |

### 需要側・現場の視点

| 委員 | 分野 | 課題・ニーズ |
|------|------|------------|
| **富岡** | 小売・物流 | 飲料陳列は数千台導入済みだが全業務の5-6%。レジ接客自動化（50%業務量）にVLM/VLAで着手 |
| **佐久間** | 小売（コンビニ等） | 2030年に店舗OP 30%削減目標。ロボット価格が高く採算合わない。協業/差別化の戦略分け |
| **田坂** | 物流 | 標準化されてない荷姿・倉庫、古い倉庫でのロボット導入が大きな課題 |
| **杉田** | 宿泊業 | 客室清掃・リネン運搬のロボット化が急務。中小規模の宿が多く高額ロボット導入困難 |
| **浅井** | 農業 | 生産システム全体をロボット前提で再構築すべき。日本の農地規模では単純機械化は限界 |
| **建山** | 土木・建設 | 自律型建設ロボットは広がりつつあるが中小現場は人手依存。廉価で使いやすいロボットが鍵 |
| **勇﨑** | 造船 | 溶接自動化率4割だが全工数では5%。1事業所に数百台規模のロボット導入を想定 |
| **伊藤** | 消防 | 危険環境下の無人化・自動化。若手の経験不足も課題 |
| **森** | 医療・介護 | 手術支援は普及しつつあるが限定的。バックヤード業務・介護支援でAIロボット導入が不可欠 |

## 🔗 我々のプロジェクトとの関連性

### RFA（Robotics Foundation Agent）との接点

1. **VLA/VLM**: 検討会議でもVLA/VLMの産業適用が主要議題。我々のIndexVLAの方向性と完全に合致
2. **データフライホイール**: AIRoAのデータ収集→基盤モデル→社会実装→データ還元のサイクルは、我々のデータフライホイール構想と同じ思想
3. **ロングテール市場**: 少量多品種（製造・物流・小売等）への対応が国家戦略レベルで重要テーマに
4. **SIノウハウのデータ化**: 座長・原田先生の「SIノウハウの規格化が普及の鍵」発言は、我々のGMR/ICFフレームワークと直結
5. **GENIAC AIロボティクス枠**: 公的支援の受け皿として注目すべき

### キーワード対応表

| 検討会議の用語 | 我々のプロジェクトでの対応 |
|--------------|------------------------|
| 多用途ロボット | RFAが目指す汎用ロボットエージェント |
| VLA/VLM | IndexVLA |
| SDR (Software Defined Robot) | RFAのソフトウェア中心アーキテクチャ |
| データ基盤・フライホイール | データフライホイール構想 |
| SIノウハウのデータ化 | GMR/ICF/PTG/STGフレームワーク |
| 先行官需 | 彩都PoCの延長線上 |
| 認証・安全性 | AIロボット安全性評価 |

- **2026年3月まで**: AIロボティクス戦略 + 実装ロードマップ策定

## 要約（5行）

1. 経産省が2026年3月までにAIロボティクス戦略を策定する動き。多用途ロボット（ヒューマノイド含む）の国産化と社会実装が目標
2. 日本は産業用ロボットでは強いがサービスロボット・AI融合で米中に大きく後れ。投資規模も桁違い
3. 供給側（サプライチェーン・基盤モデル）と需要側（実装ロードマップ）の一体推進がコンセプト
4. VLA/VLM、データフライホイール、SIノウハウのデータ化など、我々のRFAプロジェクトと方向性が完全一致
5. GENIAC AIロボティクス枠やAIRoAなど、具体的な受け皿も整備されつつある
