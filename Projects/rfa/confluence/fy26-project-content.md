# FY26 Robotics Foundation Agent — プロジェクトディスカッション用

---

## 1. プロジェクト内容

### 概要

Robotics Foundation Agent（RFA）は、人間の言語指示を理解し、物流・製造現場の属人的タスクをロボットが自律的に実行できる知能システムを実現するプロジェクトである。

**生み出される技術**
- LLM Agent + VLA + Robo Sync 統合基盤
- 現場データを学習資産に変換するDataOpsパイプライン

**何ができるようになるか**
- 言語指示だけでロボットが非定型作業を自律実行
- 失敗時の自動リカバリー、タスク再計画
- 使うほど賢くなる継続学習

**事業貢献**
- 物流・製造現場の省人化・品質安定・24h運用
- Robo Sync導入現場からのデータ蓄積による競争優位構築
- FY27事業判断に必要なPoCエビデンス確立

> 📄 詳細：[FY26 RFA 事業計画書](https://panasonic-connect.atlassian.net/wiki/spaces/CT1/pages/2484830627)（§1-3）

---

### FY26の位置づけ

FY26は **Phase 2: Robo Sync現場PoC・事業検証** への転換年度。最重要目標は「RFAが現場で回る」ことを技術・運用の両面で示すこと。

| Phase | 年度 | 内容 |
|-------|------|------|
| 0 | FY24 | 基盤立ち上げ |
| 1 | FY25 | RFAスタック確立、CXCデモ統合 |
| **2** | **FY26** | **Robo Sync現場PoC、事業検証** |
| 3 | FY27 | 社会実装、運用スキーム確立 |

### FY26実行戦略：4つの柱

| 柱 | 目的 |
|----|------|
| **① Robo Sync × VLA統合** | VLAをRobo Sync上のスキルブロックへ昇格、実行基盤確立 |
| **② DataOps** | 現場データを学習資産として蓄積、継続学習基盤構築 |
| **③ Agent自律適応** | 失敗・例外前提の再計画・適応実行構造 |
| **④ 現場実証・事業接続** | 神戸/彩都PoC、事業仮説検証、FY27判断材料整備 |

> 📄 各柱の詳細：[事業計画書 §4.3](https://panasonic-connect.atlassian.net/wiki/spaces/CT1/pages/2484830627)
