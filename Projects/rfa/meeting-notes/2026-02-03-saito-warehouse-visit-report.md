## 1. エグゼクティブサマリー

彩都パーツセンターの現場見学およびフォローアップヒアリングを通じて、**VLA（Vision-Language-Action Model）の適用可能性と現場制約**を明らかにしました。

### 得られた知見

| 知見 | 内容 |
|------|------|
| **VLA適用の有力候補** | オートストア出庫のピッキング作業（ビンから商品を取り出す） |
| **GMRフレームワークとの適合** | ビンからの取り出し動作は典型的なGMR（把持→操作→解放） |
| **リカバリースキルの重要性** | 上積み保管による回避動作、ピック失敗時のリトライが必要 |
| **ロボット導入の鍵** | 入庫時のプロパティ登録により、把持可能な商品を専用ポートに流す制御が可能 |
| **荷姿変動の課題** | 事前商品化（事前個装）により、マスターデータと実際の荷姿が異なる |

### 事業的意義

- **現場実証のフィールド確保** → 実際の倉庫タスクでVLAを検証できる
- **データフライホイールの起点** → 作業データの収集・学習サイクルを構築
- **Industrial Embodied AI連携** → 一力さんチームとの協業によりスケール可能

## 2. 彩都パーツセンターの概要

### 2.1 施設の位置づけ

彩都パーツセンターは、**パナソニックのサービスパーツ出荷拠点**です。業務用プロジェクター、ディスプレイ、リモートカメラ、カメラレコーダー、ノートパソコン（Let's Note）など、多様な製品の補修部品を国内外に出荷しています。

| 項目 | 内容 |
|------|------|
| 所在地 | 大阪府茨木市彩都あかね1 MFLP茨木 6階 |
| 稼働開始 | 2018年10月〜 |
| 面積 | 約5,000㎡ |
| 在庫品番数 | **12万品番** |
| 月間出荷点数 | **3万点** |
| 従業員 | 45人（社員）+ 30人（請負） |
| 出荷先 | 国内全国 + **海外24カ国** |

### 2.2 倉庫構成（4種類の保管エリア）

| エリア | 特徴 | 対象商品 | 備考 |
|--------|------|----------|------|
| **高稼働棚** | オートストア横の棚 | MSBD高稼働品（Aランク） | 需要大商品を配置、作業者移動あり |
| **一般棚** | 中物保管エリア | 中サイズ商品 | エリアごとに担当が分かれる |
| **オートストア** | AGVによる自動搬送 | 小物＆高稼働でない商品 | 作業者移動なし、ビン取り合いで遅延リスク |
| **ネステナー** | フォークリフトで運搬 | 大物（LCDパネル、梱包材、プロジェクターランプ等） | 入口右手の赤い棚 |

**配置基準:**
- **大きさ**: 小物→オートストア、中物→棚、大物→ネステナー
- **稼働頻度**: A,B,Cランク管理。A=高稼働、D=6カ月以上滞留
- **特殊品**: 全カテゴリーのバッテリーは指定棚で集中管理
- **年間不移動品**: 外部倉庫へ都度移動

### 2.3 オートストア設備

彩都パーツセンターの中核となるのが、**オートストア**（AutoStore）システムです。AGV（自動搬送車）がビン（コンテナ）を自動で取り出し、作業者のもとへ運ぶ自動倉庫システムです。

| 項目 | 内容 |
|------|------|
| ビン数 | 2万ビン |
| AGV台数 | 14台 |
| ビン段数 | 13段 |
| 出庫レーン | 約6箇所 |
| 充電 | 定期的に充電ステーションで充電 |

**運用特性:**
- 対象ビンより上にあるものは取り出して仮置き
- SAP（基幹システム）から事前にデータを送り、下段のビンを取り出すために上段に上げておく
- **頻度が低いものが必然的に下段に移動**する仕組み
- 入庫時に商品を入れすぎるとエラー（現在は充填率に気を付けて運用）

**ビンの間仕切り:**
- ビンに溝があり、スリットを入れて棚分割が可能
- 分割ごとにビンIDが付与され、WMSと紐づく
- 細かすぎる分割はロボットアームのアクセス精度が課題

### 2.4 導入済みソリューション

| ソリューション | 機能 |
|---------------|------|
| **積載量可視化** | カート・コンテナ・棚の充填率自動計測（上部カメラで計測） |
| **V-SLAM** | 非GPS空間での自己位置推定。フォークリフト・牽引車の動線分析 |
| **ダッシュボード** | 進捗管理、生産性分析、動線分析、収支分析、在庫管理の統合可視化 |
| **AI画像処理** | 標準化作業プロセスに基づくリアルタイム作業分解・分析 |

## 3. Industrial Embodied AIの取り組み

見学に先立ち、一力さんから**Industrial Embodied AI**の取り組みについてご説明いただきました。

### 3.1 「1秒の差」の重要性

一力さんが強調されたのは、**物流現場における「1秒」の価値**です。

> 「1秒 × 1万回 = 3時間の差になる」

作業者間のばらつき（Aさん5秒、Bさん8秒、Cさん12秒）が、計画との大きな乖離を生みます。また、サイバー（システム上のデータ）とフィジカル（実際の現場）のギャップも課題です。

### 3.2 付加価値作業の割合

| 現場タイプ | 付加価値作業の割合 |
|-----------|-------------------|
| 物流業界平均 | 1% |
| 一般的な物流現場 | 10% |
| **彩都パーツセンター** | **30%**（世界で1社だけ達成） |

彩都パーツセンターは、業界でもトップクラスの効率を実現しています。しかし、それでも**70%は非付加価値作業**（移動、待機など）であり、さらなる改善余地があります。

### 3.3 生産性改善の歴史と将来目標

| 年 | 施策 | ピッキング時間 |
|----|------|---------------|
| 2017年 | 標準化開始 | 113秒 |
| 2018年 | オートストア導入 | 160秒 → 54秒 |
| 2026年（目標） | ASRS + ピッキングロボット + タスク最適化 | **30秒** |
| 将来 | **無人化** | - |

**VLAはこの無人化に向けた重要な技術要素**となります。

## 4. オペレーション詳細

### 4.1 入庫プロセス

#### 入荷受入（1次検品）

```
①入荷予定確認（週2回：水・金）
  ↓ 入荷数により作業者割り当て調整
②トラック荷卸し・検品
  ↓
③入出庫伝票番号より入荷伝票作成
  ↓
④2次検品へ
```

**入荷経路:**
- MEBD:AVS/オーディオ佐賀 → 彩都
- MEBD:佐賀/部品 → 大阪南港倉庫経由 → 彩都
- MEBD:門真/プロジェクター・フラットパネル → 大阪南港倉庫経由 → 彩都
- MSBD:神戸 → 彩都

#### 2次検品・棚入庫

**棚指定ルール:**
- 新規品番登録時に棚指定を実施
- 基本: 小物→オートストア、中物→棚、大物→ネステナー
- 特殊: MSBD高稼働品はオートストアの横の高稼働棚に保管

#### オートストア入庫

**特徴:**
- **1ビン複数商品保管可能**（ビン内配置も登録可能）
- **上積み保管あり**
- 同じ商品でも入荷タイミングにより別ビン格納
- 滞留6カ月以上の場合、ビン積載率向上のため1ビン複数商品格納を推進

### 4.2 事前商品化（事前個装）

時間に余裕があるうちに、個包装を済ませておく運用。作業者が暇な時に事前個装を実施する。

**対象:**
- システム側で管理されている対象パーツ
- PAV（プロAV）含む

**ロボット導入への影響:**
- 事前個装されると、プチプチ包装などで荷姿が変わる
- マスターデータは「標準荷姿」のサイズ → **実際と異なる**
- 中身が見えない状態でバーコード読み取りで照合
- **ロボット把持の難易度が上がる要因**

### 4.3 出庫プロセス

#### 伝票発行（伝発）

**タイミング:**
- 売上伝票はAM8時伝発、9時には売上伝票棚に仕分け済み
- 商品カテゴリごとに伝発時間が指定（複数タイミングで発行）

**伝票の仕分け:**
- **エリア別**: 倉庫エリアごとに担当が分かれており、売上伝票はエリアごとに仕分け
- **時間帯別**: 遠方地（北海道、沖縄、九州）はトラック切り離しが早いため先に処理
- **行き先別**: 京橋行き、関東行きなど
- **保管場所別**: オートストア / 一般棚 / ネステナー

#### 棚出庫

```
①伝票棚から担当エリア分の売上伝票をピック
  ↓
②タブレットにログイン（作業者識別）
  ↓
③伝票のバーコードをスキャン → PMSに作業開始を通知
  ↓
④棚番号を確認し、該当棚へ移動
  ↓
⑤商品をピック、商品バーコードをスキャン（照合）
  ↓ ※違う商品を読み取るとエラー
⑥個装作業（袋入れ、輪ゴムくくり等）+ 個装ラベル貼付
  ↓
⑦次の伝票へ（繰り返し）
```

#### オートストア出庫

```
①売上伝票棚よりオートストア分を一括ピック
  ↓
②ポートでタブレットにログイン
  ↓
③伝票のバーコードをスキャン
  ↓
④AGVがビンをポートに搬送
  ↓
⑤商品をピック、商品バーコードをスキャン ← VLA適用ポイント
  ↓ ※複数ビンからのピックあり
  ↓ ※上積み保管のため該当商品上に別商品存在の可能性
⑥個装作業 + 個装ラベル貼付
  ↓
⑦完了ボタン → 次のビンが流れてくる
  ↓
⑧全伝票処理完了 → 梱包エリアへ
```

#### オーダーピッキング方式

| 方式 | 特徴 | 彩都での採用 |
|------|------|-------------|
| **オーダーピッキング** | 顧客ごとにピッキング、動線が長い | ✓ 現在採用 |
| **トータルピッキング** | エリアごとに商品を集約、後で仕分け | △ WMS移行後に可能性 |

**PMSを使っている理由:**
- 彩都は「工場物流」の位置づけ（パソコン工場の出荷前倉庫が分離したもの）
- WMSではなくPMS（パーツマネジメントシステム）を使用
- Blue Yonder WMSへの移行プロジェクトが進行中

### 4.4 仕分け・個装・梱包

#### 仕分け
- 仕分け品数により作業者割当を都度実施
- 売上伝票内の商品内容及び出荷先を確認

#### 個装
- ピッキング時に流通加工（個装）を実施
- 商品と売上伝票を輪ゴムでくくってセット
- 特大品（75インチ以上のパネル）は専用作業場で個装
- 45-60インチは個装状態で入庫

#### 出荷梱包
- 別エリアの作業台で実施
- 箱サイズは作業者が選定（属人的）
- 梱包材（発泡スチロール、緩衝材）の使用もマニュアルベース
- 伝票と商品をバーコードスキャンして情報一致を確認
- 伝票を入れて封函、出荷

## 5. VLA適用の有力候補：オートストア出庫タスク

### 5.1 タスクの流れとVLA適用ポイント

```
【オートストア出庫タスク】

①注文発生
  ↓
②AGVが倉庫内のビンを自動取得
  ↓
③出庫レーン（6箇所）にビン搬送
  ↓
④[人間作業] 売上伝票を確認
  ↓
⑤[人間作業] ビンから商品をピック ← VLA適用ポイント
  ※上積み保管のため、別商品を避けて取る必要あり
  ※複数ビンからのピックあり
  ↓
⑥[人間作業] バーコードスキャン
  ↓
⑦[人間作業] 個装ラベル貼付
  ↓
⑧出荷エリアへ移動
```

④〜⑦の人間作業部分が、VLA適用の対象範囲です。特に**⑤ビンから商品をピック**する作業は、GMR（Grasp-Manipulation-Release）の典型例です。

### 5.2 GMRフレームワークとの適合

池内先生との議論で確立した**GMRフレームワーク**の観点から分析:

| GMRフェーズ | 出庫タスクでの動作 |
|------------|-------------------|
| **Grasp（把持）** | ビン内の商品を掴む |
| **Manipulation（操作）** | ビンから持ち上げて移動 |
| **Release（解放）** | 出荷コンテナに置く |

ビンから商品を取り出す動作は、GMRの3フェーズにきれいに対応しており、**VLAで学習すべきスキルの粒度として適切**です。

## 6. ロボット導入に向けた技術的課題

### 6.1 主要課題

| 課題 | 内容 | 影響度 |
|------|------|--------|
| **事前商品化による荷姿変動** | 個包装されるとマスターデータと実際の荷姿が異なる | 高 |
| **バーコード位置のランダム性** | 商品のバーコード位置が統一されていない | 高 |
| **上積み保管** | 該当商品の上に別商品が存在する可能性 | 中 |
| **仕切り内の複数SKU** | 1つの仕切りに複数SKU（最大3つ程度）が入る場合あり | 中 |
| **ピース/ボール/ケースの単位** | ネジ等は袋から1個取り出す場合あり | 中 |
| **複数ビンピック** | 1注文で複数ビンからピックするケース | 中 |
| **ポート重複** | 他ポートと商品が重複した場合の待機判断 | 低 |

### 6.2 プロパティ登録による制御

入庫時に「ロボット把持可能」フラグを立てることで、出庫時の制御が可能になります。

**仕組み:**
1. 入庫時に「ロボット把持可能」フラグを立てる
2. WMS/PMSに商品のプロパティが登録される
3. 出庫時、ロボット専用ビンが専用ポートに流れるよう制御

> 「入荷の方でプロパティ登録をしてあげることによって、ピッキング指示をかけた時に、その特定のロボットアーム用ポートにしか、その掴める商品が流れてこないという仕組みができる」（中田さん）

### 6.3 段階的導入アプローチ

**Phase 1: 簡単なものから**
- 個装なし、マスターデータ通りの荷姿の商品を対象
- ABCランク分けでロボット把持可能な商品を選別
- 専用ポートに流す制御を実装

**Phase 2: 複雑な荷姿への対応**
- 事前商品化されたものも把持できるよう拡張
- 置き方の最適化（充填率 vs ロボット把持しやすさのトレードオフ）

**レイアウト案:**
- ロボット専用ポートを設置
- ピッキング → 梱包まで一貫してロボットで処理
- コンベアで商品+伝票をセットで次工程へ

## 参考：技術用語

| 用語 | 説明 |
|------|------|
| **PMS** | Parts Management System。パーツマネジメントシステム。工場物流向け |
| **WMS** | Warehouse Management System。倉庫管理システム。Blue Yonderへ移行予定 |
| **オートストア** | AutoStore。AGVがビンを自動搬送する自動倉庫システム |
| **GMR** | Grasp-Manipulation-Release。把持→操作→解放のスキル単位 |
| **ネステナー** | 重量物保管用の棚。フォークリフトで運搬 |
| **GTP** | Goods to Person。商品が人のところに来るオペレーション |
| **プロパティ登録** | 入庫時に商品の属性（ロボット把持可能等）を登録すること |
| **SKU** | Stock Keeping Unit。在庫管理単位 |
| **個装** | 個別包装。袋入れ、輪ゴムくくり等 |
| **事前商品化** | 時間に余裕があるうちに個包装を済ませておく運用 |
| **伝発** | 伝票発行の略 |
