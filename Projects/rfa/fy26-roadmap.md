## Q1（4〜6月）：基盤構築期

**目標：Q2以降の開発・実証の土台を固める**

| 柱 | 達成状態 |
|----|----------|
| ①Robo Sync×VLA | VLAスキルの入出力IF・失敗判定・停止条件が文書化され、Robo Sync側と合意済み |
| ②DataOps | 学習用RLDS/Lerobot形式スキーマ定義済み、サンプルデータで変換パイプライン動作確認済み |
| ③Agent自律適応 | 代表的失敗パターン3種（把持失敗、配置ズレ、落下）の振る舞いが定義済み |
| ④現場実証 | 神戸/彩都の対象タスク特定済み、現場担当者と作業スコープ合意 |

**見極めポイント（Q1末）**
- [ ] VLAスキルブロック仕様v1がRobo Sync側と合意
- [ ] Data Contract定義完了
- [ ] PoC対象タスクが神戸/彩都いずれかで合意

---

## Q2（7〜9月）：統合・検証期 → **α版リリース**

**目標：ラボ環境での統合動作確認を完了し、現場投入可能な状態に到達**

| 柱 | 達成状態 |
|----|----------|
| ①Robo Sync×VLA | VLAスキルがRobo Sync GUI上で配置・実行可能。1日8h連続運用でシステム停止1回以下 |
| ②DataOps | Robo Syncログ→RLDS変換が自動化、1週間分ログがクラウド蓄積される状態 |
| ③Agent自律適応 | Agentが2〜3スキル切替でタスク完遂するデモ動作、失敗時リトライ自動実行 |
| ④現場実証 | PoC実施計画書完成、必要機材・人員・スケジュール確定 |

**α版（9月）の定義**
- 状態：社内実証に投入した状態
- 成功率：70%程度
- 安全性：Robo Sync標準の安全制約に準拠、人介在でのリカバリー体制あり
- 提供形態：社内実証（無償）

**Go/No-Go判定（9月）**
- Go条件：VLAスキルブロック安定動作（稼働率95%以上）、Agent切替動作、データパイプライン稼働
- No-Go時：現場PoC延期、安定化に注力

## Q3（10〜12月）：現場実証期

**目標：社内現場での実運用を通じ、技術課題と事業価値の両面でデータを取得**

| 柱 | 達成状態 |
|----|----------|
| ①Robo Sync×VLA | 現場環境（神戸or彩都）でVLAスキル動作、環境差異による調整点洗い出し済み |
| ②DataOps | 現場データでFT→再デプロイ1回以上実施、改善効果を定量計測 |
| ③Agent自律適応 | VLMによる成功/失敗判定が現場稼働、人介在時データ記録フロー確立 |
| ④現場実証 | 社内PoC実施中、成功率・タクト・人介在頻度データ取得開始 |

---

## Q4（1〜3月）：評価・計画策定期 → **β版リリース**

**目標：PoC結果を定量化し、FY27の事業化判断に必要なエビデンスを揃える**

| 柱 | 達成状態 |
|----|----------|
| ①Robo Sync×VLA | 現場PoCで2週間以上継続運用実績、運用課題と対策が文書化 |
| ②DataOps | FT→評価→再デプロイのサイクル2回以上、改善効果レポート化 |
| ③Agent自律適応 | 人介在頻度が初期比30%以上削減（または根拠データあり） |
| ④現場実証 | PoC結果レポート完成、「成功率Y%、人手比Z%効率化」と説明可能 |

**β版（3月）の定義**
- 状態：運用安定、継続学習が回っている状態
- 成功率：90%程度
- 安全性：現場固有の安全パターン対応、異常時の自動停止
- 提供形態：社内実証（無償）

**Go/No-Go判定（3月）**
- Go条件：タスク成功率70%以上、継続学習ループ1回転以上、事業部が「検討に値する」と判断
- No-Go時：対象タスク見直し or 技術課題深掘り
