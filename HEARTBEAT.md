# HEARTBEAT.md

## 6時間ごとにやること
- `git pull` でワークスペースを最新化（konukiがGitHubで編集した場合に反映）

## モーニングブリーフ（朝 7:00-9:00 の場合）

朝のHEARTBEATでは、以下をまとめてkonukiに送信する：

### 📅 今日の予定
- カレンダーから今日のスケジュールを取得
- MTGの準備が必要なものがあれば併記

### ✅ TODO確認
- `TODO.md` から未完了タスクを抽出
- 期限が近いもの、今日やるべきものをハイライト

### 📝 昨日のサマリー
- `memory/YYYY-MM-DD.md`（昨日分）から主要トピックを要約

### 💡 提案（あれば）
- プロジェクトの状況を見て、次にやるべきことを提案
- 見つけた問題点や改善案

## MTG前の資料整理

今日のカレンダーを確認し、各MTGについて以下を準備：

### 対象MTG例
- Microsoft Weekly Sync → `Projects/rfa/` 関連資料、前回の議事録
- RFA Project Mtg → master-context.md、進捗状況
- データ戦略相談 → データフライホイール関連
- 1on1 → 相手との前回の話題、フォローアップ事項

### 出力形式
```
📋 [MTG名] (HH:MM-)
関連資料:
- ファイル1
- ファイル2
要点/アジェンダ案:
- ...
```

MTGの1時間前、または朝のモーニングブリーフ時にまとめて提示。

## ドキュメント整合性チェック（週次）

週1回、以下をチェック：

### チェック対象
- `Projects/rfa/master-context.md` （マスター）
- `Projects/rfa/fy26-business-plan.md`
- `Projects/rfa/data-flywheel-diagram-plan.md`
- その他プロジェクト資料

### チェック項目
- 用語の不一致（同じ概念に違う名前が使われていないか）
- 日付・マイルストーンの矛盾
- 古い情報が残っていないか
- TODOやWIPの放置

### 出力形式
```
🔍 ドキュメント整合性レポート (YYYY-MM-DD)
✅ 問題なし: X件
⚠️ 要確認: Y件
  - [ファイル名] 〇〇と△△で用語が不一致
  - [ファイル名] 日付が古い可能性
```
