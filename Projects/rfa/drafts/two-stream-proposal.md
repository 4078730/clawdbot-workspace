# 2ストリーム体制提案

*送信日: 2026-02-24*
*宛先: 大坪さん、奥村さん*
*ステータス: 送信済み ✅*

---

プロジェクト分割の進め方について、奥村さんとも話し合った結果を共有。

## 背景の課題認識

1. **「何がいつ出るのか」が見えにくい** — α版・β版・研究が混在し、事業化成果の見通しが不明確
2. **事業化へのコミットメントが不明確** — 全員が「全部やります」→ 優先順位不明確 → スピードが出ない
3. **マネジメントの物理的制約** — 京橋・横浜分散で1人PMでは目が届かない
4. **奥村さんの成長機会** — PM経験を通じたプロジェクトリードの幅の拡大

## 現状の課題

- **フォーカスの分散** — 研究テーマと事業化タスクが同じ優先度で並んでいる
- **責任の曖昧さ** — 黄瀬PMが全領域のボトルネック。京橋側の判断・現場調整は現地リーダーがリードすべき
- **説明の分かりにくさ** — 技術レイヤーでの整理は内部的に機能するが「何がいつ出るの？」に答えられていない

## 提案：1プロジェクト・2ストリーム体制

共通技術（DataOps/MLOps、RoboSync基盤、安全制御等）が多く完全分割は非効率。α版にも研究側技術が不可欠であり、分けるとコミュニケーションコストが増大するリスクあり。

```
RFA プロジェクト（PM: 黄瀬）

■ 共通基盤（両ストリーム横断）
  DataOps/MLOps、RoboSync基盤、MS連携
  予算・プロジェクト管理
  エンビジョニング・現場連携: 風見さん

■ Stream A: Robo Sync AI Suite（α版リリース）
  リーダー: 奥村
  メンバー: 桑田、鎌田
  ミッション: Q2末α版リリース
  権限: α版スコープ内の技術判断・現場調整は奥村が自律的に決定
  拠点: 京橋

■ Stream B: Robo Sync Seeds（VLA/Agent技術深化）
  リーダー: 黄瀬 (小栗?)
  メンバー: 小栗、Zhu、黒川、村上、加賀屋
  ミッション: β版以降のVLA事業価値実証、双腕複雑タスク
  拠点: 横浜
```

## 解決できること

- α版のデリバリー責任と権限が奥村さんに明確に移る → スピード向上
- 「どこで何の成果が出るか」がストリーム単位で可視化 → 成果の説明性向上
- 共通技術基盤や予算・成果は一元管理 → 管理コストの二重化回避
- 奥村さんがStream Aリーダーとしてプロジェクト推進を経験 → 育成
