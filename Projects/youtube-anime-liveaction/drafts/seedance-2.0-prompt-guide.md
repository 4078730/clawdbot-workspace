## なぜ「地上最強」なのか

Seedance 2.0はByteDance即梦AI (Jimeng) の最新世代ビデオモデル。最大の特徴は**「言うことを聞く」**こと。

- **画質が「ボケ」ない** — ネイティブ2K解像度対応。肌のテクスチャ・毛穴まで鮮明で、「美肌加工感」から卒業。
- **人物が「崩壊」しない** — 独自の「デュアルブランチ」技術により、指定キャラクター（@Image）が激しい動きをしても顔が別人にならない。
- **万物を「制御」できる** — 画像（見た目固定）+ 動画（動き固定）+ 音声（リズム固定）を同時にアップロードし、ブロックを組むように映像を作れる。

## `@`構文：Seedance 2.0の魂

入力欄で`@`を打つと、AIに「誰が主役か、何を参照するか」を指示できる。これがSeedance 2.0と他ツールの最大の違い。

### @Image（アイデンティティ・アンカー）

- **役割:** 「これが俳優です」
- **使い方:** キャラクター画像をアップロードし、`@Image1`としてマーク。
- **効果:** 動画がどう動いても、AIはこの画像の顔・服装・光影を**完全にロック**する。
- **上級:** キャラクターの「三面図」（正面・側面）を全てアップロードすると、AIが3D構造をより正確に理解する。

### @Video（動作ディレクション）

- **役割:** 「この通りに演じて」
- **使い方:** 自分がリビングで手を振る動画を撮影し、`@Video1`としてマーク。
- **効果:** `@Image1`のキャラクターが、あなたと全く同じ手振り動作を再現する（高度な「動画→動画」変換）。

### @Audio（リズム同期）

- **役割:** 「このビートに合わせて」
- **使い方:** 爆発音やダンスミュージックをアップロード。
- **効果:** 映像がオーディオの波形に自動で同期（ビートに合わせた振動・リップシンク）。

## 実践シナリオ（3つの即戦力パターン）

### シナリオA：写真を動かす（最もシンプル）

**用途:** 製品画像や人物写真を動画化したい場合。

1. 画像をドラッグ＆ドロップ
2. プロンプト入力：

```
@Image1 カメラが黒衣の男を追跡しながら全速で逃走する。
背後から群衆が追いかけてくる。カメラがサイドトラッキングに切り替わり、
男がパニック状態で路傍の果物屋台にぶつかって倒し、
起き上がって逃走を続ける。群衆の混乱した声。
```

3. 生成ボタン。

**原理:** AIが画像を第1フレームとして、テキストに基づいて後続の動作を補完。

### シナリオB：動作コピー（ダンス・ネタ動画）

**用途:** キャラクターに特定の動作をさせたい場合。

1. アップロード：
   - 画像：キャラクター写真（→ `@Image1`）
   - 動画：参照ダンス動画（→ `@Video1`）
2. プロンプト入力：

```
@Image1 が主体。@Video1 のダンス動作と体の動きの幅を厳密に参照。
キャラクターがダンスを踊る。
```

3. 生成ボタン。

**原理:** 動画から「骨格」を抽出し、画像の「外見」に被せる。

### シナリオC：ストーリー改変（上級）

**用途:** 既存動画のストーリーを自由に書き換える。

1. アップロード：元動画（→ `@Video1`）
2. プロンプト入力：

```
@Video1 のストーリーを覆す。
男の目が優しさから一瞬で冷酷に変わり、
女性が油断した瞬間、力強く橋から突き落とす。
動作は迷いなく、計画的な決意を感じさせる。
水に落ちる女性、叫び声はなく、信じられない目で見上げる。
```

3. 生成ボタン。

## 基本ルール

- **語数は30〜100語。** 短すぎると曖昧、長すぎると焦点がぼやける。
- **メインの被写体から書き始める。** AIが最初に読んだ要素に集中する。
- **マルチモーダル入力時は `@AssetName` で参照。** 例: `@Image1` `@Video1` `@Audio1`
- **イテレーションは1変数ずつ。** 照明だけ、動きだけ、スタイルだけ。
- **ネガティブプロンプトは控えめに。** 不要なものを除外する指示は有効だが、複雑にしすぎない。
- **日本語プロンプト対応。** Seedance 2.0は多言語対応。日本語でも動作するが、英語の方が安定する場合もある。

## Tips

- **動きの記述は具体的に**（「動く」ではなく「左を見ながらゆっくり前に歩く」）
- **カメラワークを明示**（ドリー、パン、オービット、トラッキングショット、クローズアップ、ワイドショット）
- **雰囲気は形容詞2-3個**（シネマティック・暗い・ハイコントラスト / 柔らかい・温かい・夢のような）
- **物理描写を入れる**（風に揺れる髪、雨粒、砂埃の粒子）
- **1つのカメラ動作に絞る**（ドリーイン+オービット+ラックフォーカスは欲張りすぎ）
- **スタイルは1つに決める**（スタジオ+水彩+グリッチ+フィルムは混乱する）

## よくある質問（FAQ）

**Q: 生成された顔がまだ似ていない**
A: `@Image1`の重み付けを上げるか、キャラクターの**多角度写真**（側面・背面）を補助参照としてアップロード。AIに3D構造を教える。

**Q: 画面がチラつく（Flickering）**
A: プロンプトと参照画像が矛盾している場合に起きやすい（例：画像は昼間なのにプロンプトに「夜」と記述）。**参照画像の照明条件とプロンプトの記述を一致させる。**

**Q: 参照動画がない場合は？**
A: **自分で撮る！** Seedance 2.0は素材の質を問わない。自宅でスマホ撮影した粗い歩行動画でも、完璧な動作軌道を抽出して映画キャラクターに適用できる。

## ビジュアルスタイル一貫性テンプレート

AI動画で最も頭が痛い問題の一つ：

> 内容は正しい、キャラも固定できた。でも生成された映像は——
> このカットは暖色文芸調、次のカットは冷色サスペンス調、
> その次は謎のレトロフィルター。
> 単独で見ればOKだが、つなげるとコラージュのようになる。

**原因はスタイルを固定していないこと。** 照明・色温度・雰囲気・質感・フィルターを指定しなければ、AIは毎回ランダムに決める。

### 使い方

1. プロジェクトに合うスタイルを選ぶ
2. そのスタイル記述をコピー
3. **各カットのプロンプト末尾に貼り付ける**

これだけで、全カットが「同じ作品の雰囲気」に収まる。

### スタイル設計：5次元分解

#### 1. コントラスト
- レベル（低/中/高/極高）
- 光比（2:1 / 4:1 / 8:1 / 12:1+）
- ブラックポイント（暗部ディテール保持 / 純黒許容 / 大面積の潰れ黒）

#### 2. 彩度
- レベル（低/中/高）
- 調整幅（例：-30%、+20%）
- 特殊処理（ほぼ無彩色 / 単色強調 / 高彩度コントラスト配色）

#### 3. 色温度
- 全体傾向（寒色/ニュートラル/暖色）
- ケルビン値範囲（3500K寒色、5500Kニュートラル、6500K暖色）
- 寒暖分離戦略（統一色温 / 寒暖分離 / 背景寒色＋被写体暖色）

#### 4. 影の硬さ
- 光質（硬い光/柔らかい光/ミックス）
- 影のエッジ（硬い切り分け / グラデーション / 柔らかく拡散）
- メインライト方向（正面/側面/逆光/トップライト＋角度）

#### 5. レンズ言語
- 焦点距離（広角<35mm / 標準35-50mm / 中望遠>85mm）
- 被写界深度（パンフォーカス / 浅い被写界深度で主体分離）
- 構図傾向（対称 / 三分割法 / 大胆な余白 / タイト構成）
- 特殊レンズ（アナモルフィック / ヴィンテージフレア / なし）

### 補助次元（オプション）

| 次元 | 選択肢 |
|------|--------|
| 粒子感 | なし / 微細5-10% / 目立つ15-25% |

## Kling 3.0 × Freepik ワークフロー

> Source: [@CharaspowerAI](https://twitter.com/CharaspowerAI)

**1枚の画像 + 1つのプロンプト**だけで、最大15秒のシネマティックシーケンスを生成できる。カット編集なし。

### ワークフロー

**Step 1: ベース画像を作る**

まずシネマティックで印象的な画像を生成。この1枚が映像全体のビジュアル基盤になる。

```
Full-body shot of a damaged android with torn synthetic skin
revealing chrome beneath, wearing a long black coat,
holding a glowing spear, standing among destroyed buildings
in a dystopian city. Silver sparks and reflections create
a cinematic, action movie poster-style scene with motion blur,
high-resolution photography, hyper-detailed, dynamic pose,
and hyper-realistic details.
```

**Step 2: Freepik Storyboard機能でアクション画像を生成（チートコード）**

Freepikの**Storyboard機能**を使って、ハイライトしたいアクションが**既に含まれた画像**を生成する。

- アクションを具体的に記述する
- インスピレーションがなければAIに任せてもOK

→ 結果：シネマティックな映像化のために**最初から設計された**ショットが得られる。

**Step 3: Kling 3.0に画像 + プロンプト1つで投入**

Storyboard画像をKling 3.0にドロップし、**シンプルなプロンプト1つ**を追加するだけ。

```
Make a montage of fast cut shots of an android fighting
against a swarm of insect drones, epic action scene from a movie,
include each shots from the story grid,
include some dramatic dialogue in English,
cinematic, epic, dramatic
```

→ Klingがスムーズで一貫性のある最大15秒のシネマティックシーケンスを生成。

### Pro Tips

- **同じ画像 + 同じプロンプトで2〜3回生成する。** Kling 3.0は自然にバリエーションを生成する（カメラモーション、ペーシング、微細なアクションの違い）。ベストテイクを選ぶだけ。
- **プロンプトに「montage」「fast cut shots」を入れる**とマルチショット風の映像になる。
- **「dramatic dialogue in English」**でキャラクターに台詞を喋らせる演出が可能。
