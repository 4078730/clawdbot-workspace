### 判定結果の4カテゴリ

| カテゴリ | 条件 | 制御方式 | 例 |
|---------|------|---------|-----|
| 🟢 **ルールベース十分** | PTG系 + 構造化可能 | 従来制御 / Robo Sync | 箱物P&P、固定経路搬送 |
| 🟡 **VLA有用** | 不確実性あるがSTGではない | VLA補助 | アプローチ角度の微調整 |
| 🔴 **VLA必須** | STG系 + Semantic Constraint支配 | **VLA Policy** | 袋物把持、梱包操作 |
| ⚫ **現時点で人間** | 道具操作 / 複合判断 / α版外 | 人間 | ハサミ切断、袋in袋開封 |

## 2. 彩都出荷プロセス全工程の判定結果

### 2.1 ピッキング工程

| タスク | タスクグループ | Semantic Constraint | 判定 | 根拠 |
|--------|-------------|-------------------|------|------|
| 箱物をビンから取り出す | PTG11 | なし | 🟢 ルールベース | 形状定型、把持点が自明 |
| 袋物をビンから取り出す | **STG11** | 袋を破かない | 🔴 **VLA必須** | 袋の形状・向き・掴み位置が毎回異なる |
| 仕切り内の特定パーツを認識 | — | — | 🟡 VLA有用 | VLMで対象特定。PMS情報との照合はルール |
| 袋in袋を開封→1個取り出し | STG11×複数 + 道具 | 破かない+数量+元位置 | ⚫ **人間** | 道具操作+複合判断 |
| バラ小物から1個選別 | STG11 + カウント | 数量+サイズ選別 | ⚫ **人間** | 微小物体の精密操作 |
| 段ボール開封→中身取り出し | STG11(力制御) | 箱を潰さない | ⚫ **人間**（α版） | 双腕+力制御が必要 |
| 残りパーツを元位置に戻す | **STG13** | 元位置記憶 | ⚫ **人間**（α版） | 空間記憶+慎重配置 |

### 核心的な発見

```
🟢 ルールベースで十分なタスク:
   → 箱物P&P、固定経路搬送、ラベル位置決め
   → VLAを使っても価値が出ない。Robo Sync既存機能で対応

🔴 VLA必須のタスク（STG支配）:
   → 袋物把持、化粧箱組立/封入、テープ貼り、緩衝材投入
   → ここがVLAの存在意義。Semantic Constraintの暗黙学習が鍵

⚫ 現時点で人間が担うべきタスク:
   → 袋in袋開封、バラ小物選別、輪ゴムくくり
   → 無理にVLAで攻めるべきではない。β版以降の研究課題
```

## 4. 池内先生へのディスカッション論点

### 「VLAを使わない判断」の理論的根拠

> 池内先生のPTG/STG分類は、VLAの適用判断に**必要十分な判定基準**を与えるか？

- PTG系 → ルールベースで十分という判断は妥当か？
- STG系でもSemantic Constraintが少ない場合はルールベースで対応可能では？
- **判定のグレーゾーン**（🟡 VLA有用）をどう扱うか？

### 「商品化工程がVLA集中エリア」という発見

- 商品化の4タスクが**全てSTG系 = 全てVLA必須**という結果
- これはiREXデモの梱包タスク分析と整合
- **α版のスコープを商品化に置く理論的根拠**として十分か？
