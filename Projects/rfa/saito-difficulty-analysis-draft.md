### 難易度の決定要因

| 要因 | 内容 | 影響度 |
|------|------|--------|
| **入れ子構造** | 袋の中に袋 → 何層の開封が必要か | 🔴 最大 |
| **取り出し単位** | 1個だけ取り出す vs 全体をピック | 🔴 大 |
| **ツール使用** | ハサミで切る等の道具操作 | 🔴 大（α版不可） |
| **空間記憶** | 残りを元の位置に戻す | 🟡 中（β版課題） |
| **形状の定型性** | 箱物=定型、ケーブル=不定形 | 🟡 中 |
| **サイズ** | 手のひらサイズ以上 vs 微小パーツ | 🟡 中 |
| **後工程の有無** | 個装不要（そのまま出荷）vs 個装必要 | 🟢 α版では個装スコープ外 |

## 3. 工程分解（風見さん観察ベースで更新）

### α版スコープの再定義

風見さんの動画分析から、α版で扱うべき工程を再定義。

```
  【現実のAS出庫作業フロー（風見さん動画より）】

  ① 納品書束（~10枚）を取る → スキャン
  ② AGVがビンをポートに搬送（待ち時間あり）
  ③ 納品書バーコード ↔ ビン内バーコード照合 ← NEW: 2重チェック確認
  ④ ビン内の対象パーツを特定
     ├─ 箱物 → そのまま取り出し（★☆☆）
     ├─ 袋入りパーツ → 袋ごと取り出し（★★☆）
     ├─ 袋in袋 → 外袋から内袋取り出し → パーツ1個（★★★★★）
     ├─ 段ボール内 → 開封 → パーツ取り出し（★★★★）
     └─ バラ小物 → 1個選別取り出し（★★★★★）
  ⑤ 使わないものを元のビン位置に戻す
  ⑥ 個装（袋入れ/プチプチ包み/ハサミ切断等）
  ⑦ ワゴンに配置
  ⑧ 次のオーダーへ

  α版スコープ: ④のうち★★☆以下のパターンのみ
  ──────────────────────────────────────────
  ・箱物そのまま（S4, S5）
  ・袋入りパーツそのまま（S2）
  ・手のひらサイズ個装済み小物（E袋ごと）
```

### α版の工程（限定スコープ）

| # | 工程 | 内容 | 制御方式 | 難易度 |
|---|------|------|---------|--------|
| 1 | ビン内認識 | 俯瞰カメラ+Hand-Eyeで対象パーツ特定 | VLM | ★★☆ |
| 2 | 把持計画 | 箱→Medium Wrap / 袋→Lateral Pinch or Vacuum | VLA | ★★☆ |
| 3 | アプローチ | 仕切り内へのアプローチ（深いビン対応） | VLA | ★★★ |
| 4 | 把持実行 | 吸着 or グリッパで把持 | VLA | ★★☆ |
| 5 | 引き上げ | ビンから持ち上げ（深さ対応） | VLA+ルール | ★★☆ |
| 6 | 搬送 | ビン→作業台 | ルール | ★☆☆ |
| 7 | 配置 | 作業台に安全配置 | VLA+ルール | ★☆☆ |

**α版固有の技術課題（風見さん情報から追加）:**

| 課題 | 詳細 | 対策案 |
|------|------|--------|
| **ビン深さ** | 相当深い → アームのリーチ制約 | UR5eリーチ850mm。仕切り内へのアクセス角度を検証 |
| **仕切り内アクセス** | 7割が仕切りあり → 横方向の制約 | Hand-Eyeカメラ必須。アプローチ角度の最適化 |
| **オクルージョン** | 俯瞰カメラだと手がオクルージョン | Hand-Eyeをプライマリに。俯瞰は状態確認用 |

## 4. α版の対象選定 — 3段階アプローチ

### 風見さんの示唆を反映した選定戦略

風見さんの核心的な判断:
> **「袋の中から1個取り出す工程が発生しないプロセスに限定すべき」**
> **「バラ積みをそのまま扱うだけのプロセスが望ましい」**

これを踏まえ、α版の対象を3段階で拡大する。

### Stage 1: ベースライン確立（最初の1-2ヶ月）

**対象:** 箱物そのまま出荷（S4, S5）

| 項目 | 内容 |
|------|------|
| **ビン内状態** | 段ボール箱がそのまま入っている |
| **操作** | 箱をピック → ワゴン or 作業台に配置 |
| **EE** | 2F-85（平行グリッパ） |
| **Grasp Type** | Medium Wrap / Large Diameter |
| **難易度** | ★☆☆ |
| **VLA価値** | 低（ルールベースでも可能） |
| **目的** | システム統合の検証、RoboSync連携、ログ収集基盤の確立 |

> ⚠️ VLAの価値証明には弱いが、**インフラ検証として必須のステップ**

### Stage 2: VLA価値実証（2-4ヶ月目）

**対象:** 袋入りパーツ（S2）+ 個装済み小物（E袋ごと）

| 項目 | 内容 |
|------|------|
| **ビン内状態** | 袋に入ったパーツ / 個装済み小型部品 |
| **操作** | 袋ごとピック → 作業台に配置 |
| **EE** | 2F-85（Lateral Pinch）+ SQHW-01（Vacuum） |
| **Grasp Type** | Lateral Pinch（袋端）/ Vacuum Grasp（袋面） |
| **難易度** | ★★☆〜★★★ |
| **VLA価値** | **中〜高** — 袋の形状・向き・掴み位置の判断にVLAが必要 |
| **目的** | VLA制御ポリシーの有効性を実証 |

> ✅ **α版の本丸。** 「袋ごと把持」戦略により難易度を抑えつつ、VLAの価値を示す

### Stage 3: 難易度拡張（4-6ヶ月目）

**対象:** 仕切り内の複雑配置（S7相当）

| 項目 | 内容 |
|------|------|
| **ビン内状態** | 仕切り4区画内に複数種類のパーツ、一部重なり |
| **操作** | 特定区画の特定パーツをピック（障害物回避含む） |
| **EE** | 2F-85 + SQHW-01 |
| **難易度** | ★★★〜★★★★ |
| **VLA価値** | **高** — 空間推論・障害物回避・アプローチ軌道生成 |
| **目的** | 複雑なビン内状態へのVLA対応力を実証 |

> ⚠️ 「袋の中から1個取り出す」は含めない。あくまで「見えているパーツをそのまま取る」範囲

## 5. 重要な戦略的示唆

### 5.1 α版スコープの現実的限界

風見さんの動画分析で明らかになった**最も重要な事実:**

> **最頻出パターン（袋in袋inパーツ）はα版のスコープ外**

つまり、α版で扱える商品は**出庫全体の一部**にしか過ぎない。

```
  出庫作業全体の構成（風見さん体感ベース）

  ┌──────────────────────────────────────────────┐
  │ 袋in袋inパーツ（最多）           ████████████│ → α版外
  │ 小物バラ1個取り                  ████████    │ → α版外
  │ 段ボール開封→パーツ取り出し      ██████      │ → α版外（β版）
  │ 袋入りパーツそのまま出荷         ████        │ → ✅ α版 Stage 2
  │ 箱物そのまま出荷                 ██          │ → ✅ α版 Stage 1
  │ 箱ごと出荷（ロット）             █           │ → ✅ α版 Stage 1
  └──────────────────────────────────────────────┘
       ← α版でカバーできる範囲は限定的 →
```

### 5.2 プロパティ登録との連携が鍵

中田さんヒアリング（2/6）で出た「プロパティ登録」戦略が**α版の実現可能性を大きく左右**する:

> 入庫時に「ロボット把持可能」フラグ → 専用ビンに格納 → 専用ポートに流す

これにより：
- ロボットが扱えるパーツ**だけ**がロボットポートに来る
- 袋in袋等の複雑なものは人が処理するレーンに流す
- α版の成功率を構造的に担保できる

### 5.3 レーン運用の現実的提案

風見さんの提案: **既存6レーンを潰すのではなく、レーンを増設する方向**

| 方式 | メリット | デメリット |
|------|---------|-----------|
| **既存レーン改造** | 追加設備不要 | 1レーン停止=処理量低下。現場反発 |
| **レーン増設（推奨）** | 既存オペ影響なし。段階的導入可 | 設置スペース・AutoStore改造が必要 |
| **梱包工程で導入** | レーン問題を回避。並行作業可 | α版のスコープ変更が必要 |

### 5.4 梱包作業という代替選択肢

風見さんのコメント:
> 「梱包やプチプチで包む作業の方が、まだ得意かもしれない」

これはα版のスコープ自体を見直す可能性を示唆。ただし：
- 梱包作業の動画はまだ未分析
- 双腕が必要になる可能性（β版スコープ）
- 現時点では**情報不足で判断できない**

→ **次アクション:** 梱包作業の動画も分析してもらう

## 8. 出荷プロセス全体像（中田さんレポート 2023-01-19）

konuki共有の中田さん視察レポートより、出荷プロセスの全体フローが判明。

```
出荷指示 → ピッキングバッチ生成 → ピッキング → 流通加工 → 商品化 → 梱包 → 出荷
                                      ↑              ↑          ↑        ↑
                                   AS or 棚     袋入れ等    化粧箱封入  輸送箱梱包
                                   (6レーン)    (台車上)    (別部署)   (別エリア)
```

### 工程の詳細（レポート+画像読み取り）

| 工程 | 内容 | 場所 | レーン占有 |
|------|------|------|-----------|
| **ピッキング** | ASビンからパーツ取り出し | ASカルーセル（6レーン） | 🔴 占有する |
| **流通加工** | 袋入れ等の前処理。物コメの指示に従い封入 | 台車上 / ピッキング台と共通 | 🟡 ピッキングと一体 |
| **商品化** | 前処理済み商品を指定化粧箱に封入 | **別部署（商品化部署）** | 🟢 占有しない |
| **梱包** | 出荷先ごとに輸送箱に梱包 | **別エリア** | 🟢 占有しない |

### 重要な発見：商品化・梱包はレーン外

**商品化と梱包はASレーンとは別の場所で行われる。**

- ピッカーが棚から梱包用の箱を取り、商品と一緒に商品化部署に引き渡す
- つまり、商品化・梱包工程にロボットを導入しても**ASレーンを止める必要がない**

## 9. スコープ再検討：ピッキング vs 商品化/梱包

### konukiの指摘

> 「ASのエレベーター自体が6レーンあって、これの一つを占有するというのが実証としてのハードルになってしまいそう」
> 「商品化、梱包の工程のほうが、場所を借りて部分的に対象物を回してもらうだけでレーンを止めずに済むかもしれない」

### 比較表

| 項目 | ピッキング（現α版計画） | 商品化/梱包（代替案） |
|------|----------------------|---------------------|
| **場所** | ASカルーセル（6レーンの1つ） | 別部署・別エリア |
| **レーン占有** | 🔴 1レーン停止 = 処理量低下 | 🟢 既存オペ影響なし |
| **現場調整難易度** | 🔴 高（6→5レーン） | 🟢 低（並行設置可） |
| **対象物の状態** | 多様（袋in袋が最多） | より構造化（前処理済み） |
| **作業内容** | ピック→配置 | 化粧箱封入 / 輸送箱梱包 |
| **腕の要件** | 単腕で可能（α版スコープ内） | 双腕が必要な可能性 |
| **VLA価値** | 把持の多様性で実証 | 梱包操作の柔軟性で実証 |
| **α版ハードウェア** | UR5e単腕（既定） | 双腕が必要→β版HW前倒し？ |

### konukiの難易度修正

| Rank | 修正内容 |
|------|---------|
| S4（段ボール箱ごと出荷） | ❌に変更 — **ハンドで掴めない**（サイズ超過） |
| S2（袋入りパーツ） | 頻度を「多い」→「中程度」に修正 |
