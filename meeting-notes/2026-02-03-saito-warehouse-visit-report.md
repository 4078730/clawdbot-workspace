*2026年2月3日 RFAプロジェクト VLA適用可能性調査*

---

## 1. エグゼクティブサマリー

彩都パーツセンターの現場見学を通じて、**VLA（Vision-Language-Action Model）の適用可能性が高いタスクを特定**しました。

### 得られた3つの知見

| 知見 | 内容 | RFAへの効果 |
|------|------|-------------|
| **VLA適用の有力候補タスク** | オートストア出庫のピッキング作業（ビンから商品を取り出す） | 現場実証の対象タスクとして最適 |
| **GMRフレームワークとの適合** | ビンからの取り出し動作は典型的なGMR（把持→操作→解放） | 池内先生との議論で確立した理論が直接適用可能 |
| **リカバリースキルの重要性** | 上積み保管による回避動作、ピック失敗時のリトライが必要 | リカバリー専用GMRの研究テーマとして有望 |

### 事業的意義

- **現場実証のフィールド確保** → 実際の倉庫タスクでVLAを検証できる
- **データフライホイールの起点** → 作業データの収集・学習サイクルを構築
- **Industrial Embodied AI連携** → 一力さんチームとの協業によりスケール可能

## 2. 見学の背景と目的

### 2.1 なぜ彩都パーツセンターか

RFAプロジェクトでは、**VLAを実際の現場で動かす**ことを目標としています。そのためには、以下の条件を満たす現場が必要でした。

- **繰り返し作業が存在する** → VLAが学習すべきスキルが明確
- **改善余地がある** → 自動化による価値が大きい
- **データ収集が可能** → 作業の記録・蓄積ができる環境

彩都パーツセンターは、これらすべてを満たす現場として選定しました。

### 2.2 見学の目的

1. **現場作業の実態把握**：オートストア出庫タスクの詳細を理解する
2. **VLA適用可能性の検討**：どの作業にVLAを適用できるか特定する
3. **技術的課題の洗い出し**：実適用に向けて解決すべき課題を明確化する
4. **Industrial Embodied AIチームとの連携**：一力さんチームの取り組みを理解し、協業の可能性を探る

## 3. 彩都パーツセンターの概要

### 3.1 施設の位置づけ

彩都パーツセンターは、**パナソニックのサービスパーツ出荷拠点**です。業務用プロジェクター、ディスプレイ、リモートカメラ、カメラレコーダー、ノートパソコン（Let's Note）など、多様な製品の補修部品を国内外に出荷しています。

**【写真①挿入箇所】**: 彩都パーツセンター全景

| 項目 | 内容 |
|------|------|
| 所在地 | 大阪府茨木市彩都あかね1 MFLP茨木 6階 |
| 稼働開始 | 2018年10月〜 |
| 面積 | 約5,000㎡ |
| 在庫品番数 | **12万品番** |
| 月間出荷点数 | **3万点** |
| 従業員 | 45人（社員）+ 30人（請負） |
| 出荷先 | 国内全国 + **海外24カ国** |

### 3.2 オートストア設備

彩都パーツセンターの中核となるのが、**オートストア**（AutoStore）システムです。これは、AGV（自動搬送車）がビン（コンテナ）を自動で取り出し、作業者のもとへ運ぶ自動倉庫システムです。

**【写真②挿入箇所】**: オートストア全景

| 出庫レーン | 約6箇所 |
| 保管ルール | 高頻度品→上段、低頻度品→下段に自然移動 |

## 4. Industrial Embodied AIの取り組み

見学に先立ち、一力さんから**Industrial Embodied AI**の取り組みについてご説明いただきました。この取り組みは、RFAプロジェクトが目指す方向性と高い親和性があります。

### 4.1 「1秒の差」の重要性

一力さんが強調されたのは、**物流現場における「1秒」の価値**です。

> 「1秒 × 1万回 = 3時間の差になる」

作業者間のばらつき（Aさん5秒、Bさん8秒、Cさん12秒）が、計画との大きな乖離を生みます。また、サイバー（システム上のデータ）とフィジカル（実際の現場）のギャップも課題です。例えば、倉庫の充填率はシステム上のデータと実際で大きく異なることがあります。

### 4.2 付加価値作業の割合

物流現場の改善を考える上で重要な指標が、**付加価値作業の割合**です。

| 現場タイプ | 付加価値作業の割合 |
|-----------|-------------------|
| 一般的な物流現場 | 10% |
| **彩都パーツセンター** | **30%**（世界で1社だけ達成） |
| 物流業界平均 | 1% |

彩都パーツセンターは、業界でもトップクラスの効率を実現しています。しかし、それでも**70%は非付加価値作業**（移動、待機など）であり、さらなる改善余地があります。

### 4.3 生産性改善の歴史と将来目標

**【写真③挿入箇所】**: ダッシュボード画面

| 年 | 施策 | ピッキング時間 |
|----|------|---------------|
| 2017年 | 標準化開始 | 113秒 |
| 2018年 | オートストア導入 | 160秒 → 54秒 |
| 2026年（目標） | ASRS + ピッキングロボット + タスク最適化 | **30秒** |
| 将来 | **無人化** | - |

2018年のオートストア導入後、ピッキング時間は113秒から54秒へと大幅に短縮されました。現在の目標は30秒であり、将来的には無人化を目指しています。**VLAはこの無人化に向けた重要な技術要素**となります。

## 5. 見学した各ソリューション

現場では、実際に稼働している以下の4つのソリューションを見学しました。

### 5.1 積載量可視化

**【写真④挿入箇所】**: 積載量可視化システム

カート、コンテナ、棚の**充填率を自動計測**するシステムです。画像認識により、「どのくらい空きがあるか」をリアルタイムで把握できます。これにより、システム上のデータと実際の状態のギャップを解消しています。

### 5.2 V-SLAM

**【写真⑤挿入箇所】**: V-SLAMによるフォークリフト追跡

**非GPS空間での自己位置推定**技術です。倉庫内ではGPSが使えないため、カメラ映像から自己位置を推定します。フォークリフトや牽引車の動きをリアルタイムで把握し、動線分析に活用しています。

### 5.3 ダッシュボード

**【写真⑥挿入箇所】**: 統合ダッシュボード

以下の情報を**統合的に可視化**するダッシュボードです。

- 進捗管理
- 生産性分析
- 動線分析
- 収支分析
- 在庫管理

現場の状況をリアルタイムで把握し、迅速な意思決定を支援しています。

### 5.4 AI画像処理

**【写真⑦挿入箇所】**: AI画像処理による作業分析

標準作業プロセスに基づき、**リアルタイムで作業を分解・分析**するシステムです。作業者の動きをカメラで捉え、各工程にかかる時間を自動計測します。これにより、作業者ごとのばらつきを可視化し、改善ポイントを特定できます。

## 6. VLA適用の有力候補：オートストア出庫タスク

見学を通じて、**オートストア出庫タスク**がVLA適用の有力候補として浮上しました。

### 6.1 タスクの流れ

**【写真⑧挿入箇所】**: 出庫レーン作業風景

```
【オートストア出庫タスク】

①注文発生
  ↓
②AGVが倉庫内のビンを自動取得
  ↓
③出庫レーン（6箇所）にビン搬送
  ↓
④[人間作業] 売上伝票を確認
  ↓
⑤[人間作業] ビンから商品をピック ← VLA適用ポイント
  ※上積み保管のため、別商品を避けて取る必要あり
  ※複数ビンからのピックあり
  ↓
⑥[人間作業] バーコードスキャン
  ↓
⑦[人間作業] 個装ラベル貼付
  ↓
⑧出荷エリアへ移動
```

④〜⑦の人間作業部分が、VLA適用の対象範囲です。特に**⑤ビンから商品をピック**する作業は、GMR（Grasp-Manipulation-Release）の典型例であり、VLAによる自動化の可能性が高いと考えられます。

### 6.2 GMRフレームワークとの適合

池内先生との議論で確立した**GMRフレームワーク**の観点から、このタスクを分析します。

| GMRフェーズ | 出庫タスクでの動作 |
|------------|-------------------|
| **Grasp（把持）** | ビン内の商品を掴む |
| **Manipulation（操作）** | ビンから持ち上げて移動 |
| **Release（解放）** | 出荷コンテナに置く |

このように、ビンから商品を取り出す動作は、GMRの3フェーズにきれいに対応しています。**VLAで学習すべきスキルの粒度として適切**であることがわかります。

### 6.3 技術的課題

一方で、実適用に向けてはいくつかの課題があります。

| 課題 | 内容 | GMR/リカバリーとの関連 |
|------|------|----------------------|
| **上積み保管** | 対象商品の上に別商品が存在する可能性 → 回避動作が必要 | **リカバリー専用GMR**が必要 |
| **複数ビンピック** | 1注文で複数ビンからピックするケース | LLMによるプランニングと連携 |
| **ポート重複** | 他ポートと商品が重複した場合の待機判断 | Agent層での判断ロジック |
| **商品多様性** | 12万品番 → 形状・サイズの多様性への対応 | 潜在空間でのスキル汎化 |
| **リカバリー** | ピック失敗時のリトライ動作 | **リカバリー専用GMR**が必要 |

特に**リカバリー**は重要なテーマです。池内先生との議論でも、「100%成功は困難であり、リカバリーを独立スキルとして設計すべき」という方針が確認されています。上積み保管による回避動作や、ピック失敗時のリトライは、まさにこのリカバリースキルの研究対象として有望です。

## 8. まとめ

彩都パーツセンターの見学を通じて、以下のことが明らかになりました。

1. **オートストア出庫タスク**は、VLA適用の有力候補である
2. ビンからの商品ピックは**GMRフレームワーク**に適合し、学習単位として適切
3. 上積み保管への対応やピック失敗時のリトライなど、**リカバリースキル**の研究テーマが豊富
4. Industrial Embodied AIチームとの連携により、**データフライホイール**の構築が可能

この知見を踏まえ、3月のミニデモに向けてVLA適用の具体化を進めていきます。
