## 1. 対象業務

### 1.1 現場概要

| 項目 | 内容 |
|------|------|
| **拠点** | 彩都パーツセンター |
| **業務** | オートストア出庫作業 |
| **規模** | 12万品番、月間3万点出荷 |
| **設備** | オートストア（2万ビン、AGV14台） |
| **付加価値作業比率** | 30%（業界トップ） |
| **関係者** | 一力さん（現場）、中田真一さん（ヒアリング済）、風見さん（エンビジョニング） |

### 1.2 出庫作業フロー（動画分析 2026-02-16）

```
[1] 準備・登録
    納品書の束を作業台に持ってくる → モニター操作で作業者登録

[2] スキャン・チェック
    バーコードリーダーで納品書を1枚ずつスキャン（〜10枚/束）
    ペンでチェック

[3] ビン到着・ピック
    オートストアからビンが作業者の右側に到着
    ビン内構造: 段ボール → 緩衝材（プチプチ等）→ パーツ
    納品書に対応するパーツを確認・ピック

[4] 個装（物による）
    パーツに合った袋に入れ、納品書を同封
    袋を閉じて左側のワゴンに配置

[5] 次のオーダーへ
    1オーダー: 約1〜3分
```

### 1.3 作業の複雑さ

**ルールベース（現RoboSync）では対応困難な要因：**

| 要因 | 詳細 |
|------|------|
| **商品形状の多様性** | 12万品番。箱物・袋物・シール台紙・小型ネジ等が混在 |
| **包装状態の多様性** | 段ボール封あり/なし、プチプチ包装、個装袋、仕切り付きビン等 |
| **操作の多様性** | 掴む/引き抜く/袋から取り出す/シートから切り取る等 |
| **吸着のみでは不可** | 不定形・柔軟物・小型パーツ → 把持方法の選択自体が必要 |
| **標準化不可能** | 商品ごとにオペレーションが異なり、事前定義は非現実的 |

→ **VLAによる汎用的な操作制御ポリシーが不可欠**
（把持点推定の問題ではなく、制御そのものの問題）

### 1.4 作業環境（動画スクリーンショットより）

## 2. フェーズ定義

### 設計原則

> **MVP原則:** 各フェーズはPoCではなく、スコープ限定のMVP。
> Functional → Reliable → Usable を薄くでも全層通す。
> α版→β版→GAが連続的な改善サイクルとなり、投資が累積的に価値を生む構造。

> **ルールベースとVLAの役割分担:**
> VLAはルールベースの「置き換え」ではなく「段階的補完」。
> ルールベースで対応可能なものはルールベースで。困難なタスクをVLAが補完。
> 段階的にVLAの対応範囲を拡大し、最終的にVLA主力へ移行。

> **複雑さの3軸（時間的に分離して取り組む）:**
> 1. **手の複雑さ** — エンドエフェクタの種類・指の制御
> 2. **腕の複雑さ** — 単腕/双腕、操作制御の汎用性
> 3. **移動の複雑さ** — 固定ステーション / 移動を伴う作業

### 2.1 α版（CA: Controlled Availability）— Q2末/9月

**ゴール:** RoboSync上でVLA制御ポリシーが動作し、ルールベースでは困難なパーツのピックを実現する

#### スコープ

| 項目 | 内容 |
|------|------|
| **IN** | ビン内パーツのピック → 作業台に配置 |
| **OUT** | 個装、梱包、台車への移動、納品書処理 |
| **対象パーツ** | 把持しやすい形状から開始（具体ケースはRoboSyncチームと選定） |
| **人の役割** | 個装・梱包・納品書処理は人が担当（人×ロボット協働） |

#### ハードウェア

| 項目 | 仕様 |
|------|------|
| **ロボットアーム** | UR5（単腕） |
| **エンドエフェクタ** | 吸着 / 平行グリッパ（タスクに応じて選択。要：奥村さんと検討） |
| **指** | 使用しない |
| **カメラ** | TBD（RealSense D435i想定） |
| **移動機構** | なし（固定ステーション） |

#### 技術要件

| # | 要件 | 詳細 |
|---|------|------|
| T-1 | VLA制御ポリシー | ビン内の多様な形状パーツに対して、操作軌道を生成・実行 |
| T-2 | RoboSync統合 | VLAスキルブロックとしてRoboSync上で実装・実行・管理 |
| T-3 | データ収集 | 実行ログ（画像・ロボット状態・成功/失敗）の自動収集パイプライン |
| T-4 | 安全制御 | RoboSync標準安全制約に準拠。最低限の安全制御がRoboSync上で管理されること |
| T-5 | ルールベース連携 | ルールベースで対応可能なケースはルールベースで処理。VLAは補完 |

#### 成功基準

| 指標 | 目標 | 備考 |
|------|------|------|
| ピック成功率 | TBD（要検討） | 対象パーツを絞った上での成功率。人の補助前提で現実的な値を設定 |
| VLA付加価値 | 定性的に実証 | 「ルールベースでは対応困難だったケースをVLAで処理できた」ことの実証 |
| データ蓄積 | パイプライン稼働 | 学習に使えるデータが自動で蓄積される状態 |
| 安全性 | インシデント0 | RoboSync安全制約内で運用 |

#### 事業的価値（内部レビュー向け説明）

> 彩都パーツセンターの出庫作業は12万品番の多様な形状・包装状態のパーツが混在し、
> オペレーションの標準化が不可能。吸着のみのルールベース制御では対応できない。
> VLAの汎用操作制御ポリシーをRoboSync上に実装し、
> 特定顧客向け（CA）に「ルールベースでは自動化できなかった作業の自動化」を実証する。
> 完全自律ではなく、人との協働（個装は人が担当）で段階的に価値を提供する。

### 2.2 β版 — Q4末/3月

**ゴール:** 個装・梱包を含む複合オペレーションへ拡張。継続学習サイクルが回っている状態。

#### スコープ

| 項目 | 内容 |
|------|------|
| **IN** | ピック → 梱包材からの取り出し → 個装 → 作業台配置 |
| **個装** | ✅ 含む（挑戦的目標） |
| **梱包** | ✅ 含む |
| **移動** | △ 要検討（台車への配置まで入れるか） |
| **対象パーツ** | α版で扱った範囲を多品種に拡張 |

#### ハードウェア

| 項目 | 仕様 |
|------|------|
| **ロボットアーム** | UR5（単腕）→ 一部双腕（ラボ検証並行） |
| **エンドエフェクタ** | 吸着 / 平行グリッパ |
| **指** | 使用しない（研究は並行で進行中 — 福岡拠点） |

#### 技術要件

| # | 要件 | 詳細 |
|---|------|------|
| T-6 | 梱包材取り出し制御 | プチプチ・段ボールからのパーツ取り出し。操作軌道の汎用化 |
| T-7 | 個装スキル | パーツに合った袋への挿入、納品書同封、袋閉じ |
| T-8 | 梱包スキル | 段ボールの開封を含む梱包関連操作 |
| T-9 | ピック失敗リカバリー | Agent再計画によるリトライ・代替手順 |
| T-10 | 継続学習 | α版データでFine-tuning → 再デプロイサイクルが安定稼働 |
| T-11 | 現場固有安全 | 異常時自動停止、人介在データ記録フロー |

#### 成功基準

| 指標 | 目標 | 備考 |
|------|------|------|
| ピック成功率 | 90% | 拡張した品種範囲での成功率 |
| 継続学習 | 2サイクル以上 | データ収集→FT→再デプロイ→改善確認 |
| 人介在頻度 | α版比30%削減 | または削減見込みの根拠データ |

### 2.3 GA（General Availability）— FY27〜

**ゴール:** エンドツーエンドの出庫作業自動化。正式製品版として汎用顧客向けに提供。

#### スコープ

| 項目 | 内容 |
|------|------|
| **IN** | ピック → 個装 → 梱包 → 台車配置（エンドツーエンド） |
| **移動** | ✅ 含む |
| **対象** | 複数顧客向けに横展開可能な状態 |

#### ハードウェア

| 項目 | 仕様 |
|------|------|
| **腕** | 双腕 |
| **エンドエフェクタ** | 吸着 / 平行グリッパ / 指（研究成果統合） |
| **移動** | 移動ロボット統合 |

#### 技術要件

| # | 要件 |
|---|------|
| T-12 | フルエージェンティック動作（複数スキル切替、コンテナ操作） |
| T-13 | 双腕協調制御 |
| T-14 | 指の制御（長期研究成果の統合） |
| T-15 | 移動を伴うオペレーション |
| T-16 | 安全要件文書化・監査対応、品質保証プロセス確立 |

#### 成功基準

| 指標 | 目標 |
|------|------|
| 成功率 | SLAとして定義 |
| 提供形態 | RoboSyncオプション（有償） |
