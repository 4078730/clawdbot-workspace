# 日次ノート 2026-02-26

## FY26体制MTG（島本・大坪・奥村・黄瀬）

### 主要決定事項

#### 1. α版→β版への呼称変更（重要）
- RoboSyncはCA/GAという段階を設けておらず、即GAとして出す文化
- 黄瀬が「α版」と呼んでいたもの（ピック→作業台配置・VLAスキルブロック）は、RoboSync定義では「**β版**」相当
- → **呼称を「β版」に統一する**。CA表現は使わない
- 現在の「β版」（個装・Agentic・双腕）はスキル拡張の別軸として表現

#### 2. 定義の軸を変更
- タスク基準 → **機能単位**に変更
- 「どこでどういう価値を出すか」で書く
- RoboSyncの機能ブロック（VLA Policy・DataOps・安全制御など）単位でα/β/GAをマトリックスで整理する方向

#### 3. 体制: 時間軸で分ける
- Stream A/Bを「パラで走らせる」意味はほとんどない（奥村さん）
- **上期（〜9月）**: 全員でβ版リリースに全力投球
  - 小栗・Zhu・黒川もQ2末までβ版リリースチームに入る
  - MS連携もRoboSync統合の軸として一本化
- **下期（9月〜）**: スキル拡張・エージェント（村上・加賀屋が先行）
  - 大坪さん: 「エージェント・新機能を0.5人くらいが見ておく必要がある」
- 体制図は「パラ2ストリーム」→「時間軸で分ける」形式に更新が必要

#### 4. MSとの取り組みの位置づけ
- 別扱いせず、RoboSync統合の軸として一本化
- 奥村さん: 「MSのスコープに全部突っ込んで一緒にやる方がうまくいく」
- 黄瀬: 「100%で入れる以上、他のことは事実上できない。MSを軸にせざるを得ない」
- MSは5月頃まで → 終了後に全員でRoboSyncリリースに合流

#### 5. CCドットスケール（新情報）
- POC（MSとの開発環境）と商用版（CCが作るAzure上のクラウド基盤）は別物
- MS終了後（5月以降〜9月）に商用基盤への載せ替えが必要
- 石丸さんがバイウィークリーでレビュアーとして参加（工数は期待できない）
- → スケジュールと工数への配慮が必要。アーキテクチャが乖離しないよう監視

#### 6. リソース不足
- 島本さん: 「Stream A（旧β版）だけでもリソースが全然足りない。2人で本当にできるか」
- 方針: 不足のままConfluenceの課題管理欄に書き続ける（他プロジェクトも同様に申請中）
- 根性論にしない。書き続けることで訴え続ける

#### 7. ヒューマノイド・研究テーマ
- 申請書から削除または最小化
- 部活動で探索しながら、9月以降にリリースチームのメンバーがシフトしてくる後半でやる
- テーマが決まったら新規プロジェクト起案も可（年間で起案が絶対ではない）

#### 8. 図の作成リクエスト（大坪さん）
- 「早めに図を書いてください。作るものの具体感が伝わるもの」
- → 3/10提出に向けて優先的に作成する
