# FY26 RFA ターゲットタスク選定戦略

*作成日: 2026-02-09*
*ステータス: DRAFT — チーム合意形成用*

---

## 1. 本ドキュメントの目的

FY26のRFA実証実験に向けて、**どのタスクを、どの戦略軸で、どの順序で進めるか**をチームで合意形成するための検討資料。

以下を整理する:
- タスク候補の一覧と評価
- 2軸戦略（短期/中長期）の定義と各候補の位置づけ
- ロボットハードウェア・QMS段階計画との関係
- 意思決定ポイントと次のアクション

---

## 2. FY26 全体フレーム

### 2.1 スケジュール

| 期間 | フェーズ | 目標 |
|------|---------|------|
| **Q1-Q2**（4〜9月） | 実証準備 | α版リリース（RoboSync上でVLAスキル動作、成功率70%） |
| **Q3-Q4**（10〜3月） | 実証運用 | β版リリース（現場PoC完了、継続学習稼働、成功率90%） |

### 2.2 前提条件

- **RoboSyncの上にVLA機能が載った状態**で実証を行う（α版: Q2末/9月）
- 実証現場は**社内**（神戸工場 / 彩都倉庫）が基本
- FY27の事業化判断に必要な**定量的エビデンス**を揃えることがゴール

### 2.3 判断軸

| 優先度 | 判断軸 | 説明 |
|--------|--------|------|
| **1** | **RoboSync事業への接続** | VLAスキルをRoboSync上のプロダクトとしてデプロイできるか。α版QMS定義を通す最初の実績になるか |
| **2** | **VLAの事業価値** | VLAでなければ解決できない課題に対して、事業上の効果を出せるか。事業戦略上の説得力に直結する |
| **3** | **エージェント自律性の要求度** | タスクプランニング、スキル切替、失敗時のリカバリー判断など、エージェンティックな自律判断が求められるか。RFAの中長期的な技術価値を規定する軸 |
| (参考) | 現場フィールドの確保しやすさ | 候補を絞る際のフィルタとして重要だが、優先度そのものを決定する要素ではない |

> **注:** 「VLAの事業価値」は論文や対外発表の意味ではなく、**VLAの効果を事業上で実証する**ことを指す。これがステークホルダーへの事業戦略上の説得力となる。
>
> **注:** 「エージェント自律性」は、短期的な優先度は高くないが、**将来的なRFAの価値を定める軸**として考慮する。単一スキルの実行で済むタスクよりも、複数スキルの計画・切替・リカバリーが求められるタスクの方が、RFAアーキテクチャ（Task Planning Hub + Symbol Hub + Sensorimotor Hub）の存在意義を証明できる。

## 3. 2軸戦略

### 3.1 戦略コンセプト

今回の戦略の鍵は、**短期軸と中長期軸を別々のタスクとして切り離すのではなく、同一ターゲットの難易度グラデーションとして設計する**こと。

これにより、短期の投資が中長期の成果に直接累積し、すべてのリソースが一つの方向に集約される。

```
短期軸                              中長期軸
（RoboSyncデプロイ実績）             （VLAの事業価値証明）
    │                                   │
    ▼                                   ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ 限定的な    │→│ 対象を拡張  │→│ リカバリー  │→│ エージェン  │
│ ピック操作  │  │             │  │ 状況適応    │  │ ティック    │
│             │  │             │  │             │  │ 自律動作    │
│ 難易度: 低  │  │ 難易度: 中  │  │ 難易度: 高  │  │ 難易度: 最高│
└─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘
   Q1-Q2            Q2-Q3            Q3-Q4            FY27〜
```

### 3.2 短期軸（最短コース）— Q1-Q2で準備、Q3で初期実証

**目的:** RoboSync上にVLAスキルを載せてデプロイする実績を作る。

**方針:**
- 既に実現可能で効果が分かりやすい、限定的なタスクスコープから着手
- VLA必須ではない部分もあるが、**RoboSync上でVLAスキルブロックが動く**ことの実証が主眼
- α版QMS定義を通す最初のユースケースとなる
- 中長期軸への直接のステップになるため、投資が無駄にならない

### 3.3 中長期軸（第2ステップ）— Q3-Q4〜FY27で本格検証

**目的:** VLAならではの事業価値を証明する。

**方針:**
- 今は人間にしかできていない属人的な作業を対象に拡張
- VLAの「意味理解」＋「状況適応」がなければ解決できないタスク領域
- 複数スキルの計画・切替・失敗リカバリーなど、エージェンティックな自律判断が求められる
- RFAの3層アーキテクチャ全体が機能して初めて実現できる領域

### 3.4 （参考）長期軸 — FY27以降

ヒューマノイド・移動ロボットなど、テレオペレーション前提のユースケース。シンセティックデータや人間データの活用を含む。今回の2軸戦略の対象外とするが、軸としては残しておく。

## 4. タスク候補一覧

### 4.1 候補の全体像

現時点で挙がっている候補を、VLA難易度と事業価値の2軸でマッピングする。

```
事業価値
  高 │ ③エアコン解体/産廃              ①彩都ピッキング
     │                                ④段ボール組立
     │                                  
     │ ②-b ケーブル挿入              
  中 │                                ⑤ピースピッキング
     │ ②-a コネクタ挿入             
     │                 
  低 │
     └──────────────────────────────────────→
      低              VLA難易度             高
                ← 短期軸                中長期軸 →
```

各候補がこの2軸上のどこに位置するかを踏まえた上で、§6の戦略案でどう組み合わせるかを検討する。

なお、①彩都ピッキングは難易度の段階設計が可能であり、スコープの絞り方によって短期軸から中長期軸までカバーできる特徴がある。

| サブフェーズ | 内容 | 難易度 | VLA必要性 | エージェント自律性 | 双腕 |
|-------------|------|--------|----------|------------------|------|
| **①-a** | 多様な形状パーツのビン内ピック（把持容易な形状から開始） | 低〜中 | ○〜◎ | △ | 不要 |
| **①-b** | 梱包材からの取り出し・多品種対応 | 中 | ◎ | △〜○ | 不要 |
| **①-c** | 上積み回避・ピック失敗リカバリー | 高 | ◎ | ○ | 一部 |
| **①-d** | フルエージェンティック（個装・複数スキル切替・コンテナ操作） | 最高 | ◎ | ◎ | ✓ |

> **⚠️ 2026-02-16 動画分析による重要な更新:**
> 彩都パーツセンターの出庫作業を動画分析した結果、当初想定していた「箱物のみのピック」はほぼ存在しないことが判明。
> ビン内は段ボール+プチプチ+多種多様な形状のパーツで構成され、吸着のみ（現RoboSyncのルールベース）では対応不可能。
> **VLAによる汎用的な操作制御ポリシーが最初のステップから不可欠**であり、これは「把持点推定」ではなく「制御そのもの」の問題。
> オペレーションが多様・複雑で標準化不可能 → 汎用化が必須 → VLAの本質的な価値が発揮される領域。
> 詳細分析: `assets/saito-autostore-operation-analysis.txt`

## 6. 戦略案

§4-5の候補と評価を踏まえ、以下の戦略構造を提案する。

### 6.0 提案する戦略構造: メインライン + サイドライン

```
                    RoboSync事業接続（短期）──────────→ VLA事業価値（中長期）
                         │                                    │
 ┌───────────────────────┴────────────────────────────────────┴──────────────┐
 │                                                                           │
 │  【メインライン】 彩都ピッキングの段階設計                                  │
 │                                                                           │
 │   ①-a 箱物のみ ──→ ①-b 多品種 ──→ ①-c 回避+リカバリー ──→ ①-d フル    │
 │   [単一スキル]      [VLA意味理解]   [状況適応]              [エージェント]  │
 │   Q1-Q2 α版        Q2-Q3          Q3-Q4 β版              FY27〜         │
 │                                                                           │
 └───────────────────────────────────────────────────────────────────────────┘
         │                    │                    │
         ▼                    ▼                    ▼
 ┌───────────────┐  ┌────────────────┐  ┌─────────────────────┐
 │【短期横展開】  │  │【双腕ステップ】│  │【中長期探索】        │
 │               │  │               │  │                     │
 │ ②軌道制御系   │  │ ④段ボール組立  │  │ ③エアコン解体/産廃   │
 │  コネクタ挿入  │  │  双腕協調の    │  │  Q1で実現可能性調査  │

 │               │  │               │  │                     │
 │ ⑤ピースPick   │  │               │  │                     │
 │  トレイメイク  │  │               │  │                     │
 └───────────────┘  └────────────────┘  └─────────────────────┘

 メインラインで確立した     ①-dに向けた         別ドメインへの
 RoboSync×VLAを横展開      双腕技術の準備        事業拡張候補
```

**考え方:** 彩都ピッキングを難易度のグラデーションで分解し、短期〜中長期を一本のメインラインで貫く。他の候補タスクはメインラインとの関係性に応じてサイドラインとして位置づける。これにより、すべての投資がメインラインに累積する構造にする。

### 6.1 メインライン: 彩都ピッキングの段階設計

**短期軸と中長期軸を同一ライン上に配置**することで、すべての投資が累積的に価値を生む構造にする。

| フェーズ | 期間 | 内容 | 主な成果 |
|---------|------|------|---------|
| **①-a 短期** | Q1-Q2 | 多様形状パーツのビン内ピック（把持容易パーツから） on RoboSync | α版デプロイ実績、QMS定義、VLA制御ポリシーの現場実証 |
| **①-b** | Q2-Q3 | 梱包材からの取り出し・多品種対応 | VLAの汎用操作制御による標準化不可能な作業の自動化実証 |
| **①-c** | Q3-Q4 | 上積み回避+ピック失敗リカバリー | VLAエージェンティック動作の初期検証 |
| **①-d 中長期** | FY27〜 | フルエージェンティック（個装・複数スキル切替・コンテナ操作）、双腕 | VLA事業価値の本格証明 |

**ストーリー:**
> 短期: 「彩都パーツセンターの出庫作業を分析した結果、12万品番の多様な形状・包装状態のパーツが混在し、吸着のみのルールベース制御では対応不可能であることが判明した。RoboSync上にVLAの汎用操作制御ポリシーを実装し、ルールベースでは掴めなかったパーツのピッキングを実現した」
> 中長期: 「VLAの状況適応能力により、梱包材からの取り出し、バラ積み商品の認識・選別、個装作業まで含むエンドツーエンドの出庫作業を自動化。作業パターンの事前定義なしに多様なオペレーションに汎用的に対応する能力を実証した」

## 7. ロボットハードウェアとの関係

### 7.1 2つの軸

| 軸 | ハードウェア | 特徴 | 対応 |
|----|------------|------|------|
| **産業用ロボット** | UR5等 | 現場耐久性、RoboSync親和性、QMS対応が容易 | メインライン①-a〜①-c、サイドライン②⑤ |
| **研究・実験用ロボット** | Aloha Stationary, Doraemon AI, SO1 | 安価、柔軟、SDR文脈で先行検証 | 双腕ステップ④、メインライン①-dのラボ検証 |

### 7.2 SDR（Software Defined Robot）の推進

短期軸では産業用ロボット（UR5等）が主流だが、以下の方向も並行で攻める:
- (a) SDR文脈を進める — ソフトウェアでロボットの機能を定義・更新
- (b) 身体自体を自ら拡張して現場にフィットさせる
- (c) Doraemon AI等の最新研究レベルのハードウェアを活用

ヒューマノイド・移動ロボットは今回の2軸の対象外（長期軸）。

### 7.3 タスク × ハードウェアの対応

| タスク | 単腕/双腕 | 短期ハード | 中長期ハード |
|--------|----------|-----------|------------|
| ①-a 多様形状パーツピック | 単腕 | UR5 + エンドエフェクタ（吸着/平行グリッパ選択）（RoboSync既存、奥村さんテレオペ実現済み） | - |
| ①-b〜c 梱包材取出+回避 | 単腕→一部双腕 | UR5 | 産業用双腕（要選定） |
| ①-d フルエージェンティック | **双腕** | Aloha / 研究用 | 産業用双腕（要選定） |
| ② 軌道制御系 | 単腕 | UR5 | - |
| ③ エアコン解体 | 要検討 | - | 要調査（力制御対応ハード） |
| ④ 段ボール組立 | **双腕** | Aloha / 研究用 | 産業用双腕（要選定） |

## 8. QMS段階計画との関係

### 8.1 α版 / β版 / リリースの定義

| 項目 | α版（Q2末/9月） | β版（Q4末/3月） | リリース（FY27〜） |
|------|-----------------|-----------------|-------------------|
| 状態 | 社内実証に投入 | 運用安定、継続学習稼働 | 商用提供可能 |
| 成功率 | 70% | 90% | SLAとして定義 |
| 安全性 | RoboSync標準安全制約、人介在リカバリー | 現場固有安全パターン対応、異常時自動停止 | 安全要件文書化・監査対応 |
| DataOps | データ収集パイプライン稼働 | 継続学習サイクルが安定稼働 | スケーラブルな学習基盤 |
| コンプライアンス | AI倫理・OSSライセンス対応（最低限） | リスク整理・対応拡充 | 全項目対応 |
| 提供形態 | 社内実証（無償） | 社内実証（無償） | RoboSyncオプション（有償） |

### 8.2 メインラインとQMSの対応

| QMS段階 | メインラインフェーズ | QMS要件 |
|---------|-------------------|---------|
| **α版（CA）** | ①-a 多様形状パーツのピック → 作業台に配置まで | VLAスキルの入出力IF・失敗判定・停止条件の文書化。RoboSync標準安全制約に準拠。個装は人が担当（人×ロボット協働） |
| **β版** | ①-b〜c 梱包材取出+個装含む | 現場固有安全パターン、異常時自動停止、人介在データ記録フロー。個装スキルの追加 |
| **リリース** | ①-d フルバージョン（移動含む） | 安全要件文書化・監査対応、品質保証プロセス確立 |
