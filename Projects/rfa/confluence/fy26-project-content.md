## 1. プロジェクト内容

### プロジェクトビジョン

**人間の言語指示を理解し、物流・製造現場における属人的かつ複雑で柔軟な推論が求められるタスクをロボットが自律的に実行できる知能システムを実現する。**

### 解決する課題

| 課題 | 内容 |
|------|------|
| **抽象的言語指示の自律的解釈** | 「すきまなく詰めて」等の曖昧指示から暗黙的意図（重いものは底、壊れ物は上等）を推論し具体的作業手順に自動分解 |
| **専門知識不要な作業指示** | 従来の1つ1つ動作教示から、自然言語指示のみでロボット作業実行を実現し、SIer依存を解消 |
| **未知・複雑形状物体の汎用操作** | 事前登録なしでも視覚認識から適応的把持・操作を実現し、従来ルールベース制御の限界を突破 |
| **動的環境での自律的問題解決** | 把持失敗・物体落下等の予期せぬ状況における自律的リカバリーと計画修正による作業継続 |

### 事業価値

| 観点 | 内容 |
|------|------|
| **SCM領域の労働課題解決** | 人手不足が深刻な物流・倉庫現場において、入出庫・梱包等の属人作業を自動化 |
| **競争優位性確立** | Deploy-Time Scaling戦略により、現場データを蓄積し他社追随困難な技術的優位性を構築 |
| **横展開効果** | Robo Syncをベースのコア技術として本技術を提供しスケール化 |

### 技術アーキテクチャ

RFAは以下の4つの技術コンポーネントを統合したシステムとして設計：

| コンポーネント | 役割 |
|----------------|------|
| **LLM Agent** | 抽象的な言語指示からタスク構造を生成し、状況に応じてRe-Planning・リカバリーを行う「計画・対話の頭脳」 |
| **VLA（Vision-Language-Action）** | カメラ画像・状態量・タスク表現から連続的なロボット動作を生成する汎用操作ポリシー |
| **Symbol Integration** | 言語世界と物理世界の間をつなぐ変換レイヤー（Keypoint、座標等） |
| **Robo Sync + DataOps** | 物理ロボット・周辺機器とのインターフェースと、データ収集・学習・評価を回すための基盤 |

> 📄 詳細：[FY26 RFA 事業計画書](https://panasonic-connect.atlassian.net/wiki/spaces/CT1/pages/2484830627)（§3. プロジェクト概要）

### FY26の位置づけ

FY26は **Phase 2: Robo Sync現場PoCと事業検証** として位置づけ。研究フェーズから現場PoC／事業検証フェーズへの転換年度。

| Phase | 年度 | 内容 |
|-------|------|------|
| Phase 0 | FY24 | 基盤立ち上げ（π0調査、要素技術の種まき） |
| Phase 1 | FY25 | RFAスタック確立、CXCデモ統合 |
| **Phase 2** | **FY26** | **Robo Sync現場PoC、事業検証** |
| Phase 3 | FY27 | 社会実装、運用スキーム確立 |

---

### FY26実行戦略：4つの柱

FY26は以下の4つの柱に集中投資：

| 柱 | 目的 | 主担当 |
|----|------|--------|
| **① Robo Sync × VLA統合** | VLAをRobo Sync上の正式なスキルブロックへ昇格させ、実行基盤を確立 | 奥村、Zhu、黄瀬 |
| **② DataOps** | 現場データを学習資産として蓄積し、「使うほど賢くなる」基盤を構築 | 小栗、CEC/Microsoft |
| **③ Agent自律適応** | 失敗・例外を前提とし、Agentが再計画・適応実行できる運用構造を構築 | 加賀屋、村上、黒川 |
| **④ 現場実証・事業接続** | 神戸/彩都でのPoC実施、事業仮説検証、FY27判断材料の整備 | 黄瀬、RFAチーム全体 |
