## 2. 池内先生との連携の経緯

### 2.1 RFAの前提：Agent + VLA の2層構造

RFAプロジェクトでは、**LLM Agent（計画・判断）+ VLA（実行）の2層構造**で現場タスクを自動化しようとしています。

- **LLM Agent**: 言語指示を理解し、タスクを計画・分解する
- **VLA**: 計画されたスキルを、画像を見ながら実際に実行する

この2層構造が前提にある中で、「VLAをどの単位で学習・呼び出すか」という問題が浮上しました。

### 2.2 出発点：VLAを「どう分割するか」という問題

VLA（Vision-Language-Action Model）を実際に現場で使おうとすると、根本的な問題にぶつかりました。

> **「VLAをどの粒度で学習・呼び出すべきか？」**

例えば、梱包タスクを考えたとき：
- 「梱包全体」を1つのVLAで学習する？ → 複雑すぎて学習が困難
- 「Pick」「Place」という動作ごとに学習する？ → 粒度が細かすぎて組み合わせが爆発

最初は「Pick」「Place」「Insert」といった動詞でスキルを分類しようとしました。しかし、同じ「Pick」でも実際には全く違う動作が必要です（箱物を上から掴む / 袋物を横から掴む / ケーブル付きを慎重に持ち上げる）。

**動詞は人間にとってわかりやすいですが、ロボットにとっては曖昧すぎる**のです。ここに、LLMが扱う「意味的スキル」と、ロボットが実行する「物理的スキル」の間にギャップがありました。

**【図①挿入箇所】**: 粒度の問題を示す図
- 左：粒度が大きすぎる（梱包全体）→ 学習困難
- 右：粒度が小さすぎる（Pick/Place）→ 組み合わせ爆発
- 中央：適切な粒度 → GMR

### 2.3 池内先生との連携

この問題を解決するために、池内克史先生との連携を開始しました。

池内先生は、**Learning-from-Observation（観察からの学習）** を30年以上研究されてきた方です。特に、以下の研究が私たちの課題に直結しています。

**主要な研究**：
- **接触状態遷移（Contact State Graph）**: ロボット操作を物理的に厳密に記述する理論
- **PTG/STG（Physical/Semantic Task Group）**: タスクを物理制約と意味制約で分類する体系
- **GMR（Grasp-Manipulation-Release）**: 操作を把持→操作→解放の3フェーズで分解する枠組み

これらの理論が、私たちの「VLAをどう分割するか」という問題に、明確な答えを与えてくれました。

## 3. 得られた知見①：VLAの学習単位（GMR）

### 3.1 核心的なアイデア：スキル = 接触状態の遷移

池内先生の考え方の核心は、こうです。

> **スキルを「動詞」ではなく「接触状態の遷移」で定義する**

ロボット操作とは、結局のところ「物体と環境の接触状態を変化させること」です。

| 接触状態 | 意味 | 例 |
|---------|------|-----|
| NC（非接触） | 触れていない | 空中移動中 |
| PC（面接触） | 面で接触 | テーブル上に置いた状態 |
| FT（完全固定） | 動かない | ビス締結後 |

この考え方だと、スキルは以下のように定義できます：
- **Pick** = NC（非接触）→ 把持状態
- **Place** = 把持状態 → PC（面接触）
- **Insert** = PC → FT（完全固定）

**これなら物理的に一貫性があり、曖昧さがない**。LLMとVLAの間の「翻訳」が明確になります。

池内先生はさらに、タスクを2種類に分類しています：
- **PTG（Physical Task Group）**: 純粋に物理制約だけで決まるタスク（例：ビス締結）
- **STG（Semantic Task Group）**: 意味的な制約を含むタスク（例：「壊れ物を守る」「水をこぼさない」）

梱包タスクの多くは**STG**に属します。物理的には単純でも、意味的な判断が必要なタスクです。

### 3.2 GMR（Grasp-Manipulation-Release）とは

接触状態の考え方を踏まえて、池内先生が提唱しているのが **GMR** という概念です（論文: arXiv:2008.01513, 2020）。

**GMR = 把持（Grasp）→ 操作（Manipulation）→ 解放（Release）**

| フェーズ | 内容 | 例 |
|---------|------|-----|
| **Grasp（把持）** | 対象物を安定して掴む | コップの側面を掴む |
| **Manipulation（操作）** | 掴んだまま動かす | コップを移動、フタを回す |
| **Release（解放）** | 安全に手を離す | 棚の上に置いて離す |

日常・物流・製造の作業の大半は、この「**掴んで → 何かして → 離す**」というパターンに分解できます。

**【図②挿入箇所】**: GMRの3フェーズを示す図
- Grasp → Manipulation → Release の流れ
- 各フェーズでの接触状態の変化を図示

### 3.3 GMRが「ちょうどいい粒度」である理由

GMRは、VLAの学習単位として「ちょうどいい粒度」です。

```
高レベル:  「箱を棚に収納する」（言語指示）  ← LLMが扱う
           ↓
中レベル:  GMR（把持→操作→解放）           ← VLAの学習単位
           ↓
低レベル:  joint / torque / waypoint        ← ロボット制御
```

- **大きすぎない**: 1つのGMRは数秒〜数十秒で完了する操作
- **小さすぎない**: 意味のあるまとまり（1つの目的を達成する）
- **再利用可能**: 同じGMRを異なるタスクで使い回せる

## 4. 得られた知見②：再利用可能なスキル体系

### 4.1 物理特性でスキルを分類する

池内先生からの重要な指摘があります。

> 「物体ごとにスキルを分けると扱う物が増えた時に問題。『Xを箱に入れる』という汎用スキル化が必要」

具体的に、梱包タスクをGMRで分割するとこうなります：

| GMR | 内容 | 特徴 |
|-----|------|------|
| 商品ピッキング | コンテナから商品を取り出す | 片手、不定形対応 |
| 箱詰め（片手） | 小物を箱に入れる | 片手 |
| 箱詰め（両手） | 大物を箱に入れる | 両手協調 |
| 箱閉め | フラップを折り畳む | 両手、シーケンス |
| テープ貼り | 封緘テープを貼る | 軌道追従 |

**重要なポイント**：
- 「シェーバーを入れる」と「チャージャーを入れる」は **同じGMR**（物理特性が同じ）
- 「片手で入れる」と「両手で入れる」は **別のGMR**（物理特性が異なる）

つまり、**物体ごとではなく、物理特性でスキルを分類**します。これにより、再利用性が大幅に向上します。

**【図③挿入箇所】**: 梱包タスクのGMR分割図
- 梱包フロー全体を示し、どこがどのGMRに対応するかを図示

### 4.2 iREXデモからの実証データ

この考え方が実際に機能するかを、iREX（国際ロボット展）のデモで検証しました。

**デモ内容**: シェーバー梱包タスク（4スキル連続実行）

| 項目 | 数値 |
|------|------|
| 1スキル（GMR）あたりのデータ | 約200エピソード |
| データ収集時間 | 約90分/スキル |
| 学習時間（Pi0ファインチューニング） | 約1日/スキル |

**つまり、新しいGMRを追加するコストが明確になりました**。事業計画に織り込める数字です。

### 4.3 タスク分類の二軸フレームワーク

スキルの再利用を体系的に行うため、以下の二軸で整理しています。

| 軸 | 説明 | 例 |
|----|------|-----|
| **タスクグループ**（業務フロー） | 業務単位の分解 | 梱包フロー、組立フロー |
| **GMRカテゴリ**（物理パターン） | 物理的操作の分類 | ピッキング系、配置系、貼付系 |

この二軸で整理することで：
- **横展開が容易に**: 「梱包のピッキング」で学んだスキルを「製造のキッティング」に転用
- **必要なスキルの見積もりが可能に**: 新タスクに必要なGMRを事前に特定

## 5. 得られた知見③：継続学習サイクル

### 5.1 エラーリカバリーの重要性

池内先生から、もう一つ重要な指摘をいただきました。

> 「本来、アクションというのは何らかのアクションがあって、なおかつそのアクションが終わりましたという判定まで含めたものがアクション」
> 「すべてのアクションにおいて、エラーリカバリーが必要」

### 5.2 完璧を目指さない

この指摘を踏まえて、現場運用について重要な合意がありました。

> 「100%って絶対難しいから。90%やっといて、10%があったらなんか呼ぶっていうのが一番いい」

最初から完璧な自動化を目指すのではなく、**エラー時は人間を呼ぶ運用が現実的**です。そして重要なのは、**人間が介入したデータを学習に活用する**ことです。

### 5.3 継続学習サイクルの設計

この考え方に基づいて、以下のような継続学習サイクルを設計しています。

**【図④挿入箇所】**: 継続学習サイクルの全体図

```
    ┌──────────────────────────────────────────────────┐
    │                                                  │
    ▼                                                  │
 現場で実行                                             │
    │                                                  │
    ▼                                                  │
 VLMが状態監視 ───→ 成功 ───→ 成功データとして蓄積 ────┤
    │                                                  │
    │ 失敗検知                                          │
    ▼                                                  │
 自律リカバリー試行                                      │
    │                                                  │
    │ リカバリー不可                                    │
    ▼                                                  │
 人間が介入 ───→ 介入データとして蓄積 ─────────────────┤
                                                       │
                   ┌───────────────────────────────────┘
                   │
                   ▼
              データ蓄積
                   │
                   ▼
             学習 / Fine-tune
                   │
                   ▼
              再デプロイ ───→ 現場で実行（改善版）
```

**ポイント**：
- 失敗も価値あるデータになる
- 人間介入が多いスキルから優先的に改善
- **「使いながら賢くなる」** 仕組みが差別化の核心

### 5.4 事業戦略との結びつき：段階的自律化

この知見は、RFAの事業計画における**段階的自律化戦略**と直接結びついています。

**Phase 1（FY26-27）**: Robo Sync主力 + VLAリカバリー + 人介在
- 最初から完全自律を目指さない（池内先生の「90% + 10%」）
- Robo Syncの安定動作を基盤に、VLAと人介在でリカバリー
- **この運用全体を通じてデータが蓄積される**

**Phase 2以降**: データ蓄積 → VLA能力向上 → 自律度向上
- 失敗・リカバリーデータが最も価値の高い学習データ
- 人介在は「コスト」ではなく「最高価値データ生成への投資」

**データフライホイール**：
> 「事業価値を提供しながら、同時にデータを蓄積し、自律度を高めていく」

この構造により、BigTechとは異なるアプローチで競争優位を確立できます。BigTechは大規模テレオペ施設で「力技」のデータ収集。RFAは**現場運用の中で自然にデータが蓄積される構造**です。

## 6. 事業的意義

### 6.1 スキル再利用による展開コスト削減

GMRベースでスキルを設計することの最大のメリットは、**新しいタスクへの展開コストが下がる**ことです。

**従来**（スキル再利用なし）：
- 新タスクごとに一からデータ収集・学習
- タスク数に比例してコストが増大

**GMRベース**（スキル再利用あり）：
- 既存GMRの組み合わせで多くのタスクをカバー
- 新GMRが必要な場合のみ追加学習
- **タスク数が増えてもコストの増加が緩やかになる**

**【図⑤挿入箇所】**: 展開コストの比較グラフ
- X軸: タスク数
- Y軸: 累積コスト
- 2本の曲線: 「従来（線形増加）」vs「GMRベース（漸近的に収束）」

### 6.2 理論的基盤による説明可能性

池内先生のフレームワーク（接触状態遷移、GMR）を採用することで、**技術に理論的な裏付けが生まれます**。

| 効果 | 内容 |
|------|------|
| **顧客への説明** | 「なぜこのスキル分割なのか」を物理的根拠で説明できる |
| **技術者の共通言語** | チーム内・パートナーとの議論が効率化 |
| **デバッグの容易化** | 問題が「どのGMRの、どの接触遷移で起きたか」を特定しやすい |

さらに、技術的な発展にも直結します：

| 発展方向 | 内容 |
|---------|------|
| **Agentの再計画** | 接触状態遷移で前提条件/事後条件が明確 → 失敗時の論理的な再計画が可能 |
| **RL報酬関数の設計** | 接触状態の変化を報酬に変換 → 模倣学習の「ラストワンマイル」をRLで補完 |

特に後者は重要です。模倣学習だけでは「最後の押し込み」「微細な力加減」が難しい（ラストワンマイル課題）。接触状態遷移を報酬関数に使うことで、**VLA（模倣）で大まかな軌道を学習し、RL（強化学習）で最後の詰めを学習**するハイブリッド構造が可能になります。

### 6.3 研究と事業の両立

現在、IndexVLA（池内先生の理論をVLAに適用した手法）の論文をRA-L（トップジャーナル）に投稿予定です。

これは「単にデモを作る」ではなく、**学術的にも価値のある技術を事業に活用している**ということです。技術的な強みとしてのブランディング、採用への寄与も期待できます。

## 7. VLAが価値を出す領域

### 7.1 少量多品種・動的変化が主戦場

池内先生との議論を通じて、VLAが本当に価値を出せる領域が明確になりました。

| 領域 | VLAの適性 | 理由 |
|------|----------|------|
| 少量多品種 | ◎ | SKU変更のたびに再教示不要 |
| 動的状況変化 | ◎ | 毎回違う状態に対応可能 |
| 属人的判断 | ◎ | 暗黙知をデータから学習 |
| 一品種大量・固定タスク | △ | 従来技術でも対応可能 |

**重要な示唆**：一品種大量生産の固定タスクは従来技術でも自動化できます。VLAの真価は、**「毎回状況が変わるタスク」** で発揮されます。

### 7.2 梱包タスクが最適な理由

池内先生のフレームワークで梱包タスクを分析した結果、興味深いことがわかりました。

> **梱包は「物理的には単純」だが「意味的には複雑」**

多くの工程が**STG（Semantic Task Group）**に属します。これは、物理的な制約だけでなく、「壊れ物を守る」「水平を保つ」といった意味的な制約を含むタスクです。

こうしたタスクは、ルールベースで書き下すのが難しく、**VLAがデータから学習することで価値を出しやすい**領域です。

## 9. まとめ

池内先生との連携を通じて、RFAプロジェクトの技術的方針が明確になりました。

### 得られた3つの知見

1. **VLAの学習単位はGMR（Grasp-Manipulation-Release）**

   - 人間介入データを学習に活用

### 事業的意義

- **コスト削減**: 新タスク追加のコスト・時間が大幅に下がる
- **差別化**: 「使いながら賢くなる」仕組みが競争優位に
- **技術的強み**: 学術的価値を持つ技術を事業に活用
