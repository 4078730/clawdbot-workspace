## 5. 🟢タスクの詳細

α版で確実に対応するタスク。共通点は**1回のピック＆プレースで完結**し、開封・選別・道具使用が不要であること。

### P-1: 箱物ピック

> 池内先生（2/18）: 「パラレルグリッパでガシャっと押し込んだら持てる」

| 項目 | 内容 |
|------|------|
| 動作 | ビン内の箱を認識 → グリッパで挟む → 持ち上げ → 作業台に置く |
| 物体形状 | 定型（直方体）。サイズは一定 |
| 把持点 | 自明（箱の側面） |
| 制約 | 箱幅 ≤ 85mm（2F-85ストローク）。超える場合は吸着に切替 |
| 難しいところ | ほぼない |

### P-2: 袋物そのままピック

| 項目 | 内容 |
|------|------|
| 動作 | ビン内の袋を認識 → 端/面を掴む → 持ち上げ → 作業台に置く |
| 物体形状 | **不定形。** 毎回折れ方・向きが異なる |
| 把持点 | **毎回異なる。** 画像から都度判断が必要 |
| 制約 | 持ち上げ時に中身が偏って振れる → 速度制御 |
| 難しいところ | 把持点の決定。ルールでは定義しきれない → **VLAが活きるタスク** |

**P-1 と P-2 の違い:**

| | P-1 箱物 | P-2 袋物 |
|---|---------|---------|
| 形状 | 定型 | 不定形 |
| 把持点 | 事前定義可能 | 毎回推論が必要 |
| 制御方式 | ルールベース主体 | VLA主体 |
| VLAの価値 | 低い（認識のみ） | **高い（α版での価値証明の中核）** |

## 6. 🟡タスクの条件整理

α版スコープへの拡張判断に必要な検証事項。

| タスク | 何がクリアされればα版に入るか | 検証方法 |
|--------|---------------------------|---------|
| P-3 仕切り内 | グリッパが区画内寸(~300×200mm)に物理的に入ること | UR5e + 2F-85 実機でビン挿入テスト |
| P-7 密集 | 隣接パーツを崩さずに引き出せること | VLAのアプローチ角度精度を実機評価 |
| P-8 立てかかり | 横方向からのアプローチで安定把持できること | 同上 |
| T-1/T-2 ワゴン | UR5eリーチがワゴン位置まで届くこと | 作業空間レイアウト図面で確認 |

## 7. 🔴タスクの構造分析

β版以降で対応するタスク群。難しさの本質は3パターンに集約される。

```
🔴 難しいタスク（8個）
    │
    ├── 両手が必要（4個）──→ β版: 双腕導入
    │   P-4 袋in袋、P-5 段ボール開封
    │   D-1 プチプチ、D-2 小袋入れ
    │
    ├── 計画・記憶が必要（2個）──→ β版: Agentic制御
    │   P-9 元位置戻し、P-10 下層探索
    │
    └── システム連携が必要（2個）──→ β版: RoboSync統合
        D-5 バーコード照合、(S-2, S-3)
```

特に **P-4（袋の中から袋）は出庫作業で最も頻度が高い**パターンであり、β版で双腕を導入した際に最初に取り組むべき対象になる。

## 9. 未確認事項と議論論点

### 未確認事項

| # | 項目 | 確認先 | 影響 |
|---|------|--------|------|
| 1 | 🟢タスクが出庫全体の何割か | 統計データ（現場） | α版のインパクト規模 |
| 2 | 商品化・梱包工程の動画 | 風見さん | 別ルート（レーン非占有）の評価 |
| 3 | プロパティ登録の実装可否 | 中田さん | ロボット向けパーツの選別方法 |
| 4 | UR5eとビンの物理干渉 | 実機検証 | 🟡タスクのα版スコープ判断 |

### 議論したい論点

1. **α版のカバー率:** 🟢タスクの出庫比率が限定的な場合、α版の価値をどう示すか
2. **最頻出パターンのギャップ:** P-4（袋in袋）がα版外。プロパティ登録でロボット向けパーツだけを流す戦略の現場受容性
3. **別エリアの可能性:** 商品化・梱包はレーンを占有しない。ピッキングの代わりにそちらをα版対象とする選択肢
