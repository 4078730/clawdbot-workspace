### S1-1. ビジョンと解決する課題（7分）

**スライド案:** 1枚=ビジョンステートメント、2枚=解決する課題4つ

**台本（日本語）:**

RFA — Robotics Foundation Agent は、人間の言語指示を理解して、物流・製造現場の複雑な物理タスクをロボットが自律実行する知能システムです。

解決する課題は4つです。

1. **抽象的言語指示の自律的解釈** — 「すきまなく詰めて」のような曖昧な指示から、暗黙的意図（重いものは底、壊れ物は上）を推論し、具体的作業手順に自動分解する

2. **専門知識不要な作業指示** — 従来の1つ1つの動作教示から、現場作業者の自然言語指示のみでロボット作業を実行し、SIer依存を解消する

3. **未知・複雑形状物体の汎用操作** — 事前登録なしでも視覚認識から適応的把持・操作を実現し、従来ルールベース制御の限界を突破する

4. **動的環境での自律的問題解決** — 把持失敗・物体落下などの予期せぬ状況で、自律的にリカバリーし作業を継続する

事業価値としては、SCM領域の労働力不足が深刻な物流・倉庫現場で入出庫・梱包等の属人作業を自動化し、ロボット制御PFをベースにこの技術をスケールしていきます。

### S1-2. 技術アーキテクチャ概要（7分）

**スライド案:** アーキテクチャ全体図（既存スライドから）+ 主要技術要素の対応表

**台本（日本語）:**

RFAの技術アーキテクチャは、大きく5つの要素で構成されています。

**① RFA PoC（リーディングタスク実証）**
SCMドメインタスクによる統合システムの実機検証とデモ開発。FY25はロボット制御PF経由のデータ収集→VLA学習→実動作のサイクルを初めて実現しました。

**LLM Agent（頭脳部分）：**

**② Planning** — 抽象指示を具体的な作業手順に自動分解。Backward Reasoningによる階層的タスク分解（EMNLP2025投稿済）。

**③ Skill Composition** — サブタスクをスキルの組み合わせで達成。メモリベースのRe-Planning、異なる環境間での知識転移（ICRA2026投稿予定）。

**Robot Foundation Model（実行部分）：**

**④ Symbol Integration** — 言語指示を画像上の具体的操作点（どこを掴む、どこに置く）に変換。KeypointVLA（RSJ2025発表済）、IndexVLA（ICRA2026投稿予定）。

**⑤ VLA-Robot** — 視覚・言語・行動を統合したロボット制御。π0ベースのFine-Tuning、iREX2025で4スキル連続実行の梱包デモを実現。

**技術的差別化ポイント：**
- 言語指示の直接実行 — 複雑な事前プログラミング不要
- 未知環境への適応 — 学習済みパターン以外でも自律的に判断・実行
- 失敗からの自動学習 — エラー時の自律的分析・改善による継続的性能向上

### S2-1. アーキテクチャ詳細と統合構造（25分）

**スライド案:** 全体構成図 → 各Hub詳細 → RoboSync統合ルート → MCP構成

**台本（日本語）:**

RFAの技術スタックを詳しく説明します。

**Task Planning Hub（LLM Agent）**

「計画・判断の頭脳」です。

- **タスク理解・分解:** 「この3種類の商品を壊れ物に気をつけて梱包して」→ 箱選定→ピッキング→箱詰め→緩衝材→封緘、というサブタスク列に分解
- **Skill Alignment:** 各ステップでVLAスキルを使うかRoboSyncの既存ブロックを使うかを選択
- **Re-Planning:** 把持失敗・対象物落下時に状況を分析し、再試行/別スキル/人へのエスカレーションを判断
- **要素技術:** DRIP（Backward Reasoningによる構造的タスク分解、EMNLP投稿済）、MTP（マルチステップ計画、ICRA投稿予定）

**Symbol Hub（Symbol Integration）**

「言語と物理世界の橋渡し」です。

- **意味状態の管理:** 何を、どこを、どういう状態か、を記号的に管理
- **言語↔物理の変換:** LLMの「重い箱」「壊れ物」をVLAが扱える座標・Keypoint・マスクに変換。逆に観測結果をLLMが理解できる状態記述に変換
- **VLMによる状態認識:** Gemini-ERを使った成功/失敗判定、次スキル決定
- **要素技術:** KeypointVLA（画像上の操作点指定でVLA精度を最大40%向上、RSJ発表済）、IndexVLA（タスク特性に応じたPoint/Trajectory/Vector表現の使い分け、池内先生の記号論ベース、ICRA投稿予定）

**Sensorimotor Hub（VLA）**

「身体操作」を担います。

- **VLA Policy:** π0をベースモデルとして、カメラ画像＋ロボット状態＋タスク指示から連続的なロボット動作を生成。学習コスト: 約200エピソード/約90分のデータ、スキルあたり約1日
- **KeypointVLA:** Keypoint中間表現による高精度VLA。コネクタ挿入・精密位置調整向け
- **HIL Interface:** 自律リカバリー不可時は人間教示を受入れ、そのデータを継続学習に利用
- **FY25実績:** FUK拠点で計3500エピソードの混合データセットでπ0を学習。リトライ動作の自律的生成を獲得。未学習物体への汎化性を実証

**RoboSync統合**

2つの統合ルートがあります。

- **ルート①（Agent経由）:** LLM AgentがMCP経由でRoboSyncに接続。自然言語→論理的計画→ロボット制御コード→IK実行の自律パイプライン。FY25でClaude Desktop＋MCPによるPoCを実証済
- **ルート②（VLAスキルブロック）:** VLAをRoboSyncの「スキルブロック」として提供し、GUIで配置・組み合わせ。**FY26最優先**

MCPを共通プロトコルとして、RoboSync、WMS/MES、将来的にGemini-ERなどと疎結合に接続します。

### S2-2. 保有技術の詳細と成果（20分）

**スライド案:** 各技術1-2枚（動画あり）

**台本（日本語）:**

FY25で確立した個別の技術成果を紹介します。

**① ロボット制御PF統合パイプライン（FUK拠点）**

ロボット制御PF経由でのデータ収集→π0学習→実動作のサイクルを初めて実現しました。

- BinPickingタスク: 1000エピソードの動作を学習し再現
- 10種類の物体を扱える汎用モデルを開発
- 未学習物体への汎化性を実証（スキンクリームで学習→コーンフレーク把持成功）
- **リトライ動作の自律的生成** — 教えていないのに自律的にリトライする挙動をVLAが獲得

[動画: π0によるBinPicking]

**② VLA-Robot（Idea Forge環境）**

Idea Forgeに双腕Aloha Stationary AI、SO-101を活用した実験環境を構築。ACT、π0、smolVLA等の複数モデル比較評価を実施。

[動画: SO-101箱詰め、Koch電池箱詰め]

**③ KeypointVLA → IndexVLA**

KeypointVLA: 画像上に「掴む点」「置く点」を(x,y)座標で指定。既存VLAモデルの構造を一切変更せず、データ前処理だけで適用可能。ACTモデルで最大40%の成功率向上（電池梱包タスク30%→70%）。

IndexVLA: タスクの物理特性に応じた最適な指示方法を体系化。
- Point型: ピック&プレース（離散的な始点・終点）
- Trajectory型: なぞり・拭き取り（連続的な経路追従）
- Vector型: 押し込み・コネクタ挿入（方向と力の指定）

池内先生（京大名誉教授）のパースの記号論をベースに設計。

[動画: KeypointVLAの効果比較]

**④ Skill Composition（MTP / ViReSkill）**

MTP: 過去の成功経験を参照し、新環境でのタスクプランニングに活用するメモリ転移手法。シミュレータの知識を現実環境に転移してRe-Planning。

ViReSkill: 失敗動作の映像分析から自律的に計画を修正し、次回以降の成功率を向上。Lifelong Learning。

**⑤ Planning（DRIP）**

人間のBackward Reasoningを応用。ゴールから逆算してサブゴールに分解。テキスト環境からMinecraftの3D仮想環境に拡張し、複雑な空間推論タスクでの有効性を検証。

**⑥ iREX2025梱包デモ（統合デモ）**

Gemini-ER × π0による多段タスクVLAデモ。各ステップの完了/失敗をVLM（Gemini-ER）で判定し、次ポリシーへの遷移またはリトライ/人間介入への分岐を実装。

このデモ構成が今後の現場PoCで活用可能な「標準パターン」のたたき台です。

[動画: iREXデモ]

### S2-3. データ戦略と継続学習サイクル（15分）

**スライド案:** データピラミッド → ライフサイクル図 → 課題

**台本（日本語）:**

RFAのデータ戦略です。

**データソースの3層構造：**

| 層 | ソース | 特徴 | 役割 |
|----|--------|------|------|
| 上位 | Human Teleoperation | 最高価値・高コスト | 初期スキル獲得、高精度FT |
| 中位 | Synthetic Data | スケール可能、Sim2Realギャップ | バリエーション拡張 |
| 下位 | Human Videos | 大量、ロボットコマンド含まず | 事前学習、意味表現の土台 |

**データライフサイクル（RoboSync経由）：**

1. Model Deploy → 2. Logging → 3. Data Export（→CECクラウド、RLDS形式） → 4. Train/Fine-tune → 5. Re-Deploy

これがDeploy-Time Scalingの核心です。BigTechのTraining-Time Scaling（大規模テレオペ施設での力技データ収集）に対して、我々は現場導入と同時にデータ蓄積が始まる構造を取ります。

**現在の課題（→ Microsoft協業ポイント）：**
- FT→再デプロイのサイクルがまだ手動 → Azure ML Pipelines
- モデルバージョン管理が未整備 → MLOps基盤
- 合成データ生成の体系化 → シミュレーション環境

### S2-4. Robo Sync AI Suiteに向けた技術チャレンジ（20分）

**スライド案:** チャレンジを「技術的チャレンジ」と「PF的チャレンジ」に分けて整理

**台本（日本語）:**

ここからが今日の議論の核心です。RoboSyncにVLAを統合して商用プラットフォーム — Robo Sync AI Suite — にしていくにあたっての技術的・プラットフォーム的なチャレンジを整理します。

**【技術的チャレンジ】**

**T1: 座標制御（RoboSync）↔ 関節制御（VLA）のブリッジ**
- RoboSyncはタスク空間座標ベースのオープンループ制御
- VLAは関節空間でのクローズドループ制御
- この2つをシームレスに切り替える仕組みが必要
- FY25 3Qから課題として認識、RFAチームでRoboSyncを触って検証中

**T2: VLAスキルブロックの安全制御**
- VLAが生成する動作にGuardrailを設ける必要がある
- 安全停止、速度・力のリミット、異常検知
- RoboSyncの既存安全制約との整合

**T3: 非定型物品の汎用操作精度**
- 12万品番の多様な形状・包装に対する成功率の向上
- エンドエフェクタ戦略（吸着/グリッパ/指）
- リトライ戦略の洗練

**T4: Agentic制御の実現**
- 複数VLAスキルの動的切替（ピック→個装→梱包→配置）
- 失敗検知→原因分析→自律リカバリー
- β版の核心チャレンジ

**【PF的チャレンジ（→ Microsoft協業ポイント）】**

**P1: DataOpsパイプラインの自動化**
- RoboSyncログ → RLDS/LeRobot形式への自動変換
- Fine-Tuning → 評価 → 再デプロイの自動化
- → **Azure ML Pipelines との統合**

**P2: モデル管理・配信基盤**
- サイト別・タスク別のモデルバージョン管理
- A/Bテスト、ロールバック機能
- → **Azure ML / MLOps基盤**

**P3: シミュレーション連携**
- Real2Sim / Sim2Realのギャップ
- 合成データによるバリエーション拡張
- → **Microsoft Simulation環境**

**P4: Responsible AI**
- VLA推論の不確実性下での安全性保証
- OSSモデルのライセンス管理
- → **RAIフレームワーク適用**

### S2-5. FY26実証ターゲットとマイルストーン（15分）

**スライド案:** 彩都の作業フロー → α/β定義 → 4つの柱 → タイムライン

**台本（日本語）:**

FY26の実証ターゲットは彩都パーツセンターの出庫作業です。

作業フロー: 納品書スキャン → オートストアからビン到着 → ビン内パーツ取り出し → 個装 → 台車配置。1オーダー1-3分。

12万品番、ビン内は段ボール＋プチプチ＋多種多様なパーツが混在。VLAによる学習ベースの操作制御が最初のステップから不可欠です。

**α版（9月）:** VLAピックスキルがRoboSync上で動作。ビン内パーツのピック→作業台配置。成功率90%目標。

**β版（3月）:** 出庫サイクル全体（スキャン→ピック→個装→配置）をAgentic制御で動作。

**4つの柱：**
1. RoboSync × VLA統合 — VLAスキルブロック化
2. DataOps — 継続学習パイプライン（**← Microsoft協業の中核**）
3. Agent Operation — 失敗検知・再計画・スキル切替
4. Field PoC — 彩都パーツセンター実証

Microsoftとの協業で特に加速したいのは、柱②のDataOps基盤と、柱①におけるシミュレーション連携です。
