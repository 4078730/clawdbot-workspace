**期初設定:** テーマや部門の枠を超えて協働を促し、プロジェクト全体の技術的成果と事業価値を両立する。

  - SDL FUK / CXC / CEC / PHD / DAICC等との社内横断連携を日常的に実施
- 池内先生（Microsoft/東大名誉教授）との定期ディスカッションを9月から主導し、IndexVLAの理論的基盤をRFAアーキテクチャに接続

- iREXでのパナソニック初VLAアウトプットを実現し、プレス掲載を通じて社外認知を獲得。その後のDeepMind連携、Zetes/Flexis/G-SOL等への発信につながった

## 🔷 Empathy（誠実さ・共感）

**期初設定:** 対話を通じてメンバーや他部門の立場や課題を深く理解し、丁寧かつ建設的な連携を重ねる。

### 振り返り

- 新メンバーのオンボーディングにリーダー・上司として注力
  - 黒川さん（画像認識系からの異動）: 技術背景を踏まえた段階的な任務設計。Keypoint-VLAの論文アイディアを付与し、RSJ投稿・発表まで一貫して担当する機会を提供
  - Zhuさん（新入社員）: LLM Agentからスタートし、強みと技術的優先度を見ながらVLA/MLOps方面へ段階的に育成
- プロジェクトレビュー・メンバー1on1（ABD）を通年で継続実施し、個々の状況やキャリア志向を把握
- 石野さん（企画）との対話の場を設け、「会社から求められている技術」「自分たちの立ち位置」をメンバーに共有。研究開発と事業戦略の文脈をチームが理解できるよう配慮
- 村上さんのEMNLP不採択後も前向きなフィードバックを行い、ブラッシュアップを支援
- ファミリーデーでのロボット体験会を企画し、社員の家族にも技術の面白さを体験してもらう機会を創出

### 課題・改善点

- 黒川さんに対して「もう少し積極性を持って、妥協せずにやり切る姿勢を引き出す」ためのアプローチは改善余地あり。心配性な性格を良い方向に活かせる指導方法を模索中

## 🔷 Results（顧客課題解決）

**期初設定:** 現場課題と技術シーズを結びつける視点を持ち、使える・活かせる技術開発にこだわる。

### 振り返り

- ユースケース探索を本格化し、「研究のためのデモ」から「現場で価値を出すPoC」への転換を推進
  - ファンケル物流会社を訪問（10月、黒川さん企画）し、現場の梱包・入出庫作業を体感
  - 彩都パーツセンターを訪問し、出庫作業のヒアリングを実施。12万品番・多様な形状のピッキングを動画分析し、VLAが本質的に不可欠であることを確認
  - 神戸工場（CPS本部シンさん経由）へのヒアリングを実施
  - 彩都ピッキングをFY26リーディングタスクとして選定。α/β版定義により技術・事業両面の一本化を実現
- CXCデモでは来場者視点を意識した設計（ギフト梱包・返品対応という身近なシナリオ）を採用
- iREXでは「シェーバーの梱包」という具体的な物流タスクを選定し、VLAの実用的価値を訴求
- 福岡VLA実証では、UR5eでの実環境ビンピッキングを通じて、現場適用の技術的課題を自ら体感

### 課題・改善点

- 福岡実証では性能向上まで至らなかった。現場課題と技術ギャップの解消にはFY26のMS協業MLOps基盤が鍵

## 🔷 Relentless（学習・継続的挑戦）

**期初設定:** 変化の早い技術環境で継続的にキャッチアップし、実装・検証・発信に結びつける。

### 振り返り

- LLM-Robotics勉強会を自ら企画し、毎週継続実施。全社的な技術共有の場を定着させた
- Automatica 2025（ドイツ・ミュンヘン）を1週間視察し、グローバルなロボティクス業界の最新動向をキャッチアップ → 社内共有
- π0 / OpenVLA / RT-1 / Octo等のOSSロボット基盤モデルの調査を継続し、RFAアーキテクチャへの反映を推進
- VLA Cosmos（シミュレーション/ワールドモデルによるデータオーギュメンテーション）の初期検討を開始（前野さん・松本さんチーム）
- DeepMindとの連携を通じて、クローズドモデルへのアクセスを獲得し、最先端技術の評価環境を確保
- UXDAワークショップに参加し、技術以外の視点（UX設計）の学習にも取り組んだ
- コネクトハッカソン（3日間）に参加し、Connect内外メンバーとの協働を通じて新しい開発スタイルを体験

- IndexVLA論文の投稿が遅延しており、研究成果の対外発信において計画通り進められなかった点は反省
