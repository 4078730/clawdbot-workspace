## IndexVLA 論文プロジェクト

### 基本情報
- **タイトル:** IndexVLA: 上位計画とVLA制御を接続する空間インターフェースの設計
- **英題:** IndexVLA: Designing the Spatial Interface between High-Level Planning and VLA Control
- **ターゲット:** IEEE RA-L (Best Paper Award水準)
- **現状:** 日本語フルペーパー骨子・詳細記述完了
- **次フェーズ:** 英語化（翻訳・推敲）、図表作成、追加実験

### コアコンセプト（勝ち筋）
- **課題:** VLAは言語指示（意味）を理解するが、空間的詳細が欠落（Target Specification Gap）
- **アプローチ:** 3種の記号（P2P / Trajectory / Vector）を比較検証
- **最大の発見（Hook）:**
  - 直感に反して、情報量の多いTrajectoryより P2P が contact tasks で勝利
  - 理由: Trajectoryの「強い制約（Rigidity）」がRecoveryを阻害
  - P2Pの「緩い制約」がモデルの自律修正能力を引き出す
- **結論:** P2P = Brain（Planner）と Body（VLA）をつなぐ最適インターフェース（Sweet Spot）

### 背景（AffordVLAからのPivot）
- 元々はVLMによるアフォーダンス自動生成を目指していた
- Pivot: Sender側（生成）→ Receiver側（VLAにとって最適な信号は何か）
- Discussionで再統合: 「P2PはVLMにも生成しやすい」→ システム全体最適

### 執筆方針
- **石原尚メソッド:** 構造改良→整理整頓→意味確定
- **英語化:** 直訳NG、英語論文のロジックとして再構成、冗長表現削除
- **用語統一:**
  - Brain = High-Level Planner / VLM Agent
  - Body = VLA Model / Low-Level Control
  - Constraint = Rigidity of the spatial instruction
  - Recovery = Autonomous error correction using visual feedback
