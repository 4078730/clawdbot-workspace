## 1. プロジェクト内容

**「研究フェーズ」から「現場PoC/事業検証フェーズ」へ。**

### 1.1 ビジョン

人の指示や意図を理解し、製造現場における「属人的」かつ「複雑な推論」を要するタスクを自律実行する知能システムを実現する。

### 1.2 解決する課題

- **非定型作業の多さ**：形状・サイズ・素材が多種多様な物品を扱い、事前にすべてのパターンをプログラムすることは現実的でない
- **暗黙知に基づく柔軟な判断**：「重いものは下」「壊れ物は上」等の暗黙ルール、把持失敗時の持ち替えなど言語化困難な判断
- **人手不足の深刻化**：物流・製造現場の労働力不足は年々深刻化、属人作業の自動化は喫緊の課題

### 1.3 事業価値

- **SCM領域の労働課題解決**：入出庫・梱包等の属人作業を自動化し、省人化・品質安定・24h運用を実現
- **競争優位性確立**：Robo Sync導入現場からのデータ蓄積により、他社追随困難な技術的優位性を構築
- **横展開効果**：Robo Syncをベースのコア技術として本技術を提供しスケール化

### 1.4 実行戦略：4つの柱

| 柱 | 内容 |
|----|------|
| **① Robo Sync × VLA** | VLAをRobo Sync上のスキルブロックへ昇格、実行基盤確立 |
| **② DataOps** | 現場データを学習資産として蓄積、継続学習基盤構築 |
| **③ Agent Operation** | 失敗・例外前提の再計画・適応実行構造 |
| **④ Field PoC** | 神戸/彩都PoC、事業仮説検証、FY27判断材料整備 |

### 1.5 FY26のゴール

「研究フェーズ」から「現場PoC/事業検証フェーズ」へ移行する。

**達成目標:** VLA-Agent + RoboSync + DataOps が技術/運用の両面で「現場で回る」ことを実証する。

→ FY27の事業化判断エビデンスの確立。

### 1.6 勝ち筋

「Robo Syncを通じた現場データの資産化」が差別化の源泉。

「使いながら賢くなる」RFAを実現し、SCM/製造現場の属人課題を解決し、新たな事業の柱を確立する。
